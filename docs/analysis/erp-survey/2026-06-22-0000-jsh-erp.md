## 2. 模块拆分方式

单 SpringBoot 应用（`jshERP-boot`），无微服务拆分。包结构按业务域：

- `com.jsh.erp.datasource.entities`（实体，全部在 `jshERP-boot/.../datasource/entities/`）
- `com.jsh.erp.datasource.mappers`（MyBatis mapper）
- `com.jsh.erp.service`（业务服务，每域一个 Service）
- `com.jsh.erp.controller`（REST 控制器）

## 3. 核心业务实体清单

### 进销存单据（`datasource/entities/DepotHead.java` + `DepotItem.java`）

**`DepotHead`**（出入库单据头）——**单据类型双维度分类**：
- `type`（入库/出库/其它）+ `subType`（采购订单/采购/采购退货/零售/销售/销售退货/调拨/盘点录入/盘点复盘/组装单/拆卸单/请购单）—— 见 `constants/BusinessConstants.java:40-99`
- **采购和销售靠同一张表 + subType 区分**，而非分表

财务相关字段：
- `changeAmount`（结算账户付款额）、`backAmount`（退款）、`totalPrice`（合计）
- `discount`/`discountMoney`/`discountLastMoney`（折扣三件套）
- `otherMoney`（其他费用）、`deposit`（订金）
- `debt`/`lastDebt`（本次欠款/最终欠款）
- `accountId`（单账户）、`accountIdList`+`accountMoneyList`（**逗号分隔字符串表达多账户多金额，反范式设计**）
- 状态：`status`（0未审核/1审核/2完成/3部分）、`purchaseStatus`、`payType`（预付款/现付）、`linkNumber`/`linkApply`（关联订单号）

**`DepotItem`**（出入库单据行）：
- `materialId`/`materialExtendId`（SKU 引用）、`materialUnit`/`sku`
- `operNumber`（单据数量）/`basicNumber`（基本数量，单位换算后）
- `unitPrice`/`purchaseUnitPrice`/`taxUnitPrice`/`allPrice`（多档价格）
- `taxRate`/`taxMoney`/`taxLastMoney`（税三件套）
- `depotId`/`anotherDepotId`（调拨双仓库）
- `snList`（序列号列表）、`batchNumber`/`expirationDate`（批次/效期）
- `linkId`（关联源单明细 ID）

### 财务账户流水（`AccountHead.java` + `AccountItem.java`）

**`AccountHead`**：
- `type`（收款/付款/转账/收预付款）、`organId`（往来单位）、`handsPersonId`（经手人）
- `changeAmount`、`discountMoney`、`totalPrice`、`accountId`、`billNo`、`billTime`、`status`、`source`（来源单据号）

**`AccountItem`**：
- `accountId`、`inOutItemId`（收支项目）、`billId`（关联业务单据 ID）
- `needDebt`（应收/应付欠款）、`finishDebt`（已结清欠款）、`eachAmount`（本次金额，**收款正数/付款负数**）

### 物料主数据（`Material.java` + `MaterialExtend.java` + `MaterialCategory.java`）

**`Material`**：
- `categoryId`、`name`/`mfrs`/`model`/`standard`/`brand`/`color`/`unit`/`unitId`（基本单位）
- `mnemonic`（助记码）、`expiryNum`（保质期天数）、`weight`
- `enableSerialNumber`/`enableBatchNumber`（序列号/批次开关）
- `position`（库位）、`attribute`（扩展属性 JSON）

**`MaterialExtend`**（**SKU 价格表，多单位多 barcode**）：
- `materialId`、`barCode`、`commodityUnit`（包装单位）、`sku`
- `purchaseDecimal`/`commodityDecimal`/`wholesaleDecimal`/`lowDecimal`（**采购价/零售价/批发价/最低价四档价格**）
- `defaultFlag`（是否默认单位）
- **一个物料可挂多个 SKU（单位×条码）**，`DepotItem.materialExtendId` 直接引用 SKU

**`MaterialCategory`**：树形分类。

### 基础数据

- **`Supplier`**（**供应商/客户一体表**）：`supplier`、`type`（类型区分供需）、`advanceIn`（预付款余额）、`beginNeedGet`/`beginNeedPay`（期初应收/应付）、`allNeedGet`/`allNeedPay`（累计应收/应付）、`taxNum`/`bankName`/`accountNumber`/`taxRate`（开票信息）
- `Depot`（仓库）、`Organization`（组织机构树，`parentNo=-1` 为根，状态码 1-5 表示未开业/营业/停业/终止/注销）
- **`SerialNumber`**（**序列号台账**）：`materialId`/`depotId`/`serialNumber`/`isSell`（0未售1已售）、`inPrice`、`inBillNo`/`outBillNo`（入库单号/出库单号双向回链）——**序列号是独立表，出入库时带 `snList`**

## 4. 库存扣减与业财打通机制

### 库存：余额汇总表（非流水）
- `MaterialCurrentStock`（`materialId+depotId` 唯一，存 `currentNumber`/`currentUnitPrice`）
- `DepotItemService.updateCurrentStock()`（`:1106`）在 `DepotHeadService.addDepotHeadAndDetail/updateDepotHeadAndDetail` 中循环调用
- `updateCurrentStockFun()`（`:1195`）通过 `getStockByParam()` **重算当前结存并 upsert**，同时刷新 `currentUnitPrice`（移动加权平均）
- 出入库方向由 `type`/`subType` 决定，数量通过 `operNumber → basicNumber`（单位换算）落账

### 财务：账户流水代数和 = 余额（无借贷凭证）
- 采购/销售单据直接在 `DepotHead` 上写 `accountId`/`accountIdList`/`accountMoneyList`
- `AccountService.getManyAccountSum()`（`:322`）解析逗号字符串汇总账户进出
- 独立收付款单走 `AccountHead`/`AccountItem`，通过 `billId` 回链业务单据并写 `needDebt`/`finishDebt` 完成核销
- **账户余额 = 各账户流水的代数和**（`getAccountSumByHead + getAccountSumByDetail + getManyAccountSum`）

### 欠款闭环公式（`DepotHeadService.java:185/713/1089/1723`）

```
debt = discountLastMoney + otherMoney − (deposit + changeAmount)
```

即：应付总额 = 折后金额 + 其他费用 − 订金 − 已结算。`allNeedGet`/`allNeedPay` 累计到 `Supplier` 表。

## 5. 对 nop-app-erp 的可借鉴设计点

| # | 借鉴点 | 管伊佳证据 | 对 nop 的落地建议 |
|---|---|---|---|
| 1 | **出入库单据"双维度类型"单表设计** | `DepotHead.type` + `subType`（`BusinessConstants.java:40-99`）；采购销售共用一表 | nop 进销存单据可用 `docType`(入库/出库/其它) + `bizType`(采购/销售/调拨/盘点...) 双维度字典，所有出入库单据共用头行表 |
| 2 | **物料 + SKU(多单位多barcode)分离** | `Material` + `MaterialExtend`（四档价格 + 默认单位标志） | nop 物料主数据拆 `Material`(基础属性) + `MaterialSku`(单位+条码+多档价格)，单据引用 SKU |
| 3 | **供应商/客户一体表 + 应收应付余额字段** | `Supplier.type` + `beginNeedGet/Pay` + `allNeedGet/Pay` + `advanceIn` | nop 往来单位主数据直接承载应收应付余额，配合核销子表形成欠款闭环 |
| 4 | **序列号台账独立表 + 双向回链** | `SerialNumber.isSell` + `inBillNo`/`outBillNo` | nop 序列号管理做独立台账表，出入库时带 snList，双向回链源单 |
| 5 | **行级税额独立列（价税分离）** | `DepotItem.taxRate`/`taxMoney`/`taxLastMoney`/`unitPrice`/`taxUnitPrice`/`allPrice` | nop 明细表内建税字段三件套，便于增值税核算和价税分离凭证 |
| 6 | **多账户多金额的规范化改造** | `accountIdList`+`accountMoneyList`（逗号字符串，反范式） | **反例参考**：nop 应改为 `depot_account_allocation(head_id, account_id, amount)` 子表，不要用逗号字符串 |
| 7 | **欠款闭环公式** | `DepotHeadService.java:185`（`debt = discountLastMoney + otherMoney − deposit − changeAmount`） | nop 应付/应收计算明确化：折后+其他费用−订金−已结算 |
| 8 | **库存余额汇总表 + 重算** | `MaterialCurrentStock` + `DepotItemService.updateCurrentStock()` | nop 可用流水明细为主、余额表为物化缓存；管伊佳"重算"方式适合中小规模 |
| 8a | **可选插件式多级审批** | `SystemConfig.multiLevelApprovalFlag` + `WorkflowIframe.vue` | 详见 `2026-06-22-0000-workflow-vs-state-machine.md`——管伊佳通过 SystemConfig 开关控制是否启用多级审批插件，启用后前端显示 /workflow 菜单调用外部 API |

## 6. 不建议借鉴的点

- **无借贷凭证，仅靠账户流水代数和 = 余额**——适合小微进销存但做不出标准三表（资产负债表/利润表/现金流量表）。nop-app-erp 若要"财务一体化"应采用赤龙的 Voucher/BillR/Model 三件套（见 `2026-06-22-0000-redragon-erp.md`），而非管伊佳的轻财务。
- **逗号分隔多账户字符串**（`accountIdList`/`accountMoneyList`）是反范式设计，查询需解析，不可照搬。
- **库存余额靠"重算 upsert"**而非流水驱动物化视图，在并发场景下有数据一致性风险；nop 应该用流水驱动的余额更新。
