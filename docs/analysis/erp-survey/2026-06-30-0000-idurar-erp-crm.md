# IDURAR ERP CRM 调研报告

> IDURAR 是基于 Node.js + React + MongoDB 的开源 ERP/CRM 系统，8504 Stars。聚焦于会计、发票、CRM 和项目管理，适合中小企业。前端使用 Ant Design + React，后端使用 Express + MongoDB。

## 基本信息

| 属性 | 值 |
|------|-----|
| 项目 | idurar-erp-crm |
| 组织 | idurar |
| Stars | 8504 |
| License | AGPL-3.0 |
| 技术栈 | Node.js / Express / React / MongoDB / Ant Design |
| 架构 | 前后端分离（backend/frontend） |

## 模块结构

### 后端模型（backend/src/models/）

**appModels（业务模型）**：
- 会计/发票相关模型
- CRM 相关模型
- 用户/权限模型

**coreModels（核心模型）**：
- 基础数据
- 系统配置
- 国际化

### 后端路由（backend/src/routes/）

**appRoutes（业务路由）**：
- 发票（Invoice）CRUD
- 付款（Payment）CRUD
- 报价单（Quote）CRUD
- CRM 相关路由

**coreRoutes（核心路由）**：
- 认证
- 用户管理
- 系统配置

### 前端模块（frontend/src/modules/）

| 模块 | 功能 |
|------|------|
| AuthModule | 登录/注册/权限 |
| CrudModule | CRUD 通用组件 |
| DashboardModule | 仪表盘/统计 |
| ErpPanelModule | ERP 业务面板 |
| InvoiceModule | 发票管理（创建/编辑/列表/打印） |
| PaymentModule | 付款管理 |
| ProfileModule | 用户配置 |
| QuoteModule | 报价单管理 |
| SettingModule | 系统设置 |

## 对 nop-app-erp 的参考价值

| 方面 | IDURAR | 本项目 | 结论 |
|------|--------|-------|------|
| CRM Lead→Quote 流 | 完整 | 设计骨架 → 完整 | IDURAR 有完整的 Lead→Quote 转化和发票生成，可作 CRM 细化参考 |
| 发票/PDF 打印 | Invoice + React Print | nop-report | 前端打印方案可参考 |
| 报价单管理 | Quote 审批流 | sales 域已有 | 对本项目价值有限 |
| 技术栈 | Node.js/MongoDB | Java/ORM | 技术栈不同，架构参考价值低 |

### 核心结论

IDURAR 的功能范围较窄（仅 CRM/会计/发票），对 nop-app-erp 的补充参考价值有限，但 **CRM 的 Lead→Quote→Invoice 端到端流**可验证我们 CRM 骨架设计的方向。
