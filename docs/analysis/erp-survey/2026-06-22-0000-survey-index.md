## 选型决策建议（面向 nop-app-erp 设计参考）

### 场景 A：设计业财一体化的财务模块
**首选参考**：赤龙 ERP（凭证三件套）+ Metasfresh（凭证引擎现代化）
- 理由：赤龙的中式复式记账字段最完整；Metasfresh 的类型安全注册 + EventBus 异步过账是工程演进最佳实践

### 场景 B：设计进销存单据模型
**首选参考**：管伊佳（双维度单表）+ 若依（三单链）+ Odoo（库存三层）
- 理由：管伊佳的 type+subType 单表设计简洁；若依的 order→trade→refund 闭环清晰；Odoo 的 move/picking/quant 三层是行业标杆

### 场景 C：设计主数据
**首选参考**：管伊佳（Material+SKU+Supplier 一体）
- 理由：物料/SKU 多单位多 barcode 分离、供应商客户一体表 + 应收应付余额字段，是国产标准方案

### 场景 D：设计多公司/多币种/多科目表
**首选参考**：iDempiere
- 理由：AD_Client/AD_Org 双层租户 + C_AcctSchema 多套科目表 + FactLine 双币种字段，25 年验证

### 场景 E：设计模块化策略
**首选参考**：ERPNext（粗模块）+ 星云（单体/微服务双形态）
- 理由：ERPNext ~15 粗模块 vs Odoo 625 细 addon，前者更适合单一团队；星云的双形态部署策略适合 nop 演进
- **详见** `2026-06-22-0000-module-split-comparison.md` 与 `docs/architecture/domain-module-split-analysis.md`

### 场景 F：设计 CRM（Lead→Opportunity→Quote 完整流）
**首选参考**：**Axelor CRM** + IDURAR（对照验证）
- 理由：Axelor 有完整地 Lead→Convert→Opportunity 转化流程 + Event/Meeting 活动记录；IDURAR 的 Lead→Quote→Invoice 端到端流可验证设计方向

### 场景 G：设计人力资源/薪酬/考勤
**首选参考**：**Axelor human-resource（421 Java 文件）** + AureusERP employees/recruitments/time-off
- 理由：Axelor 覆盖完整 HR 生命周期（员工/合同/薪酬/个税/工时/休假），AureusERP 的招聘和休假管理独立可拆分
