# 若依 ERP（zybfly 版）调研报告

> 另一社区版若依 ERP，README 目标"打造中国一流的 ERP 平台"。与 zccbbg 版同源不同实现，**前端自带 `ui-admin` 部署更一体化，但社区活跃度低、设计深度不及 zccbbg 版**。参考价值有限。

## 1. 基本信息

| 项 | 值 |
|---|---|
| 技术栈 | SpringBoot 3.2.3 · JDK 17 · `ui-admin` 前端 |
| License | 开源 |
| 定位 | "中国一流 ERP 平台"（社区目标） |
| 源码规模 | 约 64 MB · 1228 文件 |

## 2. 与 zccbbg 版的区别

| 维度 | zccbbg 版 | zybfly 版（本报告） |
|---|---|---|
| 前端 | 分离的 Vue 项目 | **自带 `ui-admin` 前端**，部署更一体化 |
| 社区活跃度 | 较高 | 较低 |
| 表结构 | 38 张表，`order/trade/refund` 三单链清晰 | 未深入（源码体量大但业务设计深度不及） |
| 设计完整度 | 进销存三单链 + 财务四件套 | 偏脚手架堆砌 |

## 3. 模块结构

- 后端基于若依脚手架（`sys_*` 表 + `ruoyi-common` + `ruoyi-modules`）
- 前端 `ui-admin/` 自带（一体化部署）
- 具体业务表与字段未在本次调研中深入提取（zccbbg 版更具代表性，详见 `2026-06-22-0000-ruoyi-erp.md`）

## 4. 对 nop-app-erp 的可借鉴设计点

| # | 借鉴点 | zybfly 版证据 | 对 nop 的落地建议 |
|---|---|---|---|
| 1 | **前端一体化部署** | 自带 `ui-admin/` | nop 的 AMIS 前端随 `app-erp-web` 模块一体化部署，无需分离工程 |
| 2 | （其余参考价值同 zccbbg 版） | — | 见 `2026-06-22-0000-ruoyi-erp.md` |

## 5. 不建议借鉴的点

- **社区活跃度低**，长期维护风险高。
- **设计深度不及 zccbbg 版**，作为参考优先级低。
- 同样存在若依脚手架与 nop 重复、MyBatis 不可混用的问题。
