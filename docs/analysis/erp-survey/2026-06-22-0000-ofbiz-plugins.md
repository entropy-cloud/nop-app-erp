# Apache OFBiz Plugins 调研报告

> Apache OFBiz 的插件仓库，包含固定资产维护（assetmaint）、项目管理（projectmgr）、Scrum 敏捷模块。**对 nop-app-erp 的设备维护域和项目管理域有参考价值，尤其是维护调度与工时模型。**

## 1. 基本信息

| 项 | 值 |
|---|---|
| 技术栈 | Java · Apache OFBiz 框架 · Groovy 服务 · XML 实体定义 |
| License | Apache-2.0 |
| 定位 | OFBiz 生态的可插拔业务模块 |
| 插件数 | 16 个（含 assetmaint、projectmgr、scrum 等） |

## 2. Asset Maintenance（固定资产维护）

**核心设计**：不定义新实体，复用 OFBiz 框架的 `FixedAsset`、`FixedAssetMaint`、`WorkEffort` 实体。

### 关键视图实体（`entitydef/entitymodel.xml`）

| 视图实体 | 行号 | 职责 |
|----------|------|------|
| `FixedAssetAndIdent` | :31 | 资产+标识信息（序列号等） |
| `FixedAssetMaintWorkEffortAndPartyAssignment` | :41 | 维护记录+工作计划+人员分配 |
| `FixedAssetMaintWorkEffortAndTimeEntry` | :69 | 维护记录+工作计划+工时记录 |
| `ItemIssuanceInventoryItemAndProduct` | :93 | 维护工单的备件领用 |

### 关键服务（`servicedef/services.xml`）

| 服务 | 行号 | 职责 |
|------|------|------|
| `addPartFixedAssetMaint` | :22 | 为维护记录添加备件 |
| `createFixedAssetMaintUpdateWorkEffort` | :38 | 创建维护记录并更新工作计划 |
| `updateFixedAssetMaintAndWorkEffort` | :46 | 同时更新维护记录和工作计划 |

### 维护调度机制

`FixedAssetMaint.scheduleWorkEffortId` → `WorkEffort`，通过标准的 `WorkEffortPartyAssignment` 实现人员分配。折旧计算在 OFBiz 框架核心实现（`FixedAssetDepr` 实体），本插件只做维护调度。

## 3. Project Management（项目管理）

**核心设计**：复用 `WorkEffort` 实体，通过 `workEffortParentId` 构建 Project → Phase → Task 三级层次。

### 关键视图实体

| 视图实体 | 行号 | 职责 |
|----------|------|------|
| `ProjectAndPhaseAndTask` | :184 | 项目→阶段→任务三级层次 |
| `ProjectPhaseTaskAndTimeEntryTimeSheet` | :217 | 任务+工时+周 timesheet |
| `TimesheetAndTimeEntry` | :444 | 周 timesheet（hoursDay0-6） |
| `ProjectPhaseTaskActualRatedHoursView` | :401 | 带费率的工时计算 |

### 工时模型

- `TimeEntry`：工时记录（日期+小时数）
- `Timesheet`：周汇总（hoursDay0-6 对应周一到周日）
- `PartyRate`：人员费率，用于计算人工成本

### 关键服务

- `createTimeEntryInTimesheet`：自动创建周 timesheet
- `updateTimeEntryByWorkEffort`：批量更新 7 天工时

## 4. Scrum 敏捷模块

在 `WorkEffort` 上叠加敏捷视图：
- `ScrumProjectSprintBacklog`：Sprint 待办
- `ProductBacklog`：产品待办
- 通过 `CustRequest` → `CustRequestWorkEffort` → `WorkEffort` 关联

## 5. 对 nop-app-erp 的可借鉴设计点

| # | 借鉴点 | OFBiz 证据 | 对 nop 的落地建议 |
|---|--------|-----------|-------------------|
| 1 | **维护+工单+工时三表关联** | FixedAssetMaint + WorkEffort + TimeEntry | nop 维护工单关联工时记录，支持人工成本归集 |
| 2 | **备件领用与维护工单关联** | ItemIssuance → InventoryItem | nop 维护工单可领用备件，自动扣减库存 |
| 3 | **项目三级层次** | WorkEffort + workEffortParentId | nop 项目→任务用 parent 关联，不需独立表 |
| 4 | **周 Timesheet 模型** | Timesheet（hoursDay0-6） | nop 工时按周汇总，每天一个字段（适合固定工时制） |
| 5 | **费率计算人工成本** | PartyRate × TimeEntry | nop 工时 × 人员标准费率 = 人工成本 |
