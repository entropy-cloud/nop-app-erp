# Carbon ERP+MES+QMS 调研报告

> 开源 ERP+MES+QMS 一体化平台，专为制造业设计。**对 nop-app-erp 的制造域、质量域、固定资产域均有重要参考价值——尤其是质量管理（QMS）模块，是 18 个项目中最完整的质量管理体系。**

## 1. 基本信息

| 项 | 值 |
|---|---|
| 技术栈 | React Router v7 · TypeScript · Supabase (PostgreSQL) · Kysely · Tailwind CSS · Radix UI · Inngest（后台任务）· SST（AWS 部署） |
| License | 开源 |
| 定位 | 制造业 ERP+MES+QMS 一体化 |
| 架构 | Monorepo（pnpm + Turborepo），ERP 和 MES 是两个独立 App |

## 3. 质量管理模块（QMS）— 18 个项目中最完整

**模块路径**：`apps/erp/app/modules/quality/`

### 核心实体

| 实体 | 文件:行号 | 职责 |
|------|-----------|------|
| **InspectionDocument** | `quality.types.ts` | 检验文档，支持 PDF 标注（Balloon/Feature） |
| **Balloon** | `quality.types.ts` | PDF 上的检验区域标注 |
| **Gauge** | `quality.models.ts:373-425` | 量具管理，含校准记录与状态追踪 |
| **NonConformanceIssue** | `quality.models.ts:462-485` | 不合格品问题，含状态工作流、审批任务 |
| **QualityDocument** | `quality.models.ts:587-665` | 质量文档，含版本化步骤与审批 |
| **RiskRegister** | `quality.models.ts:677-700` | 风险登记，严重性×可能性矩阵 |
| **InboundInspection** | `quality.models.ts:702-775` | 来料检验，含 AQL 抽样方案 |
| **SamplingPlan** | `quality.types.ts` | 抽样计划 |

### 质量 KPI

- Issue trends（问题趋势）
- Pareto analysis（帕累托分析）
- Supplier quality（供应商质量）
- Weeks-open tracking（问题开放周数追踪）

### 不合格品处理

`NonConformanceIssue` 关联到：Items, Customers, Suppliers, JobOperations, POLines, SOLines, Shipments, Receipts, TrackedEntities, InboundInspections——覆盖全业务链。

## 7. 对 nop-app-erp 的可借鉴设计点

| # | 借鉴点 | Carbon 证据 | 对 nop 的落地建议 |
|---|--------|------------|-------------------|
| 1 | **QMS 全套实体** | InspectionDocument + Balloon + Gauge + NonConformance + RiskRegister | nop 质量域照搬：检验文档+量具+不合格品+风险登记 |
| 2 | **来料检验+AQL 抽样** | InboundInspection + SamplingPlan | nop 来料检验支持 AQL 抽样方案 |
| 3 | **不合格品关联全业务链** | NonConformanceIssue 关联 10+ 实体 | nop 不合格品可关联采购/销售/生产/库存 |
| 4 | **MES 工位质检** | steps.inspection.tsx | nop 工序执行中可触发质检 |
| 5 | **故障模式分析** | FailureMode（production 模块） | nop 工单可记录故障模式，支持帕累托分析 |
| 6 | **BOM+配置+版本** | Item + MakeMethod + BOM + Revision | nop BOM 支持版本管理和配置规则 |

## 8. 不建议借鉴的点

- **TypeScript 技术栈**：nop 用 Java，需注意语言差异
- **Supabase**：nop 用 MyBatis + MySQL/PostgreSQL
- **ERP 和 MES 分离部署**：nop 可考虑合一或分离，按需决定
