# 输入索引

使用此目录存放原始外部输入。

示例：

- PM 笔记
- 卡片集文档
- 原型笔记
- 复制的用户反馈
- 文章摘录
- 带注释的截图
- 旧系统行为摘要

这些文件不是自动实现就绪的。

从 `00-input-processing-guide.md` 开始。

## 当前状态（L-4，plan 2026-07-20-2200-1）

> **本目录当前为空（仅含指南文件）是设计选择，不是遗漏。**

`nop-app-erp` 处于**业务逻辑深化与运营成熟度收尾阶段**（详见 `AGENTS.md §当前项目阶段`）：

- 18 业务域 + 1 跨域通知派发子系统的 ORM 模型已设计完成并经多轮审计
- CRUD 全 18 域、核心业务逻辑、扩展 13 域业务逻辑、业财一体端到端、运营成熟度、报表子系统、看板子系统均已落地
- 当前重点：看板运行时视觉/浏览器回归、各域细化端到端验证

**已过需求澄清期**：项目已通过 `docs/requirements/`（实现就绪需求综合）+ `docs/design/`（稳定应用层设计）+ `docs/architecture/`（技术设计）三层层级完成需求综合。新需求直接进入 `docs/discussions/` 或 `docs/requirements/`，不再经过 `docs/input/` 的原始材料阶段。

**何时会重新使用本目录**：

- 引入新业务域（如未来添加 ERP 第三批扩展域）需要消化 PM 笔记/原型截图时
- 集成外部系统（如银行接口、税务接口）需要保留对方提供的文档原文时
- 用户提出新需求且无法直接综合为 `docs/requirements/` 时

历史已使用过的输入材料在综合完成后或归档到 `docs/archive/`，或直接转化为 `docs/requirements/` 与 `docs/design/` 的稳定产出。
