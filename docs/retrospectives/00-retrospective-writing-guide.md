# 回顾编写指南

## 目的

当团队需要理解为什么即使在实施工作完成后交付仍然未达到预期时，使用 `docs/retrospectives/`。

## 适用场景

- 原型和实现的差异超出预期
- 需求来源不足
- PM 到开发的交接跳过了分析工作
- 相同的流程差距不断重复出现
- 应该提取新的可复用技能或审计提示

## 包含内容

- 原始源输入
- 缺少或误导的内容
- 仅在实施过程中发现的内容
- 下次应提前在流程中移动的内容
- 应创建哪些新技能、审计提示或工作流规则

## 文件名指南

优先使用日期文件名：

- `docs/retrospectives/YYYY-MM-DD-HHmm-topic.md`
