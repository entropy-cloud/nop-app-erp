# 应收应付核销机制

## 目的

说明 ERP 系统中应收应付核销的业务模型、实现机制与账龄分析。核销是确认债权债务清偿的核心环节，保证应收应付余额的准确性。

## 核销模型

### 核销对象

| 对象 | 说明 | 业务域 |
|------|------|--------|
| 应收发票（SalesInvoice） | 确认对客户的应收债权 | sales |
| 收款单（Receipt） | 收取客户款项 | sales |
| 应付发票（PurchaseInvoice） | 确认对供应商的应付债务 | purchase |
| 付款单（Payment） | 向供应商支付款项 | purchase |

### 核销关系

核销是发票与收付款之间的多对多关系：

```
发票（应收/应付）
    │
    ├─► 被付款全额核销 → 余额 = 0，状态 = 已核销
    │
    ├─► 被付款部分核销 → 余额 > 0，状态 = 部分核销
    │      └─► 继续核销至全额
    │
    └─► 多次核销 ← 付款可核销多张发票
```

### 核销明细表

核销关系通过 `ErpFinReconciliation` 表记录：

```
核销明细（Reconciliation）
  ├─ reconciliationId（核销ID）
  ├─ reconciliationType（AR/AP）
  ├─ sourceBillType（发票/付款）
  ├─ sourceBillId（源单据ID）
  ├─ targetBillType（付款/发票）
  ├─ targetBillId（目标单据ID）
  ├─ reconciliationAmount（核销金额）
  ├─ reconciliationDate（核销日期）
  ├─ reconciliationType（全额/部分）
  └─ reversalFlag（是否已冲销）
```

## 核销流程

### 应收核销流程（收款核销应收）

```
收款核销应收
        │
        ├─► 步骤1：选择收款单
        │      └─► 收款单已审核状态
        │
        ├─► 步骤2：选择待核销发票
        │      ├─► 查询该客户的未核销应收发票
        │      ├─► 按发票日期或到期日排序
        │      └─► 显示每张发票的未核销金额
        │
        ├─► 步骤3：输入核销金额
        │      ├─► 默认显示发票未核销金额
        │      ├─► 可调整核销金额（部分核销）
        │      └─► 系统自动计算剩余未核销金额
        │
        ├─► 步骤4：确认核销
        │      ├─► 生成核销明细记录
        │      ├─► 更新发票核销状态与余额
        │      ├─► 更新收款单核销状态与余额
        │      └─► 触发财务凭证生成（若配置）
        │
        └─► 步骤5：生成核销凭证（可选）
               ├─► 核销不直接生成凭证
               ├─► 凭证由收款单/付款单审核时生成
               └─► 核销影响应收应付余额的抵减
```

### 应付核销流程（付款核销应付）

```
付款核销应付
        │
        ├─► 步骤1：选择付款单
        │      └─► 付款单已审核状态
        │
        ├─► 步骤2：选择待核销发票
        │      ├─► 查询该供应商的未核销应付发票
        │      ├─► 按发票日期或到期日排序
        │      └─► 显示每张发票的未核销金额
        │
        ├─► 步骤3：输入核销金额
        │      ├─► 默认显示发票未核销金额
        │      ├─► 可调整核销金额（部分核销）
        │      └─► 系统自动计算剩余未核销金额
        │
        ├─► 步骤4：确认核销
        │      ├─► 生成核销明细记录
        │      ├─► 更新发票核销状态与余额
        │      ├─► 更新付款单核销状态与余额
        │      └─► 触发财务凭证生成（若配置）
        │
        └─步骤5：生成核销凭证（可选）
```

## 自动核销

### 自动核销规则

支持配置自动核销规则，减少手工操作：

| 规则类型 | 说明 | 配置项 |
|----------|------|--------|
| 按金额自动核销 | 到款金额匹配发票金额时自动核销 | `auto-match-exact-amount` |
| 按比例自动核销 | 按发票金额比例自动核销 | `auto-match-by-ratio` |
| 按账龄自动核销 | 优先核销账龄长的发票 | `priority-by-aging` |
| 按到期日自动核销 | 优先核销已到期的发票 | `priority-by-due-date` |

### 自动核销触发

- **手动触发**：用户主动执行自动核销
- **定时触发**：每日凌晨自动执行核销匹配
- **付款审核触发**：付款审核时自动匹配并核销

## 核销状态

### 发票核销状态

| 状态 | 说明 | 计算方式 |
|------|------|----------|
| 未核销（UNRECONCILED） | 发票金额全部未核销 | `balance = invoiceAmount` |
| 部分核销（PARTIAL） | 发票金额部分核销 | `0 < balance < invoiceAmount` |
| 已核销（RECONCILED） | 发票金额全部核销 | `balance = 0` |
| 超额核销（OVER） | 核销金额超过发票金额 | `balance < 0`（异常） |

### 收付款核销状态

| 状态 | 说明 | 计算方式 |
|------|------|----------|
| 未核销（UNRECONCILED） | 收付款金额全部未核销 | `balance = receiptAmount` |
| 部分核销（PARTIAL） | 收付款金额部分核销 | `0 < balance < receiptAmount` |
| 已核销（RECONCILED） | 收付款金额全部核销 | `balance = 0` |

## 余额计算

### 应收余额计算

```
客户应收余额 = Σ(应收发票金额) - Σ(收款核销金额) - Σ(红字发票冲减)

具体计算：
  1. 查询该客户的所有应收发票（AR_INVOICE）
  2. 查询该客户的所有红字发票（金额取负）
  3. 查询该客户的所有收款单核销明细
  4. 余额 = 发票金额合计 + 红字发票合计 - 核销金额合计
```

### 应付余额计算

```
供应商应付余额 = Σ(应付发票金额) - Σ(付款核销金额) - Σ(红字发票冲减)

具体计算：
  1. 查询该供应商的所有应付发票（AP_INVOICE）
  2. 查询该供应商的所有红字发票（金额取负）
  3. 查询该供应商的所有付款单核销明细
  4. 余额 = 发票金额合计 + 红字发票合计 - 核销金额合计
```

## 账龄分析

### 账龄分级

| 账龄区间 | 说明 | 风险等级 |
|----------|------|----------|
| 0-30天 | 正常账期 | 低 |
| 31-60天 | 逾期初期 | 中 |
| 61-90天 | 逾期中期 | 高 |
| 91-180天 | 严重逾期 | 很高 |
| 180天以上 | 坏账风险 | 极高 |

### 账龄计算基准

| 类型 | 计算起点 | 说明 |
|------|----------|------|
| 应收 | 发票日期 或 到期日 | 按配置选择 |
| 应付 | 发票日期 或 到期日 | 按配置选择 |
| 预收/预付 | 收款/付款日期 | 无到期日概念 |

### 账龄报表

账龄分析报表展示：

```
客户应收账龄分析
  ├─ 客户名称
  ├─ 发票编号
  ├─ 发票日期
  ├─ 到期日
  ├─ 发票金额
  ├─ 已核销金额
  ├─ 未核销余额
  ├─ 账龄（天）
  ├─ 账龄区间
  └─ 风险等级
```

## 核销冲销

### 冲销场景

| 场景 | 说明 |
|------|------|
| 收付款错误 | 收付款金额或对象错误，需要冲销重做 |
| 发票调整 | 发票金额调整，需要重新核销 |
| 退货冲销 | 销售退货生成红字发票，需要冲销原核销 |

### 冲销流程

```
核销冲销
        │
        ├─► 步骤1：选择要冲销的核销明细
        │      ├─► 查询历史核销记录
        │      └─► 选择需要冲销的条目
        │
        ├─► 步骤2：确认冲销
        │      ├─► 生成反向核销明细（reversalFlag = true）
        │      ├─► 恢复发票未核销余额
        │      ├─► 恢复收付款未核销余额
        │      └─► 记录冲销原因
        │
        └─步骤3：后续处理
               ├─► 重新核销（新金额）
               └─► 或保留发票待下次核销
```

## 业务规则

### 核销约束

1. **同方向核销**：应收与应收核销，应付与应付核销
2. **同客户/供应商**：发票与收付款必须是同一往来单位
3. **金额校验**：核销金额不能超过任一方未核销余额
4. **日期校验**：核销日期不能早于发票日期
5. **状态校验**：发票与收付款必须是已审核状态

### 核销分摊规则

当一笔收款/付款核销多张发票时，按以下优先级分摊：

| 分摊方式 | 规则 | 适用场景 |
|----------|------|----------|
| 按发票余额比例 | 按各发票未核销余额占总余额的比例分摊 | 默认方式 |
| 按发票日期顺序 | 从最早发票开始核销 | FIFO 核销策略 |
| 手工指定 | 用户手动指定每张发票的核销金额 | 特殊场景 |

> 分摊精度：每张发票的核销金额四舍五入到配置精度（`erp-fin.reconcile-precision`），尾差调整到最后一张发票。

### 汇兑损益核销规则

多币种核销时的汇兑处理：

- **核销时**：按核销当天汇率将源币种核销金额折算为本位币
- **差异处理**：源币种已核销金额 × 核销日汇率 - 原发票本位币金额 = 汇兑差异
- **汇兑差异凭证**：汇兑收益/损失计入当期财务费用（汇兑损益科目）
- **原币已核销**：发票的 `reconciledAmountSource` 记录已核销的源币种金额

> 汇兑损益核销凭证示例：借：银行存款（本位币） / 贷：应收（本位币） + 借/贷：财务费用-汇兑损益。

### 核销权限

| 操作 | 权限要求 |
|------|----------|
| 手工核销 | 财务员 |
| 自动核销配置 | 财务管理员 |
| 核销冲销 | 财务员 + 原因记录 |
| 账龄报表查看 | 财务员/管理层 |

## 与其他域的协作

| 对端域 | 协作内容 |
|--------|----------|
| sales | 销售发票生成应收、核销收款 |
| purchase | 采购发票生成应付、核销付款 |
| finance | 核销凭证生成、余额计算 |

## 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `erp-fin.ar-aging-base` | due_date | 应收账龄计算基准（invoice_date/due_date） |
| `erp-fin.ap-aging-base` | due_date | 应付账龄计算基准（invoice_date/due_date） |
| `erp-fin.auto-reconcile` | false | 是否启用自动核销 |
| `erp-fin.reconcile-precision` | 0.01 | 核销金额精度（容忍分摊误差） |
| `erp-fin.allow-over-reconcile` | false | 是否允许超额核销 |

## 异常处理

| 异常场景 | 处理 |
|----------|------|
| 超额核销 | 检查并拒绝，提示核销金额超限 |
| 跨客户/供应商核销 | 拒绝核销，提示往来单位不匹配 |
| 已结账期间核销 | 拒绝核销，需先反结账 |
| 并发核销冲突 | 使用悲观锁保证数据一致性 |
