# 财务域页面设计要点

> 本文档定义财务域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-finance.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`posting.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **借贷平衡实时校验**：凭证录入时借方合计与贷方合计实时显示并对比，不平衡时过账按钮禁用并给出明确提示。
2. **业财回链可见**：凭证详情页必须显示来源业务单据（回链可跳转），业务单据详情页必须显示关联凭证状态。
3. **期间控制硬约束**：已结账期间的所有财务操作（过账/红冲/修改）在前端即拦截并提示，避免后端才报错。
4. **辅助核算维度可扩展**：凭证分录行除了科目金额外，预留"往来单位/部门/项目/仓库"等多维辅助核算字段，在页面中以可选显示列呈现。
5. **红冲轨迹可追溯**：红字凭证与蓝字凭证之间双向关联，详情页展示完整冲销链条。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 |
|------|------|----------|--------|
| 会计凭证编辑 | 表单+子表 | 财务员 | ★★★ |
| 会计凭证查询 | 列表+详情 | 财务员 | ★★☆ |
| 凭证模板配置 | 表单 | 财务管理员 | ★★★ |
| 会计期间管理 | 列表+操作 | 财务管理员 | ★★☆ |
| 总账/明细账查询 | 只读报表 | 财务员 | ★★☆ |
| 应收应付核销 | 表单+子表 | 财务员 | ★★★ |
| 期末结账向导 | 分步向导 | 财务管理员 | ★★★ |

## 各页面设计要点

### 会计凭证录入

**页面入口**：财务管理 → 凭证管理 → 新建凭证

**凭证头区**：
```
┌────────────────────────────────────────────────┐
│ 凭证字: [收/付/转]  凭证号: (自动)  日期: [选择] │
│ 附件数: [___]  制单人: (当前用户)               │
│ 摘要: [___] (全局摘要，可被分录行覆盖)           │
└────────────────────────────────────────────────┘
```

**要点**：
- 凭证字下拉：收/付/转，选定后凭证号按字分类自动连续编号 [赤龙#凭证三件套]
- 日期默认当前业务日期，但不能早于最近结账日期

**分录子表（核心交互）**：
```
┌────────────────────────────────────────────────────────┐
│ [+] [×]   [快速模板]   [借贷自动平衡]                    │
│ ┌────┬────────┬──────────┬────────┬────────┬──────────┐│
│ │摘要│ 科目   │ 辅助核算  │借方金额│贷方金额│ 币种/原币││
│ ├────┼────────┼──────────┼────────┼────────┼──────────┤│
│ │ 1  │ 材料采购│ 项目:A   │ 10,000 │        │ RMB      ││
│ │ 2  │ 应交税费│          │ 1,300  │        │ RMB      ││
│ │ 3  │ 应付账款│ 供应商:X │        │ 11,300 │ RMB      ││
│ └────┴────────┴──────────┴────────┴────────┴──────────┘│
│ 合计:                         11,300    11,300  ✅ 平衡  │
└────────────────────────────────────────────────────────┘
```

**要点**：
- **每行只填一侧**：借方金额和贷方金额不同时输入（输入借方则贷方自动清 0，反之亦然） [赤龙#借贷分录行只填一侧]
- 科目选择使用树形弹窗（按科目表层级展开），支持编码/名称搜索
- 辅助核算列可选显示（默认隐藏，通过配置显式开启），显示时占用一行多列扩展
- 合计栏实时计算，不平衡时显示 ❌ 红色警告，过账按钮禁用
- "借贷自动平衡"按钮：将差额自动填入最后一行（借方<贷方时补借方，反之补贷方）
- "快速模板"按钮：弹窗选择凭证模板，选中后自动填充分录行 [赤龙#凭证模板]

### 凭证查询与详情

**列表页**：
- 筛选条件：凭证字、凭证号范围、日期范围、科目、来源业务类型、状态（全部/草稿/已过账/已作废）
- 列表显示：日期 | 凭证字-号 | 摘要 | 借方合计 | 贷方合计 | 状态 | 来源
- 操作：草稿可编辑/过账/作废，已过账可查看/红冲，已作废只查看

**详情页特殊区域**：
```
┌────────────────────────────────────────────┐
│ 业财回链                                    │
│ 来源业务单据: 采购发票 INV-001 [查看▶]      │
│ 业务单状态: 已审核 · 已过账                  │
│ ────────────────────────────────────────── │
│ 红冲关联 (若有)                              │
│ 本凭证被: 红字凭证 ACC-202406-010 冲销 [查看▶]│
│ 原凭证: 蓝字凭证 ACC-202406-005 [查看▶]      │
└────────────────────────────────────────────┘
```

**要点**：
- 来源单据可点击跳转（跨域跳转）
- 红冲链条双向展示（本凭证冲销了谁 / 被谁冲销）
- 凭证打印/导出按钮：输出标准中式凭证格式

### 凭证模板配置

**页面入口**：财务管理 → 凭证模板

**模板列表**：按业务类型（PURCHASE_INPUT / SALES_OUTPUT / AP_INVOICE / AR_INVOICE / PAYMENT / RECEIPT）分组。

**模板编辑**：
```
┌────────────────────────────────────────────────┐
│ 模板名称: 采购入库-暂估应付                      │
│ 业务类型: PURCHASE_INPUT                        │
│ ────────────────────────────────────────────── │
│ 分录行模板:                                     │
│ ┌────┬────────┬──────────┬──────┬──────┬──────┐│
│ │序号│ 方向   │ 科目来源  │金额占位│摘要模板│条件 ││
│ ├────┼────────┼──────────┼──────┼──────┼──────┤│
│ │ 1  │ 借方   │ 存货科目  │AMOUNT │采购入库│     ││
│ │ 2  │ 贷方   │ 暂估应付  │AMOUNT │暂估应付│     ││
│ └────┴────────┴──────────┴──────┴──────┴──────┘│
└────────────────────────────────────────────────┘
```

**要点**：
- 科目来源：可选择固定科目 / 按物料映射 / 按仓库映射 / 按伙伴映射
- 金额占位符：AMOUNT（全额）/ TAX_AMOUNT（税额）/ DISCOUNT（折扣额）
- 条件列：满足特定条件时触发该分录行（如"仅当物料配置了批次管理时"）
- 新增模板后可在测试区输入模拟业务数据预览生成的凭证

### 会计期间管理

**页面入口**：财务管理 → 期间管理

**年度期间表格**：
```
┌────────────────────────────────────────────────────┐
│ 年度: [2026]  [新建期间]                            │
│ ┌───────┬──────────┬──────────┬─────────────────┐ │
│ │ 期间   │ 起始     │ 截止     │ 状态           │ │
│ ├───────┼──────────┼──────────┼─────────────────┤ │
│ │ 1月   │ 01-01    │ 01-31    │ ✅ 已结账        │ │
│ │ 2月   │ 02-01    │ 02-28    │ ✅ 已结账        │ │
│ │ 3月   │ 03-01    │ 03-31    │ ⏳ 结账中        │ │
│ │ 4月   │ 04-01    │ 04-30    │ ● 开启           │ │
│ │ ...   │ ...      │ ...      │ ...             │ │
│ └───────┴──────────┴──────────┴─────────────────┘ │
└────────────────────────────────────────────────────┘
```

**要点**：
- 状态可视化：开启=绿色、结账中=黄色（带进度指示器）、已结账=蓝色（带锁定标记）
- 结账按钮点击触发"期末结账向导"，分步执行：成本核算 → 汇兑损益 → 结转损益 → 凭证审核 → 结账

### 总账/明细账查询

**页面入口**：财务管理 → 总账 / 明细账

```
┌─ 筛选条件 ─────────────────────────────────────────┐
│ 科目: [树形选择]  期间: [选择]  币种: [选择]         │
│ 包含未过账: [☐]  辅助核算: [科目/部门/项目]          │
│ [查询] [打印] [导出]                                 │
├────────────────────────────────────────────────────┤
│ 科目 | 期初余额 | 本期借方 | 本期贷方 | 期末余额     │
│ 1001 库存现金 | 10,000 | 5,000 | 3,000 | 12,000     │
│ 1002 银行存款 | 50,000 | 20,000 | 10,000 | 60,000   │
├────────────────────────────────────────────────────┤
│ 点击行展开明细账: 日期 | 凭证号 | 摘要 | 借方 | 贷方 │
└────────────────────────────────────────────────────┘
```

**要点**：
- 总账按科目汇总，点科目行展开当期明细账
- 明细账行可点击跳转到凭证详情
- 支持"联查凭证"——从总账 → 明细账 → 凭证 三级下钻

## 跨页面导航流

```
业务单据审核
    ↓ (异步过账)
凭证详情 ← [业财回链] ← 业务单据详情
    ↓                  ↑
凭证列表 → [凭证模板] → 触发凭证生成
    ↓
总账 / 明细账 ← [科目汇总]
    ↓
期间管理 → 期末结账向导 → 结账
```

## Line 子实体 form 分组模板

> 适用范围：财务域 6 个 Line 子实体（ErpFinVoucherLine 已由 1500-1 覆盖，不在此节范围）独立 `view.xml` 的 `<form id="view">` / `<form id="edit">` 分组。
> 决策来源：`docs/plans/2026-07-19-1818-2-f3-core-line-and-remaining-main-form-layout.md` Phase 0.D。
> 财务域 Line 模板**各异**（与 inventory 同属分化，但更碎片化），按业务关键字段分组。

### 模板分化决策

| 实体 | 分组结构 |
|------|----------|
| ErpFinVoucherTemplateLine | baseInfo + template + audit（3 组） |
| ErpFinReconciliationLine | baseInfo + amount + reference + audit（4 组） |
| ErpFinBankReconciliationLine | baseInfo + amount + reference + audit（4 组） |
| ErpFinBankStatementLine | baseInfo + amount + match + reference + audit（5 组） |
| ErpFinExpenseClaimLine | baseInfo + amount + reference + audit（4 组） |
| ErpFinBudgetLine | baseInfo + auxiliary + amount + reference + audit（5 组；字段最多） |

### ErpFinVoucherTemplateLine 模板

```xml
<form id="view" size="lg">
    <layout x:override="replace">
=========>baseInfo[基本信息]======
 lineNo[行号] subjectCode[科目编码]
 dcDirection[借贷方向] amountExpression[金额表达式]
=========>template[模板信息]======
 accountKey[科目键] amountKey[金额键]
 memoTemplate[摘要模板] templateId[凭证模板ID]
=========^audit[审计信息]=========
 createdBy[创建人] createTime[创建时间]
 updatedBy[修改人] updateTime[修改时间]
    </layout>
</form>
```

### ErpFinBudgetLine 模板（最大实体）

```xml
<form id="view" size="lg">
    <layout x:override="replace">
=========>baseInfo[基本信息]======
 lineNo[行号] orgId[核算组织]
 acctSchemaId[账套] periodId[期间]
 subjectId[科目ID] subjectCode[科目编码]
=========>auxiliary[辅助核算]======
 costCenterId[成本中心] departmentId[部门]
 projectId[项目] partnerId[往来单位]
 warehouseId[仓库] materialId[物料]
=========>amount[金额信息]======
 budgetAmountSource[预算金额(源币种)] budgetAmountFunctional[预算金额(本位币)]
 currencyId[币种] exchangeRate[汇率]
=========>reference[业务关联]======
 scenarioId[预算方案ID]
=========^audit[审计信息]=========
 remark[备注]
 createdBy[创建人] createTime[创建时间]
 updatedBy[修改人] updateTime[修改时间]
    </layout>
</form>
```

### 各 Line 特化点

| 实体 | 特化 |
|------|------|
| ErpFinReconciliationLine | baseInfo 含 `paymentItemId/invoiceItemId`；amount 含 `settledAmountSource/settledAmountFunctional`；reference 仅 `reconciliationId` |
| ErpFinBankReconciliationLine | baseInfo 含 `adjustmentType/description/dcDirection/side`；amount 仅 `amount`；reference 仅 `reconciliationId` |
| ErpFinBankStatementLine | baseInfo 含 `transactionDate/description/refNo/dcDirection`；amount 含 `amount/currencyId/balanceAfter`；match 含 `matchStatus/matchedLineId`；reference 仅 `statementId` |
| ErpFinExpenseClaimLine | baseInfo 含 `expenseType/projectId/costCenterId/subjectId/subjectCode`；amount 含 `amountWithoutTax/taxRate/taxAmount/amountWithTax`；reference 仅 `claimId` |

### query 表单基线（5+ 字段 + filterOp）

所有 finance Line 实体的 `<form id="query">` 至少含 5 个查询字段。BankStatementLine 等含 `transactionDate` 的实体配 `filterOp=date-between`；其余实体的 `lineNo`、`subjectCode`、`dcDirection` 等无 `like` filterOp 时显式 `filterOp=eq`。ErpFinVoucherLine 已由 1500-1 closure audit 裁决「line entity, query via parent context, 0 filterOp by design」，本节不重做。
