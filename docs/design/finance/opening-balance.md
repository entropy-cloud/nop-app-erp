# 期初余额导入

## 目的

说明 ERP 系统上线或新账套启用时，如何将历史数据作为期初余额导入系统，确保新旧系统数据衔接。

## 导入范围

| 科目类型 | 导入内容 | 说明 |
|----------|----------|------|
| 总账科目 | 各科目期初余额（借方/贷方） | 按会计科目表逐科目导入 |
| 应收明细 | 每笔未结清发票的余额 | 按客户+发票维度导入 |
| 应付明细 | 每笔未结清发票的余额 | 按供应商+发票维度导入 |
| 存货 | 各物料×仓库×批次的数量与成本 | 按存货明细导入 |
| 固定资产 | 每项资产的原值、累计折旧、净值 | 按资产卡片导入 |

## 导入流程

```
期初余额导入流程
        │
        ├─► 步骤1：准备导入模板
        │      ├─ 下载标准导入模板（Excel/CSV）
        │      └─ 按模板格式填写数据
        │
        ├─► 步骤2：数据校验
        │      ├─ 校验科目是否存在
        │      ├─ 校验借贷是否平衡（总账）
        │      ├─ 校验辅助核算维度是否有效
        │      └─ 生成校验报告（错误/警告）
        │
        ├─► 步骤3：导入确认
        │      ├─ 用户确认校验报告
        │      └─ 标记导入状态为 IMPORTED
        │
        ├─► 步骤4：生成期初凭证
        │      ├─ 总账：生成期初结转凭证（摘要："期初余额导入"）
        │      ├─ AR/AP：创建未结清明细记录
        │      ├─ 存货：创建库存余额记录
        │      └─ 固定资产：创建资产卡片+折旧计划
        │
        └─► 步骤5：启用期间开启
               ├─ 期初余额导入完成后开启第一个 OPEN 期间
               └─ 期初凭证不可修改，只能通过反结账冲销
```

## 导入状态

| 状态 | 含义 |
|------|------|
| DRAFT | 导入模板已上传，未校验 |
| VALIDATING | 校验中 |
| VALIDATED | 校验通过，等待确认 |
| IMPORTED | 已导入，期初凭证已生成 |
| REJECTED | 校验失败，需修正后重新导入 |

## 三字段设计

每条导入记录携带三个字段用于追溯：

| 字段 | 含义 |
|------|------|
| `importBatchId` | 导入批次号（同一批次的数据共享） |
| `importSource` | 数据来源标识（如 "legacy-erp"、"manual-entry"） |
| `importStatus` | 导入状态（DRAFT/VALIDATED/IMPORTED/REJECTED） |

## 约束规则

- 期初余额只能在第一个会计期间开启前导入
- 期初凭证的 `posted=true`（直接生效，不走异步过账）
- 已有业务单据的账套不允许重新导入期初余额
- 期初余额导入后不可修改，纠错只能通过常规业务调整

## 与其他域的协作

| 对端域 | 协作内容 |
|--------|----------|
| master-data | 校验科目/物料/客户/供应商是否存在 |
| inventory | 导入存货期初余额（数量+成本） |
| assets | 导入固定资产期初数据（原值+累计折旧） |
| AR/AP | 导入应收应付未结清明细 |
