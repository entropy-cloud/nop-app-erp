# 多账套并行核算机制

## 目的

说明 ERP 系统中多套会计科目表并行核算的架构设计与实现机制。多账套支持满足企业同时维护财务账、管理账、税务账、预算账等多套账务的需求。

## 多账套场景

### 典型应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| 财务账 + 管理账 | 按会计准则记账，同时满足内部管理需求 | 财务账按历史成本，管理账按公允价值 |
| 财务账 + 税务账 | 同时满足财务报告和税务申报需求 | 税务调整项单独核算 |
| 集团合并账 | 子公司独立核算，集团统一合并 | 币种转换、内部交易抵消 |
| 预算账 | 与实际账并行，追踪预算执行 | 预算科目与实际科目映射 |

### 账套类型

| 账套类型 | 说明 | 使用对象 |
|----------|------|----------|
| 财务账（Financial） | 按会计准则记账 | 外部报告、审计 |
| 管理账（Management） | 满足内部管理需求 | 管理层、经营者 |
| 税务账（Tax） | 满足税务申报需求 | 税务部门 |
| 集团账（Consolidation） | 集团内部交易抵消 | 集团总部 |
| 预算账（Budget） | 预算执行追踪 | 预算管理部门 |

## 账套模型

### 科目表结构

```
会计科目表（AccountingSchema）
  ├─ schemaId（账套ID）
  ├─ schemaName（账套名称）
  ├─ schemaType（账套类型：财务/管理/税务/合并/预算）
  ├─ currencyId（本位币）
  ├─ costingMethod（成本核算方法）
  ├─ isActive（是否启用）
  └─ parentSchemaId（上级账套，用于账套继承）

会计科目（Account）
  ├─ accountId（科目ID）
  ├─ schemaId（所属账套）
  ├─ accountCode（科目编码）
  ├─ accountName（科目名称）
  ├─ accountType（科目类型：资产/负债/权益/收入/费用）
  ├─ parentAccountId（上级科目）
  ├─ isDetail（是否明细科目）
  ├─ balanceDirection（余额方向：借/贷）
  └─ isActive（是否启用）
```

### 账套继承关系

```
财务账（基础账套）
    │
    ├─► 继承
    │       │
    │       ├─► 管理账
    │       │      ├─ 新增科目（如内部利润中心）
    │       │      └─ 调整科目余额方向
    │       │
    │       ├─► 税务账
    │       │      ├─ 新增科目（如纳税调整）
    │       │      └─ 调整计量属性
    │       │
    │       └─► 集团账
    │              ├─ 统一科目编码
    │              └─ 币种转换
    │
    └─► 独立（无继承）
            └─► 预算账
```

## 并行核算机制

### 凭证生成时机

业务单据审核通过时，同时在所有启用的账套中生成凭证：

```
业务单据审核通过
        │
        ├─► 查询所有启用的账套
        │      ├─► 财务账
        │      ├─► 管理账
        │      ├─► 税务账
        │      └─► 集团账
        │
        ├─► 逐账套生成凭证
        │      │
        │      ├─► 账套1：凭证模板查找 → 科目映射 → 生成凭证
        │      ├─► 账套2：凭证模板查找 → 科目映射 → 生成凭证
        │      ├─► 账套3：凭证模板查找 → 科目映射 → 生成凭证
        │      └─► 账套N：凭证模板查找 → 科目映射 → 生成凭证
        │
        └─► 更新业务单据 posted=true
```

### 科目映射规则

同一业务在不同账套中可能使用不同科目：

| 业务场景 | 财务账科目 | 管理账科目 | 税务账科目 |
|----------|-----------|-----------|-----------|
| 销售商品 | 主营业务收入 | 内部销售收入 | 营业收入（含税分离） |
| 采购商品 | 原材料 | 内部采购成本 | 可抵扣进项税 |
| 折旧费用 | 管理费用-折旧 | 部门折旧费用 | 税务允许折旧额 |
| 坏账准备 | 资产减值损失 | 呆账准备 | 不可抵扣 |

### 账套级联字段

所有涉及金额的实体需携带账套维度：

```
凭证（Voucher）
  ├─ voucherId
  ├─ schemaId（账套ID）← 关键维度
  ├─ voucherNo
  └─ voucherLines[]

凭证分录行（VoucherLine）
  ├─ lineId
  ├─ voucherId
  ├─ schemaId（账套ID）← 关键维度
  ├─ accountId
  ├─ drAmount
  ├─ crAmount
  └─ dimensions[]

库存流水（StockLedger）
  ├─ ledgerId
  ├─ costingSchemaId（成本账套）← 关键维度
  ├─ unitCost
  └─ totalCost

余额表（Balance）
  ├─ balanceId
  ├─ schemaId（账套ID）← 关键维度
  ├─ accountId
  ├─ periodId
  ├─ drBalance
  ├─ crBalance
  └─ dimensions[]
```

## 账套管理

### 创建账套

```
创建新账套
        │
        ├─► 选择账套类型
        │      ├─ 独立账套（无继承）
        │      └─ 继承账套（从现有账套继承）
        │
        ├─► 配置账套参数
        │      ├─ 账套名称
        │      ├─ 本位币
        │      ├─ 成本核算方法
        │      └─ 科目继承策略
        │
        ├─► 复制或新建科目表
        │      ├─ 从模板复制
        │      ├─ 从已有账套复制
        │      └─ 从零新建
        │
        ├─► 配置账套级凭证模板
        │      └─ 为每类业务配置该账套的凭证模板
        │
        └─► 启用账套
               └─ 设置 isActive=true
```

### 账套参数差异

不同账套可能有不同的参数设置：

| 参数 | 财务账 | 管理账 | 税务账 |
|------|--------|--------|--------|
| 成本核算方法 | 移动加权平均 | 实际成本 | 移动加权平均 |
| 本位币 | 人民币 | 人民币 | 人民币 |
| 汇率处理 | 即期汇率 | 即期汇率 | 历史汇率 |
| 计量属性 | 历史成本 | 公允价值可选 | 计税基础 |
| 期间规则 | 自然月 | 自然月 | 税务期间 |

## 账套查询与报表

### 多账套查询

支持按账套维度查询数据：

```
凭证查询
  ├─ 按账套过滤
  │      ├─ 财务账
  │      ├─ 管理账
  │      └─ 全部账套
  │
  ├─ 按币种过滤
  │      ├─ 本位币
  │      └─ 源币种
  │
  └─ 按时间过滤
         ├─ 业务日期
         └─ 凭证日期
```

### 账套对账

定期核对不同账套间的数据一致性：

```
账套对账任务
        │
        ├─► 科目余额对账
        │      └─ 各账套余额一致性检查
        │
        ├─► 凭证数量对账
        │      └─ 各账套凭证数量一致性检查
        │
        ├─► 业务单据关联对账
        │      └─ 同一业务在各账套中的凭证一致性
        │
        └─► 异常报告
               └─ 列出不一致的账套和科目
```

### 账套转换

支持账套间的数据转换：

| 转换类型 | 说明 | 场景 |
|----------|------|------|
| 期间结转 | 将一账套的期末余额转入另一账套的期初 | 年终账务调整 |
| 余额调整 | 调整不同账套间的余额差异 | 税务调整项处理 |
| 报表合并 | 合并多账套数据生成综合报表 | 集团报表 |

## 账套关闭与归档

### 账套关闭

```
账套关闭
        │
        ├─► 前置检查
        │      ├─ 期间结账完成
        │      ├─ 报表已生成
        │      └─ 无未过账业务
        │
        ├─► 凭证归档
        │      ├─ 导出凭证数据
        │      └─ 标记为归档状态
        │
        ├─► 账套状态更新
        │      └─ isActive = false
        │
        └─► 历史数据保留
               └─ 归档数据只读保留
```

### 账套归档

长期不使用的账套可归档：

- 归档后账套数据只读
- 归档数据可导出为历史备份
- 支持按需恢复归档账套

## 权限控制

### 账套级权限

| 权限 | 说明 |
|------|------|
| 账套查看 | 查看特定账套的凭证和报表 |
| 账套维护 | 创建、修改账套参数 |
| 跨账套查询 | 同时查看多账套数据 |
| 账套对账 | 执行账套间对账 |

### 数据隔离

- 用户只能访问被授权的账套
- 未授权账套对用户不可见
- 敏感账套（如税务账）需额外授权

## 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `erp-fin.default-schema` | 财务账 | 默认生成的账套 |
| `erp-fin.multi-schema-enabled` | false | 是否启用多账套 |
| `erp-fin.schema-inheritance` | true | 是否支持账套继承 |
| `erp-fin.auto-create-all-schemas` | true | 业务单据是否自动在所有账套生成凭证 |

## 性能优化

### 批量处理

多账套凭证生成采用批量处理：

```
批量生成多账套凭证
        │
        ├─► 收集待处理业务单据
        │      └─ 按账套分组
        │
        ├─► 批量生成凭证
        │      ├─ 按账套批量插入凭证
        │      └─ 批量更新 posted 状态
        │
        └─► 并行处理优化
               └─ 多账套并行生成（可选）
```

### 缓存策略

- 账套配置信息缓存
- 科目映射规则缓存
- 凭证模板缓存

## 异常处理

| 异常场景 | 处理 |
|----------|------|
| 部分账套生成失败 | 记录失败账套，继续其他账套；失败账套标记待重试 |
| 科目映射缺失 | 在失败账套中标记异常，等待配置 |
| 借贷不平衡 | 在失败账套中标记异常，人工介入 |
| 账套间数据不一致 | 生成对账差异报告，人工核对 |
