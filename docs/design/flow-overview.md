# 流程总览

## 目的

展示 nop-app-erp 的全局业务流程、状态映射、业财打通机制与跨域协作全景。

---

## 一、四层流程架构

### L1：业务域层（Domain Layer）

各业务域独立维护自己的业务单据与状态机：

```
采购域          销售域          库存域          财务域
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ 采购订单 │     │ 销售订单 │     │ 移动单   │     │ 凭证     │
│ 采购入库 │     │ 销售出库 │     │ 库存流水 │     │ 会计期间 │
│ 采购发票 │     │ 销售发票 │     │ 库存余额 │     │ 成本核算 │
│ 付款单   │     │ 收款单   │     │          │     │          │
└────┬────┘     └────┬────┘     └────┬────┘     └────┬────┘
     │               │               │               │
     └───────────────┼───────────────┴───────────────┘
                     │
                     ▼
### L2：跨域协作层（Cross-Domain Layer）

业务域之间通过接口调用与事件进行协作：

```
采购入库审核 → IErpInvStockMoveBiz → 生成入库移动单 → 库存流水+余额更新
销售出库审核 → IErpInvStockMoveBiz → 生成出库移动单 → 可用量校验 + 库存扣减
```

**同步调用**（强一致性）：
- 采购入库 → 库存入库
- 销售出库 → 库存扣减（含可用量校验）
- 生产领料 → 库存出库

**事件驱动**（最终一致性）：
- 业务单据审核 → 发布过账事件 → 财务域生成凭证

### L3：业财打通层（Finance Integration Layer）

所有业务单据通过统一机制生成会计凭证：

```
业务单据审核通过
        │
        ▼
    设置 posted=false
        │
        ▼
    发布过账事件 (post-commit)
        │
        ▼
IErpFinAcctDocProvider 查找 Provider
        │
        ▼
IErpFinAcctDocProvider.createFacts()
        │
        ▼
   生成凭证分录行
        │
        ▼
   校验借贷平衡
        │
        ▼
写入凭证 + 业财回链 + 设置 posted=true
```

**业务类型与凭证映射**：

| 业务单据 | businessType | 凭证方向 |
|----------|--------------|----------|
| 采购入库 | PURCHASE_INPUT | 借：存货 / 贷：暂估应付 |
| 销售出库 | SALES_OUTPUT | 借：结转成本 / 贷：存货 |
| 采购发票 | AP_INVOICE | 借：费用/采购 / 借：进项税 / 贷：应付 |
| 销售发票 | AR_INVOICE | 借：应收 / 贷：收入 / 贷：销项税 |
| 付款 | PAYMENT | 借：应付 / 贷：银行存款 |
| 收款 | RECEIPT | 借：银行存款 / 贷：应收 |
| 折旧 | DEPRECIATION | 借：折旧费用 / 贷：累计折旧 |
| 资产资本化 | CAPITALIZATION | 借：固定资产 / 贷：在建工程或存货 |
| 资产处置 | DISPOSAL | 借：累计折旧 / 借：清理损益 / 贷：固定资产 |

### L4：期末结算层（Period Closing Layer）

月末/季末/年末的财务结算流程：

```
期末结账触发
        │
        ├─► 检查本期单据是否全部过账 (posted=true)
        │
        ├─► 执行成本核算（移动加权平均/FIFO/批次）
        │
        ├─► 计提折旧（所有固定资产）
        │
        ├─► 摊销待摊费用
        │
        ├─► 结转损益（收入→本年利润，费用→本年利润）
        │
        ├─► 生成期末结转凭证
        │
        └─► 标记期间为已结账
```

## 二、核心业务流程

### 2.1 采购到付款流程

```
采购申请 → 采购订单 → 采购入库 → 采购发票 → 付款 → 闭环
    │           │           │           │         │
    │           │           │           │         ▼
    │           │           │           │    核销应付
    │           │           │           ▼
    │           │           │    生成应付凭证
    │           │           ▼
    │           │    库存入库（同步）
    │           ▼
    │    三单匹配校验（订单-入库-发票）
    ▼
审批流程
```

**关键控制点**：
- 采购订单审核后锁定价格，不允许修改
- 入库数量超容差需审批
- 发票三单匹配失败拒绝审核
- 付款核销检查发票状态

### 2.2 销售到收款流程

```
销售报价 → 销售订单 → 销售出库 → 销售发票 → 收款 → 闭环
    │           │           │           │         │
    │           │           │           │         ▼
    │           │           │           │    核销应收
    │           │           │           ▼
    │           │           │    生成应收凭证
    │           │           ▼
    │           │    可用量校验 + 库存扣减（同步）
    │           ▼
    │    订单信用额度检查
    ▼
审批流程
```

**关键控制点**：
- 销售订单审核时检查客户信用额度
- 出库前校验库存可用量（不足拒绝出库）
- 发票金额超过订单金额需审批
- 收款核销按订单/发票维度

### 2.3 生产流程

```
销售订单/预测 → MRP运算 → 工单 → 领料 → 报工 → 完工入库 → 成本结转
                     │       │      │      │         │           │
                     │       │      │      │         │           ▼
                     │       │      │      │         │    生成成本凭证
                     │       │      │      │         ▼
                     │       │      │      │    库存入库（产成品）
                     │       │      │      ▼
                     │       │      │    记录工时（直接人工）
                     │       │      ▼
                     │       │    库存出库（原材料）
                     │       ▼
                     │    齐套校验（子件可用量）
                     ▼
                 BOM展开（多级）
```

**关键控制点**：
- 工单审核时执行齐套校验
- 领料按BOM标准消耗控制（支持超耗配置）
- 完工入库时计算产成品成本
- 工艺路线工时记录用于成本归集

### 2.4 库存管理流程

```
入库流程                出库流程                盘点流程
    │                      │                      │
    ▼                      ▼                      ▼
移动单(DRAFT)        可用量校验            盘点单创建
    │                      │                      │
    ▼                      ▼                      ▼
移动单(CONFIRMED)    移动单创建            盘点执行
    │                      │                      │
    ▼                      ▼                      ▼
写库存流水           移动单确认            差异计算
    │                      │                      │
    ▼                      ▼                      ▼
更新库存余额         写库存流水           生成盘盈/盘亏移动单
    │                      │                      │
    ▼                      ▼                      ▼
发布过账事件         更新库存余额         走正常移动单流程
                      │
                      ▼
                  发布过账事件
```

**关键控制点**：
- 库存流水不可变（冲销走反向流水）
- 余额由流水驱动更新（非独立维护）
- 预留量机制保证出库有序
- 批次/序列号校验确保追溯

### 2.5 委外加工流程

```
销售订单/预测 → MRP运算 → 委外订单 → 发料给供应商 → 收货入库 → 委外发票 → 付款
                     │           │              │           │           │
                     │           │              │           │           ▼
                     │           │              │           │    核销应付
                     │           │              │           ▼
                     │           │              │    生成委外应付凭证
                     │           │              ▼
                     │           │    库存入库（产成品，含加工费）
                     │           ▼
                     │    库存出库（原材料，供应商领料）
                     ▼
                BOM 展开（仅外协工序）
```

**关键控制点**：
- 委外订单关联 BOM 的外协工序（区别于自制工单）
- 发料给供应商走"供应商库存"或"在途库"（不立即核销）
- 收货入库时材料成本 + 加工费 = 委外产成品成本
- 加工费发票触发应付凭证；损耗按合同约定（超损拒付）

### 2.6 生产异常处理流程

```
工单执行中
    │
    ├─► 质量不合格（NCR 触发）
    │       ├─► 返工：创建返工工单（关联原工单）→ 返工领料 → 返工报工 → 返工入库
    │       ├─► 报废：创建报废单（数量+原因）→ 报废出库 → 损益凭证
    │       └─► 降级：降级入库（降级产品 BOM）→ 走正常完工入库
    │
    └─► 退料（多领/错领）
            ├─► 未开工：直接释放预留，删除领料单
            └─► 已领料：创建退料单（关联原领料单）→ 退料入库 → 反向库存移动
```

**关键控制点**：
- 返工工单关联原工单（双向回链），返工成本单独归集
- 报废走专用报废库位，触发"营业外支出"或"制造费用"科目
- 退料是反向移动单（冲销原领料的库存流水，非删除）

### 2.7 两步调拨流程（在途场景）

```
调拨单创建（TransferOrder）
    │
    ├─► 一步调拨（同一物理仓库内库位间）
    │       └─► 单个移动单同时扣来源库位加目的库位
    │
    └─► 两步调拨（跨仓库，需在途）
            ├─► 步骤1：发出库 → 移动单扣来源仓库 + 加"在途库"
            ├─► 步骤2：接收库 → 移动单扣"在途库" + 加目的仓库
            └─► 在途库存可查可对账（TransferOrder 头级状态：SHIPPED/IN_TRANSIT/RECEIVED）
```

**关键控制点**：
- 两步调拨的"在途库"是逻辑库位（warehouseType=IN_TRANSIT）
- 在途期间库存所有权属发出方（不转移法人所有权，除非跨法人调拨）
- 跨法人调拨触发内部交易凭证（视同买卖）

## 三、状态映射总览

### 3.1 审批状态映射（approveStatus）

> 本表只描述审批轴（approveStatus）。`CANCELLED` 属于 docStatus 独立轴，见 §3.2。`SUBMITTED` 为"已提交待审批"态（提交到工作流后、审核人处理前），`domain-design-guidelines.md §16.3` 有统一定义。

| 单据类型 | 初始状态 | 中间状态 | 终态 |
|----------|----------|----------|------|
| 采购订单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 采购入库单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 采购发票 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 付款单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 销售订单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 销售出库单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 销售发票 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |
| 收款单 | UNSUBMITTED | SUBMITTED | APPROVED / REJECTED |

**收付款状态**（派生）：采购侧 UNPAID → PARTIAL → PAID；销售侧 UNRECEIVED → PARTIAL → RECEIVED。

### 3.2 业务生命周期映射（docStatus）

| 单据类型 | 初始状态 | 中间状态 | 终态 |
|----------|----------|----------|------|
| 采购/销售 订单/出入库/发票/收付款 | DRAFT | — | CANCELLED |
| 库存移动单 | DRAFT | CONFIRMED | DONE / CANCELLED |
| 库存盘点单 | DRAFT | COUNTING | DONE / CANCELLED |
| 会计凭证 | DRAFT | — | POSTED / CANCELLED |
| 资产卡片 | DRAFT | IN_SERVICE, IDLE | SCRAPPED / SOLD |
| 折旧计划条目 | PENDING | — | EXECUTED / REVERSED |

### 3.3 制造域状态映射（docStatus）

| 单据类型 | 初始状态 | 中间状态 | 终态 |
|----------|----------|----------|------|
| 工单 | DRAFT | SUBMITTED → NOT_STARTED →（STOCK_PARTIAL/STOCK_RESERVED）→ IN_PROCESS（±STOPPED） | COMPLETED / CLOSED / CANCELLED |

> 工单状态机共 10 态，权威定义见 `docs/design/manufacturing/state-machine.md`（无 INSPECTING 态——质检判定经 config-gated 钩子在完工入库前阻塞/放行，不引入独立工单状态）。

## 四、业财打通详细流程

### 4.1 凭证生成机制

```
凭证生成流程
        │
        ▼
业务单据审核通过
        │
        ├─► 设置 posted=false
        │
        ├─► 发布 PostingEvent(businessType, billData)
        │
        └─► 返回审核成功（不等待凭证生成）
                │
                ▼ (post-commit 异步)
        IErpFinAcctDocProvider.getProvider(businessType)
                │
                ▼
        provider.createFacts(billData, acctSchema)
                │
                ├─► 读取凭证模板
                │
                ├─► 解析科目映射（多维决策）
                │
                ├─► 填充金额占位符
                │
                └─► 返回 VoucherLine 列表
                        │
                        ▼
                校验借贷平衡
                        │
                        ▼
                创建 Voucher + VoucherLine
                        │
                        ▼
                创建 VoucherBillR（业财回链）
                        │
                        ▼
                设置业务单据 posted=true
                        │
                        ▼
                失败：记录错误 + 等待兜底扫描
```

### 4.2 科目映射解析

科目映射按"specific → generic"优先级匹配：

```
业务单据 → 业务类型(businessType) → 物料类别 → 物料 → 默认科目
            │                       │          │
            ▼                       ▼          ▼
        AP_INVOICE            原材料      钢材A
            │                       │          │
            ▼                       ▼          ▼
        应付科目族            原材料科目  钢材科目（如果存在）
                                    │
                                    ▼
                            最终匹配：原材料科目
```

**映射维度**：
- 业务类型：AP_INVOICE、AR_INVOICE、PAYMENT、RECEIPT 等
- 物料类别：原材料、产成品、半成品、服务等
- 往来单位组：国内供应商、国外供应商、客户等
- 仓库：普通仓、在途仓、报废仓等
- 部门/项目：辅助核算维度

### 4.3 多币种处理

```
业务单据（多币种）
        │
        ├─► 源币种金额 (amountSource)
        │
        ├─► 业务日期汇率 (exchangeRate)
        │
        ├─► 本位币金额 = amountSource × exchangeRate
        │
        └─► 凭证分录行同时记录双币种
                │
                ▼
        凭证查询支持按源币种或本位币筛选
```

### 4.4 多套科目表并行

```
同一业务单据
        │
        ├─► 管理账科目表 → 生成管理账凭证
        │
        ├─► 税务账科目表 → 生成税务账凭证
        │
        └─► 集团合并账科目表 → 生成合并凭证
                │
                ▼
        各科目表独立核算，期末按需合并报表
```

## 五、异常处理与恢复

### 5.1 过账失败处理

| 失败类型 | 处理策略 | 重试机制 |
|----------|----------|----------|
| 模板缺失 | 标记异常，等待人工配置 | 兜底扫描每 1 分钟 |
| 科目映射缺失 | 标记异常，等待人工配置 | 兜底扫描每 1 分钟 |
| 借贷不平衡 | 记录错误，人工介入 | 不自动重试（配置错误） |
| 期间已结账 | 记录错误，提示反结账 | 不自动重试 |
| 系统异常 | 自动重试（指数退避） | 最多重试 3 次 |

### 5.2 冲销流程

```
业务单据作废
        │
        ├─► 按业财回链反查凭证
        │
        ├─► 生成红字凭证（金额取负）
        │
        ├─► 红字凭证走正常过账流程
        │
        └─► 更新业务单据状态为已作废
                │
                ▼
        红字凭证与原凭证双向关联，保留审计轨迹
```

### 5.3 撤销提交与反审核流程

#### 撤销提交（SUBMITTED → UNSUBMITTED）

提交人在审核人尚未处理时可撤回修改：

```
已提交单据（approveStatus=SUBMITTED）撤销请求
        │
        ├─► 校验前置：审核人是否已开始处理（nop-wf 已激活？）
        │
        ├─► nop-wf 未激活：允许撤回
        │       └─► approveStatus SUBMITTED → UNSUBMITTED
        │
        └─► nop-wf 已激活：拒绝撤回，提示联系审核人驳回
```

> **约束**：仅提交人可操作；审核人一旦开始审核，提交人不可再撤回。

#### 反审核（APPROVED → REJECTED）

已审核单据反审核请求：
        │
        ├─► 校验前置：是否已生成下游单据（如已开发票的订单）
        │
        ├─► 校验：是否已过账（posted=true）
        │
        ├─► 已过账：先生成红字冲销凭证（按业财回链反查）→ posted 反向置 false
        │
        ├─► 已生成下游单据：拒绝反审核，提示先冲销下游
        │
        └─► 通过校验：业务状态 APPROVED → REJECTED（非回 UNSUBMITTED）
                │                ▼
        REJECTED 状态允许修改后重新 SUBMITTED（重新审核），或显式作废到 CANCELLED
```

> **状态语义说明**：反审核的目标态是 `REJECTED`（可重新提交）而非 `UNSUBMITTED`（初始态）。`UNSUBMITTED` 是新建单据的初始态，反审核的单据已发生过业务，语义上不应回退为"未提交"。各域状态机见对应 `state-machine.md`。

## 六、数据一致性保障

### 6.1 事务边界

| 操作 | 事务范围 | 一致性级别 |
|------|----------|------------|
| 单据审核 + 库存变更 | 跨域事务（REQUIRED） | 强一致性 |
| 单据审核 + 凭证生成 | 异步事件 + 最终一致 | 最终一致性 |
| 期末结账 | 单库事务（REQUIRED） | 强一致性 |

### 6.2 兜底机制

```
定时任务（每分钟执行）
        │
        ├─► 扫描 posted=false 且已审核超过 5 分钟的单据
        │
        ├─► 重新触发过账
        │
        └─► 记录重试日志
                │
                ▼
        连续失败超过 3 次 → 发送告警 → 人工介入
```

### 6.3 对账机制

**每日对账任务**：
- 库存余额 vs 库存流水汇总
- 应收余额 vs 发票汇总 - 收款汇总
- 应付余额 vs 发票汇总 - 付款汇总
- 总账余额 vs 凭证明细汇总

## 八、流程设计特点总结

1. **松耦合**：通过事件驱动解耦业务域与财务域
2. **最终一致性**：非关键路径采用异步事件，不阻塞主流程
3. **幂等性**：所有操作支持重复调用
4. **可追溯**：业财回链保证凭证与业务单据双向追溯
5. **容错性**：兜底扫描 + 重试机制确保不丢失数据
6. **审计性**：红字冲销保留原记录，不删除数据
7. **扩展性**：凭证生成 Provider 支持插件化扩展
8. **配置化**：科目映射、容差规则等支持配置
