# 采购退货详细流程

## 目的

说明采购退货的完整业务流程、状态机、与库存/财务域的协作、以及退货与红字发票的处理机制。

## 退货场景

### 退货触发场景

| 场景 | 说明 | 处理方式 |
|------|------|----------|
| 质量不合格 | 来料检验发现物料质量问题 | 退货并可能触发 NCR |
| 数量不符 | 实收数量超过订单数量或规格不符 | 退货或部分退货 |
| 规格不符 | 物料规格/型号与订单不符 | 退货或换货 |
| 重复采购 | 误采购或采购过量 | 退货 |
| 延期交货 | 供应商延期导致业务取消 | 退货（全额或部分） |
| 发票错误 | 供应商发票与实际不符 | 退货+重新开票 |

### 退货类型

| 类型 | 说明 | 影响 |
|------|------|------|
| 部分退货 | 退货数量小于原入库数量 | 仅冲销部分库存和应付 |
| 全额退货 | 退货数量等于原入库数量 | 冲销全部库存和应付 |
| 换货 | 退货同时重新采购 | 退货+新采购订单 |

## 退货流程

### 退货流程总览

```
采购退货流程
        │
        ├─► 步骤1：创建退货单
        │      ├─ 手工创建退货单
        │      ├─ 或从来料检验不合格触发
        │      └─ 关联原采购入库单
        │
        ├─► 步骤2：填写退货信息
        │      ├─ 选择退货物料与数量
        │      ├─ 填写退货原因
        │      ├─ 选择退货仓库/库位
        │      └─ 关联原入库单行
        │
        ├─► 步骤3：提交审核
        │      └─ 状态：UNSUBMITTED → SUBMITTED
        │
        ├─步骤4：审核通过
        │      └─ 状态：SUBMITTED → APPROVED
        │
        ├─► 步骤5：库存出库
        │      ├─ 调用 IErpInvStockMoveBiz
        │      ├─ 生成出库移动单
        │      └─ 库存减少
        │
        ├─► 步骤6：应付红字凭证
        │      ├─ 采购退货触发红字发票（若已开票）
        │      ├─ 或冲减暂估应付（若未开票）
        │      └─ 生成红字凭证
        │
        ├─► 步骤7：更新关联单据
        │      ├─ 更新原入库单未退货量
        │      ├─ 更新原采购订单未交货量
        │      └─ 更新应付余额
        │
        └─► 步骤8：完成退货
               └─ 状态：APPROVED → DONE
```

## 退货单状态机

> **实现偏离记录（plan 0456-1 Decision）**：本节原提的三轴含 `returnStatus`（初始/部分退货/全额退货）。实现期**不新增** `returnStatus` ORM 字段——其语义是**源入库行的累计退货进度**（派生视图），非退货单固有状态。复用现有 `docStatus`+`approveStatus` 两轴表达退货单生命周期（终态 = ACTIVE+APPROVED+`posted=true`），「部分/全额退货」以 `ReturnQtyValidator` 的聚合查询表达（Σ 已审核退货行 / 入库行量）。`ErpPurReturn`/`ErpPurReceiveLine` 均无 `returnStatus`/`returnedQuantity` 列。触发加冗余列的条件见 plan 0456-1 Deferred。

### 三轴状态分离

采购退货单同样采用三轴状态分离：

| 状态轴 | 字段 | 说明 |
|--------|------|------|
| 单据状态 | docStatus | 草稿/生效/作废 |
| 审核状态 | approveStatus | 未提交/已提交/已审核/已驳回 |
| 退货状态 | returnStatus | 初始/部分退货/全额退货 |

### 状态迁移

```
UNSUBMITTED
  ├─ 提交 → SUBMITTED
  │            ├─ 审核通过 → APPROVED
  │            │              ├─ [触发库存出库]
  │            │              ├─ [生成红字凭证]
  │            │              └─ [更新关联单据]
  │            └─ 驳回 → REJECTED
  │                          └─ 修改后重新提交
  └─ 作废 → CANCELLED
```

## 与库存域协作

### 退货出库

退货审核通过后调用库存域生成出库移动单：

```
退货单审核通过
        │
        ├─► 调用 IErpInvStockMoveBiz.generateReturnMove()
        │      ├─ 传入参数：
        │      │   ├─ sourceBillType = PURCHASE_RETURN
        │      │   ├─ sourceBillId = 退货单ID
        │      │   ├─ materialId = 物料ID
        │      │   ├─ warehouseId = 退货仓库
        │      │   ├─ locationId = 退货库位
        │      │   ├─ quantity = 退货数量
        │      │   ├─ sourceReceiveLineId = 原入库单行ID（追溯用）
        │      │   └─ batchId = 原批次ID（若启用批次管理）
        │      │
        │      └─ 返回：出库移动单ID
        │
        ├─► 生成出库移动单
        │      ├─ 方向：OUTGOING（出库）
        │      ├─ 源单据类型：采购退货
        │      └─ 关联原入库单追溯链
        │
        └─► 更新库存
               ├─ 库存流水写入（数量取负）
               ├─ 库存余额扣减
               └─ 批次/序列号更新（如适用）
```

### 批次追溯

启用批次管理时，退货使用原入库的批次：

```
批次追溯
        │
        ├─► 读取原入库单的批次信息
        │      └─ sourceReceiveLineId → 入库批次
        │
        ├─► 退货出库时使用相同批次
        │      └─ 优先从原批次出库
        │
        └─► 若原批次已全部出库
               ├─ 提示批次不足
               └─ 允许选择其他批次
```

## 与财务域协作

### 红字发票处理

退货与发票的交互有以下几种场景：

| 场景 | 说明 | 处理 |
|------|------|------|
| 未开票退货 | 退货时发票尚未开具 | 仅冲减暂估应付 |
| 已开票退货 | 退货时发票已开具 | 生成红字发票冲销 |
| 部分退货+发票 | 退货数量小于发票数量 | 部分红字发票 |

### 暂估应付冲减

入库后未开票时，应付为暂估。退货冲减暂估应付：

```
暂估应付冲减流程
        │
        ├─► 查询原入库单的暂估应付
        │      └─ 入库单关联的应付凭证
        │
        ├─► 生成红字冲销凭证
        │      ├─ 凭证类型：红字（RED_FLUSH）
        │      ├─ 借：暂估应付（原贷方方向取反）
        │      ├─ 贷：存货（原借方方向取反）
        │      └─ 关联原入库凭证
        │
        ├─► 更新入库单退货后应付余额
        │      └─ 未退货数量对应的暂估应付
        │
        └─► 业财回链
               ├─ 新凭证关联原入库单
               └─ 标记为冲销凭证
```

### 红字发票处理

已开票情况下，退货生成红字发票：

```
红字发票流程
        │
        ├─► 创建红字发票单
        │      ├─ 关联原蓝字发票
        │      ├─ 金额取负
        │      ├─ 关联退货单
        │      └─ 原因代码（退货/折让/折扣）
        │
        ├─► 审核红字发票
        │      ├─ 状态：SUBMITTED → APPROVED
        │      └─ 触发凭证生成
        │
        ├─► 生成红字凭证
        │      ├─ 借：应付（原贷方转借方）
        │      ├─ 贷：进项税转出（如适用）
        │      └─ 凭证关联红字发票
        │
        ├─► 更新蓝字发票状态
        │      ├─ 蓝字发票冲销标志 = true
        │      └─ 关联红字发票
        │
        └─► 更新供应商应付余额
               └─ 余额 = 原余额 - 红字金额
```

### 凭证分录示例

#### 场景1：未开票退货（冲减暂估应付）

```
原入库凭证（暂估应付）：
  借：存货 10,000
  贷：暂估应付 10,000

退货红字凭证：
  借：暂估应付 3,000 （红字）
  贷：存货 3,000 （红字）

结果：暂估应付减少 3,000，存货减少 3,000
```

#### 场景2：已开票退货（红字发票）

```
原发票凭证（确认应付）：
  借：采购成本 8,547
  借：进项税 1,103
  贷：应付账款 9,650

退货红字凭证：
  借：应付账款 3,000 （红字）
  贷：采购成本 2,564 （红字）
  贷：进项税 436 （红字）

结果：应付账款减少 3,000，采购成本和进项税相应冲减
```

## 退货与关联单据

### 关联追溯

退货单需关联以下单据：

| 关联类型 | 关联单据 | 作用 |
|----------|----------|------|
| 源单关联 | 原采购入库单 | 追溯入库信息、批次 |
| 订单关联 | 原采购订单 | 更新未交货量 |
| 发票关联 | 原采购发票（如有） | 确定是否需要红字发票 |
| 凭证关联 | 原入库凭证/发票凭证 | 业财回链 |

### 未交货量更新

退货后更新原采购订单的未交货量：

```
采购订单未交货量更新
        │
        ├─► 原订单未交货量 = 原订单数量 - 原入库数量 + 退货数量
        │
        ├─► 示例：
        │      ├─ 订单数量：100
        │      ├─ 已入库数量：60
        │      ├─ 退货数量：10
        │      └─ 更新后未交货量 = 100 - 60 + 10 = 50
        │
        └─► 退货数量可重新形成新的采购需求
```

## 异常处理

### 退货数量限制

| 限制类型 | 说明 | 处理 |
|----------|------|------|
| 超额退货 | 退货数量 > 可退数量 | 拒绝，提示最大可退数量 |
| 已核销退货 | 发票已全额核销付款 | 需先撤回核销 |
| 已结账期间退货 | 退货期间已结账 | 拒绝，需先反结账 |

### 批次处理异常

| 场景 | 处理 |
|------|------|
| 原批次已用完 | 允许选择其他批次或提示批次不足 |
| 批次效期已过 | 提示批次效期问题，需审批 |
| 序列号追溯失败 | 提示序列号未找到，拒绝退货 |

### 并发控制

- 同一入库单行并发退货：使用乐观锁防止超额退货
- 退货与付款并发：校验付款状态后再处理退货

## 业务规则

### 退货约束

1. **退货数量限制**：退货数量 ≤ 原入库未退货数量
2. **金额限制**：退货金额不能超过原入库金额
3. **期间限制**：退货应在合理的业务周期内完成
4. **状态限制**：原入库单必须已审核

### 仓库/库位规则

- 退货仓库默认为原入库仓库
- 允许选择其他退货仓库
- 退货库位按仓库配置默认
- 批次/序列号必须与原入库一致

### 审批规则

| 条件 | 审批要求 |
|------|----------|
| 普通退货 | 采购员提交 → 审核人审核 |
| 高额退货（>阈值） | 采购员提交 → 部门主管审核 |
| 频繁退货供应商 | 需供应商确认 |

## 与销售退货的区别

| 对比项 | 采购退货 | 销售退货 |
|--------|----------|----------|
| 库存方向 | 出库（库存减少） | 入库（库存增加） |
| 凭证方向 | 冲减应付 | 冲减应收 |
| 关联入库 | 关联采购入库 | 关联销售出库 |
| 批次追溯 | 追溯原采购批次 | 追溯原销售批次 |

## 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `erp-pur.return-qty-limit` | 入库未退货量 | 退货数量上限 |
| `erp-pur.return-period-days` | 365 | 退货有效期（天） |
| `erp-pur.auto-create-invoice` | false | 退货是否自动生成红字发票 |
| `erp-pur.return-approval-required` | true | 退货是否需要审核 |
| `erp-pur.return-reason-required` | true | 退货是否必须填写原因 |
