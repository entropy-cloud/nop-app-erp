# 采购域（purchase）

## 目的

说明采购域的业务语义、单据生命周期、状态机与跨域协作。采购域管理从供应商采购物料的完整业务闭环，是"应付（AP）侧"业务单据的来源。

## 边界

- 本域负责：采购订单、采购入库、采购发票、付款、采购退货。
- 本域不负责：物料/供应商主数据（master-data 域）；入库写库存（inventory 域，本域调用）；应付凭证生成（finance 域，本域单据触发）。
- 持久化字段、字典、状态码以 `model/app-erp-purchase.orm.xml` 为准。
- 跨域协作规则见 `../domain-design-guidelines.md`，全局流程见 `../flow-overview.md`。

## 核心业务对象

| 对象 | 业务含义 | 头/行 |
|------|----------|-------|
| 采购订单（PurchaseOrder） | 与供应商签订的采购意向 | 头/行 |
| 采购入库单（PurchaseReceive） | 实际收货单据，触发库存入库 | 头/行 |
| 采购发票（PurchaseInvoice） | 供应商发票，应付凭证来源 | 头/行 |
| 付款单（Payment） | 向供应商付款 | 头/核销明细行 |
| 采购退货单（PurchaseReturn） | 向供应商退货，触发库存出库 | 头/行 |

### 单据关系

```
采购订单 → [部分/全部] → 采购入库 → 采购发票 → 付款 → 闭环
                                        ↑
                                   三单匹配（订单-入库-发票）
```

- 采购订单可多次入库（部分收货）。
- 入库后产生应付（暂估或实际），发票到达后确认应付。
- 发票与付款分离：可先收货后开票，也可先开票后付款。
- 采购退货可由入库单触发或独立创建。

## 状态机

采购单据采用三轴状态分离（业务/审核/付款），按 `docs/skills/state-machine-business-review-prompt.md` 的 10 个审查维度设计。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 采购入库写库存 | inventory | 入库单审核 → `IErpInvStockMoveBiz` 生成入库移动单 |
| 采购退货出库 | inventory | 退货单审核 → 生成出库移动单 |
| 采购发票触发应付凭证 | finance | 发票审核 → 财务域生成应付凭证（AP_INVOICE） |
| 付款触发付款凭证 | finance | 付款单审核 → 财务域生成付款凭证（PAYMENT） |
| 物料/供应商/仓库引用 | master-data | 单据引用主数据编码 + `IErpMd*Biz` |

## 关键业务规则

1. **三单匹配**：采购订单 → 入库 → 发票，发票行可回链入库行，确保数量与金额一致。详细规则见 [`three-way-match.md`](three-way-match.md)。
2. **部分收货**：一张采购订单可多次入库，订单未交量 = 订单数量 − 累计入库数量。
3. **付款核销**：付款与发票多对多，一张付款可核销多张发票，一张发票可被多次部分付款。头级付款进度状态聚合行级核销结果。
4. **价税分离**：行级金额、税额、税率独立列，便于增值税核算。
5. **多币种**：单据引用币种，金额按业务日期汇率转换本位币。
6. **取消影响**：已审核单据作废需冲销（生成反向单据或冲销凭证），不可直接删除。
