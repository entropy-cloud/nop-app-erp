# 采购域页面设计要点

> 本文档定义采购域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-purchase.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`three-way-match.md`、`returns.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **状态驱动UI**：操作按钮、可编辑字段、页面导航均受三轴状态（docStatus / approveStatus / paidStatus）控制，不依赖角色硬编码。
2. **单据头行分离**：所有业务单据采用"头（卡片式分组）+ 明细行子表（可编辑表格）"结构，头承载组织/日期/币种等公共维度，行承载物料/数量/金额等业务明细。
3. **子表行内编辑**：明细行直接在表格内编辑（inline editing），不弹窗逐行编辑，提升批量操作效率。物料选择使用弹窗选择器（M2M picker）。
4. **自动推算**：选择物料后自动带出默认单价/税率/单位；修改数量/单价后自动重算金额/税额/价税合计，减少手工计算。
5. **单据回链可视化**：页面顶部或侧边展示上下游单据链（如采购订单→入库单→发票），支持点击跳转。

## 通用页面模式

### 列表页结构

```
┌────────────────────────────────────────────────────┐
│ [新建]  [批量审核]  [导出]                          │
│ ┌─ 筛选区 ──────────────────────────────────────┐ │
│ │ 单据编号: [___]  供应商: [选择]  状态: [下拉]  │ │
│ │ 业务日期: [从] ~ [至]  组织: [选择]            │ │
│ │ [查询] [重置]                                  │ │
│ └───────────────────────────────────────────────┘ │
├────────────────────────────────────────────────────┤
│ 结果表格: (支持列排序/自定义列)                     │
│ ┌────────────────────────────────────────────────┐ │
│ │ # │ 单号  │ 供应商  │ 日期    │ 金额    │ 状态│ │
│ │ 1 │ PO-001│ 某某公司 │ 06-20  │ 10,000 │ 已审│ │
│ │   │                     [查看] [操作▾]       │ │
│ └────────────────────────────────────────────────┘ │
│ 分页: [<] 1 2 3 ... [>]  每页: 20                  │
└────────────────────────────────────────────────────┘
```

**要点**：
- 状态列用标签（tag/chip）展示，颜色区分：DRAFT(草稿)=灰色、SUBMITTED(已提交)=蓝色、APPROVED(已审核)=绿色、REJECTED(已驳回)=红色、CANCELLED(已作废)=灰色删除线 [注：列表页展示的"状态"为 docStatus(单据状态) 与 approveStatus(审核状态) 的组合概览]
- 操作按钮组（操作▾）根据单据状态动态渲染：草稿态(DRAFT+UNSUBMITTED)显示[编辑][提交][删除]，已提交(ACTIVE+SUBMITTED)显示[审核][驳回]，已审核(ACTIVE+APPROVED)显示[反审核][打印]
- 金额列对齐方式右对齐，数字千分位格式
- 单据编号链接点击进入详情/编辑页

- 底栏关联单据区域显示上下游链条，点击可跳转

## 各页面设计要点

### 采购订单

**页面入口**：采购管理 → 采购订单

**头区特殊字段**：
- 供应商（必填）：使用 M2M 弹窗选择，支持按名称/编码/税号搜索
- 付款条件：下拉选择，选中的付款条件自动影响订单行价格计算（预付比例、账期等）
- 交货地址：默认取供应商主数据地址，允许修改

**子表特殊行为**：
- 物料选择弹窗显示：物料编码、名称、规格型号、库存单位、参考采购价、库存可用量 [管伊佳#Material+MaterialExtend]
- 允许输入物料不存在的"备注行"（非物料类费用，如运费、杂费），类型标记为"费用"
- 行级别支持"赠品行"标记（单价为0、数量不为0，仍影响库存且生成成本）

**状态联动**：
- 订单审核通过后，自动生成"待入库"待办事项
- 订单部分入库后，头区显示"已入库 X / Y 行"进度条

**页面间导航**：
- 订单详情页顶部"关联单据"区域显示该订单生成的入库单列表、发票列表
- 支持从订单页"创建入库单"一键跳转（携带订单信息）

### 采购入库

**页面入口**：采购管理 → 采购入库

**与采购订单的区别**：
- 入库单必须关联（或不关联）采购订单行
- 入库单审核需要校验：物料/仓库有效性、序列号唯一性（如启用序列号管理）

**头区特殊字段**：
- 来源订单：[选择] 弹窗选择已审核采购订单，选中后自动复制订单行到子表
- 仓库（必填）：默认从行级仓库，或头级统一仓库
- 入库类型：下拉选择（正常入库 / 退货入库 / 赠品入库），影响库存科目方向 [注：当前 ORM 中 ErpPurReceive 尚无 receiveType 字段，设计意图，待补充 ORM] [ERPNext#Stock Entry Type]

**子表特殊行为**：
- 支持分批入库：同一订单行可多次入库，子表显示"已入库/未入库"数量 [管伊佳#type+subType]
- 启用批次管理时，每行显示批次号输入列、效期列
- 启用序列号管理时，每行显示序列号录入弹窗（批量扫描输入）
- 超收校验：入库数量 > 订单数量 × (1 + 容差) 时审核报错 [three-way-match.md]

**后续操作**：
- 审核通过后：库存增加、生成暂估应付凭证（异步）、标记订单已入库数量
- 入库单详情页底部显示生成的凭证号、库存移动单号

### 采购发票

**页面入口**：采购管理 → 采购发票

**头区特殊字段**：
- 供应商发票号（必填）：供应商开具的发票号码
- 发票类型：下拉选择（增值税专用发票 / 增值税普通发票 / 收据） [注：当前 ORM 中 ErpPurInvoice 尚无 invoiceType 字段，设计意图，待补充 ORM]
- 发票日期：不同于业务日期，记录供应商开票时间

**子表特殊行为**：
- 发票行必须关联（或不关联）入库单行
- 三单匹配校验在提交/审核时触发：发票数量 ≤ 入库数量，单价差异在容差内 [three-way-match.md]
- 价税分离自动计算：发票行金额 = 不含税金额 + 税额 [管伊佳#价税分离]
- 运费/杂费输入行：类型标记为"费用"，不关联物料，用于 landed cost 分摊

**状态信息区**：
```
┌─────────────────────────────────────┐
│ 付款状态: ● 未付                    │
│ 已核销金额: ¥ 0.00                  │
│ 关联付款单: [PAY-001▶] [PAY-002▶]   │
│ 凭证状态: ● 已过账                  │
│ 凭证号: ACC-202406-001              │
└─────────────────────────────────────┘
```

**后续操作**：
- 审核通过后：生成应付凭证
- 发票详情页可直接创建付款单（携带发票信息）
- 红字发票与蓝字发票通过"冲销来源"关联，详情页展示红冲链条 [returns.md]

### 付款单

**页面入口**：采购管理 → 付款单

**头区特殊字段**：
- 供应商（必填）、付款账户（必填，银行存款/现金）
- 付款方式：下拉选择（电汇 / 支票 / 现金 / 票据）
- 付款日期、单据编号（自动）
- 整单折扣金额（供应商折扣）

**核销明细子表**（特殊结构，非物料行）：
```
┌────────────────────────────────────────────────┐
│ [+] [自动核销]                                 │
│ ┌────┬────────┬────────┬────────┬────────┬────┐│
│ │选择│发票号   │发票金额│未付金额│本次付款│备注││
│ │ ☑  │ INV-001│ 10,000 │ 3,000  │ 3,000  │   ││
│ │ ☐  │ INV-002│ 5,000  │ 5,000  │ 0      │   ││
│ └────┴────────┴────────┴────────┴────────┴────┘│
│ 本次付款合计: 3,000                              │
└────────────────────────────────────────────────┘
```

**要点**：
- 核销明细显示该供应商所有未付清发票
- "自动核销"按钮按先进先出或指定顺序自动分配付款金额
- 本次付款合计不能超过付款单金额
- 多账户付款：拆分明细行按不同账户支付 [管伊佳#反例：多账户逗号字符串→应改为子表]

### 采购退货

**页面入口**：采购管理 → 采购退货

**头区特殊字段**：
- 来源入库单：[选择] 弹窗选择已审核采购入库单
- 退货类型：单选（部分退货 / 全额退货 / 换货）
- 退货原因（必填）：下拉选择（质量不合格 / 数量不符 / 规格不符 / 重复采购 / 其他）
- 退货仓库：默认取原入库仓库

**子表特殊行为**：
- 从来源入库单导入入库行数据，显示"原入库数量 / 可退数量 / 本次退货数量"三列
- 退货数量 ≤ 可退数量（原入库数量 - 已退货数量），超限报错 [returns.md]
- 退货金额自动计算（取原入库单价 × 退货数量，或按配置的其他退货成本策略）

**状态信息**：
- 显示原入库单状态、关联订单号
- 若原入库单已开票，显示"已开票"标记，并提示退货将生成红字发票

## 跨页面导航流

### 正向采购流程导航

```
采购订单列表 ──[新建]──→ 采购订单编辑
      ↓ [审核通过]
      ↓ (自动生成待入库待办)
采购入库列表 ──[新建/从订单]──→ 采购入库编辑
      ↓ [审核通过]
      ↓ (库存增加，暂估凭证)
采购发票列表 ──[新建/从入库]──→ 采购发票编辑
      ↓ [审核通过]
      ↓ (应付凭证)
付款单编辑 ←──[从发票创建]── 采购发票详情
      ↓ [审核通过]
      ↓ (付款凭证 + 核销)
      → 闭环
```

### 逆向导航（追溯）

```
采购订单详情
  ├── [关联入库单列表] → 采购入库详情 → [库存移动单] → 库存流水
  ├── [关联发票列表]   → 采购发票详情 → [应付凭证]   → 凭证详情
  └── [关联付款单]     → 付款单详情   → [付款凭证]   → 凭证详情
```

## Line 子实体 form 分组模板

> 适用范围：采购域 8 个 Line 子实体独立 `view.xml` 的 `<form id="view">` / `<form id="edit">` 分组。
> 决策来源：`docs/plans/2026-07-19-1818-2-f3-core-line-and-remaining-main-form-layout.md` Phase 0.A。
> 父视图内嵌的子表行内编辑（child-table-editor）属 F4 Phase 2 范围，不在本节约定内。

### 统一模板（5 组）

采购域所有 Line 实体共享以下 5 组结构（缺字段则该组省略，不强行保留空组）：

```xml
<form id="view" size="lg">
    <layout x:override="replace">
=========>baseInfo[基本信息]======
 lineNo[行号] materialId[物料]
 skuId[SKU] uoMId[计量单位]
 warehouseId[收货仓库]
=========>quantity[数量与价格]======
 quantity[数量] unitPrice[单价]
 amount[金额(不含税)] receivedQuantity[已收货数量]
 invoicedQuantity[已开票数量]
=========>tax[税务信息]======
 taxRate[税率(%)] taxRateId[税率主数据]
 taxAmount[税额] amountWithTax[金额(含税)]
=========>reference[业务关联]======
 orderId[订单ID] projectId[项目]
=========^audit[审计信息]=========
 remark[备注]
 createdBy[创建人] createTime[创建时间]
 updatedBy[修改人] updateTime[修改时间]
    </layout>
</form>
```

### 各 Line 特化点

| 实体 | 特化 |
|------|------|
| ErpPurOrderLine | 完整 5 组（baseInfo / quantity / tax / reference / audit） |
| ErpPurReceiveLine | quantity 额外 `rejectedQuantity/batchNo`；reference 含 `orderLineId`；tax 无 `amountWithTax`（实体未含） |
| ErpPurInvoiceLine | baseInfo 无 `warehouseId/skuId`；reference 含 `receiveLineId`；quantity 无 `receivedQuantity` |
| ErpPurPaymentLine | 仅 3 组（baseInfo + amount[amount] + reference[paymentId/invoiceId] + audit）；无 quantity/tax 组 |
| ErpPurReturnLine | quantity 含 `reason`；reference 含 `receiveLineId`；tax 无 `amountWithTax` |
| ErpPurRequisitionLine | 无 tax 组；quantity 含 `requiredDate/suggestedSupplierId/projectId`；reference 仅 `requisitionId` |
| ErpPurQuotationLine | quantity 含 `leadTimeDays`；无 `amountWithTax`/`receivedQuantity`；reference 仅 `quotationId` |
| ErpPurRfqLine | 极简（baseInfo + quantity + reference[rfqId] + audit）；无 tax 组 |

### query 表单基线（5+ 字段 + filterOp）

所有 purchase Line 实体的 `<form id="query">` 至少含：`lineNo`（filterOp=eq）+ `materialId` + `warehouseId`（视实体而定）+ `orderId`/对应父头 ID + `businessDate`（filterOp=date-between，视实体是否含日期字段）。PaymentLine 等无 `businessDate` 的实体，沿用 `lineNo` + `invoiceId` 等替代字段达到 ≥5 字段基线。
