# 物流/运输域页面设计要点

> 本文档定义物流/运输管理（TMS）模块关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-logistics.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`carrier-integration.md`。
> 调研引用格式 `[源项目#要点]`，参考 🟢 Metasfresh shipping UI 范式。

## 设计原则

1. **状态驱动 UI**：操作按钮、可编辑字段、页面导航均受发运单状态（DRAFT/ADVISED/DISPATCHED/IN_TRANSIT/DELIVERED/CANCELLED）控制。
2. **承运商网关可视化**：网关调用状态（成功/失败/重试中）在发运单详情页直观展示，支持手动重试。
3. **追踪时间线地图**：在途发运单展示追踪时间线和位置地图，用户无需跳转承运商页面即可查看物流轨迹。
4. **承运商凭证安全**：API 密钥和令牌页面脱敏显示，不暴露明文。加密状态和最后修改时间可查。
5. **多包裹可视化**：发运单的包裹拆分以卡片或子表形式呈现，每个包裹状态独立展示。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 | 对应实体 |
|------|------|----------|--------|----------|
| 发运单列表 | 列表页 | 发货员 | ★★☆ | ErpLogShipment |
| 发运单编辑 | 表单+子表 | 发货员 | ★★★ | ErpLogShipment + ShipmentLine + ShipmentParcel |
| 发运单详情 | 详情页 | 发货员/物流主管 | ★★★ | ErpLogShipment（含追踪时间线） |
| 承运商管理 | 列表+详情 | 物流管理员 | ★★☆ | ErpLogCarrier |
| 承运商配置 | 表单 | 物流管理员 | ★★★ | ErpLogCarrierConfig（含凭证设置） |
| 追踪仪表板 | 概览页 | 物流主管 | ★★☆ | 聚合查询 |
| 比价面板 | 弹窗/侧栏 | 发货员 | ★★☆ | 实时 RateQuote |

## 各页面设计要点

### 发运单列表

**页面入口**：运输管理 → 发运单

**特殊筛选器**：
- 状态筛选（多选，支持排除已签收/已取消）
- 异常筛选（仅显示网关异常/追踪超期/退回）
- 承运商筛选
- 预计送达日期范围

**列表操作**：
- 状态列支持快速过滤（点击状态标签 → 筛选该状态全部记录）
- 运单号列显示复制按钮
- 支持批量"获取面单"（选中多条 → 批量下载 PDF/ZIP）
- 支持批量"确认发运"（仅 DRAFT 状态）

```
│ └────────────────────────────────────────────────┘  │
│ 底部: [确认选择: 京东物流 - 标准 ¥18]               │
└────────────────────────────────────────────────────┘
```

**要点**：
- 比价结果按"推荐"排序（默认按 运费×时效 加权评分）
- 支持手动切换排序方式（按运费升序/按时效升序）
- 选择后自动回填发运单的承运商 + 服务类型 + 运费
- 比价失败或超时时隐藏对应承运商行并显示"报价失败"
- 结果缓存 30 分钟
