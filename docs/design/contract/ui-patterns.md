# 合同管理域页面设计要点

> 本文档定义合同管理域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-contract.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`invoice-plan.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **版本可追溯**：每次合同条款变更创建新版本（ContractVersion），详情页集中管理版本列表，支持任意两版本差异对比。当前执行版本以 isCurrent 标记。
2. **开票计划驱动财务**：合同的财务影响通过开票计划（InvoicePlan）管理，按预付款/里程碑/月结/完工四种条款生成计划项，计划执行后关联实际发票。
3. **合同模板化**：常用合同条款以模板（Template）形式管理，按合同类型（采购/销售/服务）分类。新建合同时从模板继承条款，减少重复录入。
4. **弱指针关联订单**：合同不与订单做 ORM 强外键关联，通过弱指针（relatedBillType/relatedBillCode）引用下游订单/发票，保持合同域独立可拔。
5. **消耗计费可追溯**：用量计费场景的消耗记录（ConsumptionLine）逐条记录，标记来源单据类型与单号，支撑订阅计费和收入确认。

**要点**：
- 状态标签颜色：DRAFT=灰色、NEGOTIATION=蓝色、ACTIVE=绿色、SUSPENDED=橙色、EXPIRED=灰色、TERMINATED=红色

- 合同号链接点击进入详情页
- 临近到期合同（30天内到期）行高亮提醒

## 各页面设计要点

### 合同详情/编辑

**页面入口**：合同管理 → 合同列表 → 点击合同号

**头区特殊字段**：
- 父合同：支持合同拆分/补充合同场景，父合同ID指向原合同
- 合同方向：进向（我方付款，采购合同）vs 出向（客户付款，销售合同），影响开票计划的方向
- 合作方：M2M弹窗选择客户或供应商

**合同行子表**：
- 物料选择弹窗显示物料编码、名称、规格型号、单位
- 非物料行（服务费、软件授权等）允许物料为空，填写行描述
- 行金额变更后触发合同总额重算

**版本逻辑**：
- 合同草稿状态下直接编辑不计入版本
- 合同签署后修改必须创建新版本，旧版本只读归档
- 版本内容存储为结构化字段或富文本，支持差异对比算法（diff）[Axelor#ContractVersion]
- 版本审批：草稿→定稿→已签署，定稿后不可编辑

**业务规则**：
- 合同到期前30天系统自动发送续约提醒
- 中止（SUSPENDED）状态暂停开票计划执行
- 合同终止（TERMINATED）后未执行的开票计划自动取消

### 版本对比

**页面入口**：合同详情 → 版本历史 Tab → 勾选两版本 → [对比]

**布局**：
```
┌──────────────────────────────────────────────────────────┐
│ 对比版本: v.2 (06-20)  ← →  v.3 (06-28)                │
├──────────────────────────────────────────────────────────┤
│ ┌──── v.2 ──────────────────┐ ┌──── v.3 ──────────────┐ │
│ │ 合同总额: ¥ 500,000       │ │ 合同总额: ¥ 550,000   │ │
│ │                          │ │ [+¥ 50,000]           │ │
│ │ 付款条件: 30天            │ │ 付款条件: 45天         │ │
│ │                          │ │ [变更]                 │ │
│ │ 条款3.1: 交货期30天       │ │ 条款3.1: 交货期45天    │ │
│ │                          │ │ [变更]                 │ │
│ └──────────────────────────┘ └────────────────────────┘ │
│ 差异摘要: 金额+¥50,000, 付款条件变更, 交货期变更        │
└──────────────────────────────────────────────────────────┘
```

**要点**：
- 差异高亮：新增内容绿色背景，删除内容红色背景删除线，修改内容黄色背景
- 数值字段差异显示差值（↑↓ 箭头）
- 支持仅显示差异行（过滤相同行）

### 开票计划

**页面入口**：合同详情 → 开票计划 Tab / 合同管理 → 开票计划

**布局**：
```
┌────────────────────────────────────────────────────────────┐
│ [+新建开票计划]  [批量生成]                                │
│ ┌──────┬────────┬────────┬──────┬────────┬──────┬────────┐│
│ │ 选择 │ 计划日  │ 条款类型│ 金额 │ 已开票 │ 发票号│ 状态  ││
│ │ ☐    │ 06-15  │ 预付   │100,000│100,000 │ INV01│ ✅已开 ││
│ │ ☐    │ 08-15  │ 里程碑 │200,000│ 0      │ —    │ ⏳待开 ││
│ │ ☐    │ 10-15  │ 完工   │200,000│ 0      │ —    │ ⏳待开 ││
│ └──────┴────────┴────────┴──────┴────────┴──────┴────────┘│
│ 合计: 500,000         已开票: 100,000   待开票: 400,000   │
└────────────────────────────────────────────────────────────┘
```

**要点**：
- 条款类型：预付/里程碑/月结/完工，影响开票触发条件 [Axelor#InvoicePlan InvoiceTerm]
- "批量生成"按合同行×条款类型自动生成计划行
- 状态标签：待开(灰色)/部分开票(蓝色)/已开票(绿色)/已付款(深绿)
- 已开票行显示关联发票号，链接可跳转销售/采购发票详情
- 合同方向为进向时，开票计划触发的发票进入采购域；出向时进入销售域

### 消耗记录

**页面入口**：合同详情 → 消耗记录 Tab / 合同管理 → 消耗记录

**布局**：
```
┌────────────────────────────────────────────────────────┐
│ ┌──────┬────────┬────────┬───────┬────────┬──────────┐│
│ │ 日期  │ 物料   │ 消耗数量│ 单价  │ 金额   │ 来源单据  ││
│ │ 06-10│ API调用│ 10,000 │ 0.5   │ 5,000  │ 计费系统#││
│ │ 06-11│ API调用│ 8,000  │ 0.5   │ 4,000  │ 计费系统#││
│ │ 06-12│ API调用│ 12,000 │ 0.5   │ 6,000  │ 计费系统#││
│ └──────┴────────┴────────┴───────┴────────┴──────────┘│
│ 本月合计: 15,000   累计: 180,000   含税: 194,400      │
└────────────────────────────────────────────────────────┘
```

**要点**：
- 消耗记录由外部系统（计费/用量采集）写入，非手工录入
- 来源单据位标记原始单据类型和单号，支持追溯
- 金额 = 消耗数量 × 单价（取自合同行单价或配置单价）
- 消耗记录可作为开票计划的输入源（月结场景）

### 合同模板管理

**页面入口**：合同管理 → 合同模板

**布局**：标准列表页 + 编辑弹窗/页面
- 编码、名称、适用合同类型、是否启用
- 模板内容：富文本编辑器，支持变量占位符如 `{合同名称}` `{合作方}` `{合同总额}`
- 新建合同时选模板 → 自动填充条款内容，变量占位符替换为实际值

## 跨页面导航流

### 正向合同管理流程导航

```
合同模板列表 ──[参考模板]──→ 合同编辑(新建从模板)
      ↓
合同列表 ──[新建]──→ 合同详情/编辑
      ↓ [提交审批] → [审批通过] → [签署]
      ↓ (状态变为 ACTIVE，当前版本变为 v.1)
合同详情 → [版本历史] Tab ──[新建版本]──→ 版本编辑
      ↓ [签署定稿]
      ↓ (版本 v.2 生效)
合同详情 → [开票计划] Tab ──[批量生成]──→ 开票计划列表
      ↓ (计划执行/实际开票)
      → 开票计划行标记已开票
合同详情 → [消耗记录] Tab
      ↓ (消耗累计触发开票)
      → 开票计划行按消耗生成
      ↓ (合同到期)
      → 合同状态 → EXPIRED
```

### 逆向导航（追溯）

```
合同详情
  ├── [版本历史] → 版本对比页 ← 勾选两版本对比
  ├── [开票计划] → 开票计划行 → [发票号] → 销售/采购发票详情
  ├── [消耗记录] → 消耗行 → [来源单据] → 原始计费记录
  ├── [父合同] → 父合同详情
  └── [关联订单(弱指针)] → 订单详情
```

## 主交易实体 form 布局分组

> 适用范围：contract 域 8 个主交易实体（不含已 1500-1 覆盖的 `ErpCtContract`）独立 `view.xml` 的 `<form id="view">` / `<form id="edit">` 分组。
> 决策来源：`docs/plans/2026-07-20-2059-2-f3-p2p3-ext-masterdata-form-layout.md` Phase 0.D。
> 合同版本（ContractVersion）突出版本号/生效日期/isCurrent；返利协议（RebateAgreement）/计提（Accrual）/结算（Settlement）三实体按 金额 + 状态 + 过账 分组。

### query 表单基线

所有 contract 域主实体的 `<form id="query">` 至少含 5 个查询字段。`code`/`versionNo`/`ruleName`/`checklistItem`/`docName` 配 `filterOp=like`；`contractId`/`partnerId`/`orgId`/`rebateAgreementId`/`approvalMatrixId`/`contractVersionId`/`status`/`approvalStatus`/`isCurrent` 配 `filterOp=eq`；含日期字段（`agreementDate`/`versionDate`/`settlementDate`/`businessDate`）配 `filterOp=date-between`。
