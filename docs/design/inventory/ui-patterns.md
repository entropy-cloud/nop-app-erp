# 库存域页面设计要点

> 本文档定义库存域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-inventory.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`cross-domain.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **三层库存可见**：页面设计需让用户清晰感知移动单（计划）、流水（记录）、余额（结果）三层的对应关系。操作移动单时实时显示对余额的预期影响。
2. **作业类型驱动 UI**：移动单页面结构根据作业类型（收/发/内/制）动态显示/隐藏字段（如调拨显示双仓库，盘点显示盘点前/后数量）。
3. **批次/序列号行内管理**：启用批管/序列号的物料，子表行附带批次/序列号列，支持行内选择或扫码录入。
4. **不可变流水只读**：库存流水页面纯只读（不可变），无编辑/删除操作，仅支持查询和导出。冲销通过新建反向移动单实现。
5. **余额实时更新**：库存余额页面支持按物料×仓库×批次多维筛选，移动单确认后余额页面数字即时反映变化。

## 各页面设计要点

### 库存移动单

**页面入口**：库存管理 → 库存移动

**页面功能**：独立创建的移动单（非业务单据联动）、审核/执行移动单、查询移动单

**头区特殊字段**：
- 作业类型（必填）：下拉选择（采购入库 / 销售出库 / 调拨入库 / 调拨出库 / 生产领料 / 完工入库 / 盘盈 / 盘亏 / 其他入库 / 其他出库）
- 作业类型选定后，页面动态调整：
  - 收类型→目的库位必填、来源库位隐藏
  - 发类型→来源库位必填、目的库位隐藏
  - 调拨→来源库位、目的库位均必填
  - 显示对应作业类型的默认库位、默认科目

**子表特殊行为**：
- 物料选择后自动带出：基本单位、换算系数、默认仓库/库位（从作业类型配置取）
- 启用批次管理的物料：行内显示批次下拉选择框，显示该批次可用量
- 启用序列号管理的物料：行内显示序列号录入框（手动或扫描枪），每行可录入多个序列号
- 出库类型移动单：提交确认时做可用量校验，可用量不足的行高亮标记
- 行级显示"移动前可用量"和"移动后可用量"（提交确认前即可预览）

**状态展示**：
- 列表页状态列：DRAFT=灰色、CONFIRMED=蓝色、DONE=绿色、CANCELLED=灰色删除线
- 移动单详情页显示上下游追溯链（源业务单据、关联的移动单）

### 库存流水

**页面入口**：库存管理 → 库存流水

**纯只读查询**：
```
┌─ 筛选区 ──────────────────────────────────────────┐
│ 物料: [选择]  仓库: [选择]  日期范围: [从] ~ [至]   │
│ 批次: [选择]  业务类型: [下拉]                      │
│ [查询] [重置] [导出]                                │
└───────────────────────────────────────────────────┘
┌─ 流水列表 ─────────────────────────────────────────┐
│ 时间 | 物料 | 方向 | 数量 | 变动后余量 | 单价 | 来源 │
│ 06-20 | MAT001 | + | 100 | 200 | 10.5 | PO-001  │
│ 06-21 | MAT001 | - | 50  | 150 | 10.5 | SO-001  │
└───────────────────────────────────────────────────┘
```

**要点**：
- 不可执行任何编辑/删除操作
- 方向用 +/- 和颜色区分（入库绿/出库红）
- "来源"列可点击跳转到源单据详情
- 支持按物料×仓库×批次分组汇总

### 库存余额

**页面入口**：库存管理 → 库存余额

**多维筛选与展示**：
```
┌─ 筛选区 ──────────────────────────────────────────┐
│ 物料: [选择]  仓库: [选择]  库位: [选择]            │
│ 批次: [选择]  含零库存: [☐]                         │
└───────────────────────────────────────────────────┘
┌─ 余额表格 ─────────────────────────────────────────┐
│ 物料编码 | 物料名称 | 仓库 | 批次 | 现有量 | 预留量 │
│ 可用量 | 单位成本 | 库存价值                        │
│ MAT-001 | 螺丝 M6 | WH1 | LOT01 | 200 | 50 | 150  │
│ MAT-001 | 螺丝 M6 | WH1 | LOT02 | 100 | 20 | 80   │
└───────────────────────────────────────────────────┘
```

**要点**：
- 现有量 = 所有流水变动之和，预留量 = 已确认未完成的出库移动单数量之和
- 可用量 = 现有量 − 预留量，用绿色（充足）/ 黄色（不足）/ 红色（为 0）展示
- 点击行弹出"流水明细"抽屉，展示该维度的库存流水

### 库存调拨

**页面入口**：库存管理 → 库存调拨

**特点**：生成两张移动单（来源库位出库 + 目的库位入库），由同一调拨单驱动。

**页面字段**：
- 调出仓库 / 调入仓库（必填，不能相同）
- 调出库位 / 调入库位
- 调拨类型：同级调拨 / 上下级调拨（影响成本计算方式）

**子表**：物料、数量、批次（启用批管时）。提交确认后生成两条关联的移动单。

### 库存盘点

**页面入口**：库存管理 → 库存盘点

**盘点流程页面**：
```
┌────────────────────────────────────────────────┐
│ 阶段 1: 创建盘点单                              │
│ 盘点仓库: [选择]  盘点类型: (全盘 / 抽盘)       │
│ ────────────────────────────────────────────── │
│ 阶段 2: 录入实盘数量                            │
│ ┌────────────────────────────────────────────┐ │
│ │ 物料 | 库位 | 账面数量 | 实盘数量 | 差异   │ │
│ │ MAT01| A01  | 100       | 98      | -2    │ │
│ │ MAT02| A02  | 50        | 52      | +2    │ │
│ └────────────────────────────────────────────┘ │
│ ────────────────────────────────────────────── │
│ 阶段 3: 审核确认 → 自动生成盘盈/盘亏移动单      │
└────────────────────────────────────────────────┘
```

**要点**：
- 盘点单状态基于 `erp-inv/move-status` 字典：DRAFT(草稿) → CONFIRMED(盘点中) → DONE(已完成) → CANCELLED(已取消)
- CONFIRMED 阶段可录入实盘数量，系统自动计算差异
- 盘点期间在业务层面建议锁定该仓库的出入库操作（ORM 无显式锁字段，通过盘点期间禁止出入库校验实现）
- DONE 后自动生成盘盈/盘亏移动单（差异为正→盘盈入库单，差异为负→盘亏出库单）
- 盘点差异在审核前可修改

### 批次/序列号台账

**批次台账**：每行显示物料、批号、生产日期、效期、库存量、状态（正常/过期/冻结）。支持查询各批次的出入库流水。

**序列号台账**：每行显示序列号、物料、当前状态（在库/已售/冻结/报废）、入库单号、出库单号。支持按序列号全局搜索。 [管伊佳#SerialNumber]

## 跨页面导航流

```
业务单据审核 → 自动生成移动单(DRAFT)
    ↓ 系统联动确认(CONFIRMED)
    ↓ 执行完成(DONE)
库存流水(不可变) ← ← ← 写流水
    ↓
库存余额(实时更新)
    ↓ 查询追溯
移动单详情 → [源业务单据] → [上下游移动单] → [流水明细]
```

手动操作导航：
```
移动单列表 → [新建] → 选择作业类型 → 填单 → 确认
    ↓
调拨单 → [审核] → 自动生成出入库移动单
盘点单 → [开始] → [录入盘点] → [审核] → 自动生成盘盈/盘亏移动单
```
