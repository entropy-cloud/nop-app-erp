# 库存域（inventory）

## 目的

说明库存域的业务语义、三层库存模型、状态机与跨域协作。库存域是采购、销售、财务三个域的共享执行底座，所有影响库存的业务动作最终都转化为库存移动单。

## 边界

- 本域负责：库存移动单（计划/执行）、库存流水（不可变）、库存余额（快照）、调拨、盘点、批次/序列号台账、作业类型。
- 本域不负责：物料/仓库/库位主数据（master-data 域）；采购/销售单据本身（purchase/sales 域）；存货估值过账的凭证生成（finance 域，本域只提供流水数据）。
- 持久化字段、字典、状态码以 `model/app-erp-inventory.orm.xml` 为准。
- 跨域协作规则见 `../domain-design-guidelines.md`，全局流程见 `../flow-overview.md`。

## 三层库存模型

库存域采用"移动—流水—余额"三层结构：

| 层 | 业务含义 | 可变性 |
|----|----------|--------|
| 库存移动单（StockMove） | 计划层"一次移动意图"，承载来源/目的库位、数量、关联源单 | 可确认/取消 |
| 库存流水（StockLedger） | 不可变的库存变动记录，含移动后余量、单位成本、库存价值、FIFO 队列 | 不可变，冲销走反向流水 |
| 库存余额（StockBalance） | 按物料×仓库×库位×批次的当前存量快照，含可用量、预留量 | 由流水驱动更新 |

业务单据（采购入库/销售出库等）只生成库存移动单；移动单确认后产生库存流水并更新余额。

### 核心业务对象

| 对象 | 业务含义 |
|------|----------|
| 库存移动单头/行（StockMove） | 一次移动意图，含来源/目的库位、数量、状态、上下游追溯链、关联源单类型与编号 |
| 库存流水（StockLedger） | 不可变变动记录，含变动数量（入库正/出库负）、变动后余量、单位成本、库存价值、FIFO 队列 |
| 库存余额（StockBalance） | 物料×仓库×库位×批次维度的现有量、预留量、可用量、平均成本 |
| 调拨单（TransferOrder） | 内部仓库间或库位间的库存移动 |
| 盘点单（StockTake） | 账面与实物差异的调整 |
| 批次台账（Batch） | 物料的批次/效期管理，含入库日期、生产日期、保质期、状态 |
| 序列号台账（SerialNumber） | 单品序列号追踪，含状态（在库/已售）、入库单号、出库单号 |
| 作业类型（OperationType） | 参数化的库存作业分类（收/发/内/制），绑定默认库位与科目 |

## 状态机

库存移动单有独立状态机（草稿→已确认→已完成/已取消），按 `docs/skills/state-machine-business-review-prompt.md` 的 10 个审查维度设计。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

库存域既是被调用方（业务单据触发生成移动单），也是数据提供方（财务域消费流水过账）。详细规则见 [`cross-domain.md`](cross-domain.md)。

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 采购入库 | purchase | 入库单审核 → `IErpInvStockMoveBiz.generateMove(...)` 生成入库移动单 |
| 销售出库 | sales | 出库单审核 → 生成出库移动单 |
| 采购退货 | purchase | 生成反向出库移动单 |
| 销售退货 | sales | 生成反向入库移动单 |
| DRP 补货调拨 | drp | DRP APPROVED 且补货类型=TRANSFER → 创建仓间调拨单 |
| 存货估值过账 | finance | 财务域监听移动单完成事件，读流水生成存货估值凭证 |
| 主数据引用 | master-data | 移动单引用物料/SKU/仓库/库位编码 |

## 关键业务规则

1. **库存流水不可变**：一旦写入不可修改，冲销走反向流水（生成数量取负的新流水）。
2. **余额由流水驱动**：余额表不是独立维护，每次流水写入时同步更新对应维度的余额。
3. **预留量机制**：已确认未完成的出库移动单占用量计入预留量；可用量 = 现有量 − 预留量。
4. **可用量不足拒绝出库**：出库移动单确认时校验可用量（除非配置允许负库存，默认禁止）。
5. **单位换算**：移动单数量按 SKU 包装单位录入，落账时按换算系数转为基本单位数量写入流水。
6. **批次/序列号校验**：启用批次/序列号的物料，移动单必须指定批次/序列号；出库校验批次在库、序列号未售。
7. **上下游追溯**：移动单记录上游/下游移动单关联，支持全链路追溯（采购到货→生产领料→销售出库）。
8. **作业类型参数化**：作业类型绑定默认库位与科目映射，供财务域过账使用。

### 报废出库移动单

报废出库使用专用作业类型 `SCRAP`：

- **operationType**：`SCRAP`（报废出库）
- **默认库位**：报废库位（`warehouseType=SCRAP`）
- **科目映射**：借：营业外支出（或制造费用） / 贷：存货
- **触发场景**：质检不合格报废、盘点盘亏、过期报废
- **配置**：`erp-inv.scrap-operation-type` 可按物料分类配置默认报废作业类型

### 批次选择策略

出库时的批次自动选择策略：

| 策略 | 规则 | 适用场景 |
|------|------|----------|
| FIFO（先进先出） | 按入库日期从早到晚选择 | 默认策略 |
| FEFO（先到期先出） | 按保质期从近到远选择 | 保质期管理物料 |
| 手工指定 | 用户手动选择批次 | 高价/定制物料 |

**配置优先级**（从高到低）：

| 优先级 | 配置维度 | 说明 |
|--------|----------|------|
| 1 | 按仓库（ORM 字段） | `ErpMdWarehouse.batchSelectionStrategy` — 同一物料在不同仓库可用不同策略 |
| 2 | 按物料（ErpSysConfig） | `erp-inv.batch-strategy.<materialId>` — 物料级别覆盖 |
| 3 | 全局默认（ErpSysConfig） | `erp-inv.batch-selection-strategy`（默认 FIFO） |

> 仓库级配置的 ORM 字段定义见 `module-master-data/model/app-erp-master-data.orm.xml` → `ErpMdWarehouse.batchSelectionStrategy`。FEFO 仅对启用保质期管理的物料生效。
