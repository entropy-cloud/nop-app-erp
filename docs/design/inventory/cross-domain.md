# 库存域跨域协作

## 目的

说明库存域与采购、销售、财务、主数据域的协作规则、数据契约与一致性约束。

## 与采购域协作

### 采购入库（写库存）

- **触发**：采购入库单审核通过（purchase 域）。
- **调用**：purchase 域 Processor 调用 `IErpInvStockMoveBiz.generateMove(...)`，传入物料/SKU、仓库/库位、数量、批次、关联源单（采购入库单号）。
- **结果**：生成交库移动单（作业类型 incoming），数量为正数（增加库存）。
- **同步采购订单未交量**：purchase 域自己维护订单未交量（订单数量 − 累计入库数量），本域不感知。

### 采购退货（反向出库）

- **触发**：采购退货单审核通过。
- **结果**：生成出库移动单（outgoing），数量为正数（减少库存，因为是退给供应商）。

## 与销售域协作

### 销售出库（扣减库存）

- **触发**：销售出库单审核通过（sales 域）。
- **调用**：sales 域 Processor 调用 `IErpInvStockMoveBiz.generateMove(...)`。
- **结果**：生成出库移动单（outgoing），数量为正数（减少库存）。
- **可用量校验**：生成移动单前校验可用量（现有量 − 预留量）是否充足；不足时拒绝并回滚整个出库单审核。

### 销售退货（反向入库）

- **触发**：销售退货单审核通过。
- **结果**：生成入库移动单（incoming），数量为正数（增加库存）。

## 与财务域协作（存货估值过账）

### 数据契约

库存域在移动单完成（DONE）时，向财务域提供以下数据用于生成存货估值凭证：

| 数据 | 来源 | 用途 |
|------|------|------|
| 移动单号、移动类型（收/发/内/制） | StockMove 头 | 凭证业务类型映射（PURCHASE_INPUT/SALES_OUTPUT 等） |
| 物料/SKU、仓库/库位 | StockMove 行 | 科目映射（按产品/仓库配置存货科目） |
| 变动数量、单位成本 | StockLedger 流水 | 金额计算（数量 × 单位成本） |
| 库存价值变动 | StockLedger 流水 | 借贷金额 |

### 触发机制

- 库存域在移动单 DONE 时发布事件（含上述数据）。
- 财务域订阅事件，异步生成存货估值凭证（post-commit，失败可重试）。
- 库存域不依赖财务域的过账结果——移动单完成即视为库存记账成功，凭证生成是后置异步动作。

### 成本数据归属

- 单位成本的计算（移动加权平均/FIFO/批次）属于库存域还是财务域，取决于实现：
  - 若成本由库存流水维护（StockLedger 含 `unitCost`/`stockValue`），则库存域提供，财务域消费。
  - 若成本由独立的成本核算模块维护，则属于财务域，库存域只提供数量。
- 当前基线约定：**单位成本由库存流水维护**，财务域直接消费流水中的成本字段。

## 与主数据域协作

库存域引用主数据域的数据（物料/SKU/仓库/库位/计量单位），通过 `IErpMd*Biz` 接口查询，不做 ORM 跨工程 `refEntityName`。

| 主数据 | 库存域用途 |
|--------|------------|
| 物料/SKU | 移动单行的物料标识、批次/序列号开关 |
| 仓库/库位 | 移动单的来源/目的库位 |
| 计量单位 | 数量换算（包装单位 → 基本单位） |

## 余量校验规则（跨域通用）

**主 owner**：本节。`flow-overview.md` 的 L3 节摘要引用此处。

- **出库前校验**：销售出库、生产领料等减少库存的业务，在移动单确认前校验可用量（现有量 − 预留量）是否充足。
- **校验维度**：物料 × 仓库 × 库位 × 批次（启用批次的物料按批次校验）。
- **不足处理**：可用量不足时拒绝移动单确认（抛业务异常），整个业务单据审核回滚。
- **负库存配置**：全局配置 `erp-inv.allow-negative-stock`（默认 false）。设为 true 时跳过可用量校验，允许余额为负（特殊业务场景，如先发货后入库）。
- **预留量释放**：出库移动单取消或冲销时释放预留量，恢复可用量。

### 在途库存规则

两步调拨（跨仓库）中的在途库存处理：

- **在途库**：`warehouseType=IN_TRANSIT` 的逻辑库位
- **所有权**：在途期间库存所有权属发出方（不转移法人所有权，除非跨法人调拨）
- **在途查询**：支持按调拨单号查询在途库存数量和金额
- **在途对账**：发出方已扣减、接收方未增加的数量在在途库中体现
- **在途超时**：超过配置天数（`erp-inv.in-transit-timeout-days`）未接收的在途库存自动告警

### 负库存会计处理

负库存是临时容错机制，不产生独立的会计事件：

- **成本计算**：负库存余额时出库，按移动加权平均成本出库（即使余额为负）
- **凭证生成**：凭证在业务单据审核时生成，与库存余额正负无关
- **不触发调整**：负库存不自动触发成本调整凭证
- **自然平滑**：补货入库后成本自然平滑（入库金额加入余额，正数后恢复正常）
- **监控告警**：负库存持续超过阈值（`erp-inv.negative-stock-alert-days`）时自动告警

## 快照语义

- **成本快照**：移动单确认时固化当时的单位成本，写入不可变流水。后续成本变化不修改已写入的流水。
- **批次快照**：移动单指定批次后，批次状态变化（如过期）不影响已完成的移动单。

## 跨法人调拨（参考 iDempiere 多公司架构）

### 业务场景

跨法人调拨是指货物在不同法人实体（独立核算公司）之间的转移，涉及内部交易和内部往来核算。

### 业务流程

```
调出公司创建调拨单 → 审核 → 发货出库 → 在途 → 调入公司收货 → 入库 → 内部交易凭证
     │               │          │        │          │           │          │
     ▼               ▼          ▼        ▼          ▼           ▼          ▼
  记录调出需求   调出库存减少  在途库存增加  在途库存减少  调入库存增加  生成内部往来凭证
```

### 与同法人调拨的区别

| 特性 | 同法人调拨 | 跨法人调拨 |
|------|------------|------------|
| 库存所有权 | 转移库位，不转移法人 | 转移法人所有权 |
| 财务处理 | 无凭证，仅库存移动 | 视同买卖，生成内部交易凭证 |
| 成本处理 | 同一成本体系 | 各自独立成本体系 |
| 税务处理 | 无税务影响 | 可能涉及增值税视同销售 |
| 审批流程 | 简单审批 | 需要双方公司审批 |

### 财务处理（参考 iDempiere 多公司内部交易）

#### 调出公司凭证

```
借：内部往来-调入公司（应收）
贷：存货（按成本价）
```

#### 调入公司凭证

```
借：存货（按成本价+内部利润）
贷：内部往来-调出公司（应付）
```

#### 内部利润处理

- **成本价调拨**：调出公司按成本价，调入公司也按成本价，无内部利润
- **加价调拨**：调出公司按成本价，调入公司按成本价+内部利润，差额挂"内部利润"
- **市价调拨**：调出公司按市价，调入公司按市价，差额视同销售

### 实现要点

| 设计点 | 实现方式 |
|--------|----------|
| 法人识别 | 调出方`orgId`与调入方`orgId`不同 |
| 内部往来科目 | 每个法人配置"内部往来-对方公司"科目 |
| 内部交易凭证 | 调出方与调入方各生成一组凭证 |
| 转移定价 | 按配置的成本价/加价率/市价确定内部交易价格 |
| 合并抵消 | 集团账套中自动抵消内部往来和内部利润 |

### 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `erp-inv.cross-org-transfer-pricing` | COST | 转移定价方式（COST/MARKUP/MARKET） |
| `erp-inv.cross-org-vat-treatment` | NONE | 增值税处理（NONE/DEEMED_SALE） |
| `erp-fin.intercompany-account-prefix` | IC_ | 内部往来科目前缀 |

## 一致性规则

- 库存余额更新与流水写入必须在同一事务（由库存域 Processor 保证）。
- 跨域调用（如 purchase 调 `IErpInvStockMoveBiz`）使用 Nop 分布式事务或调用方事务传播，保证业务单据审核与库存记账的一致性。
- 存货估值过账是异步后置动作，失败不阻塞库存记账；财务域通过 `posted` 标志与兜底扫描保证最终一致。
- 跨法人调拨生成内部交易凭证，调出方与调入方凭证在同一事务中生成，保证内部往来一致。
