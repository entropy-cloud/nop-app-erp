# 委外加工（外协）

## 目的

说明制造域的委外加工业务逻辑，包括委外订单、发料、收货、成本核算的完整流程。参考 Odoo `mrp_subcontracting` 模块设计，结合 nop-app-erp 的业财一体化架构。

## 业务流程

### 标准委外流程

```
委外订单创建 → 审核 → 发料给供应商 → 供应商生产 → 收货入库 → 委外发票 → 付款
     │           │          │            │           │           │         │
     │           │          │            │           │           │         ▼
     │           │          │            │           │           │    核销应付
     │           │          │            │           │           ▼
     │           │          │            │           │    生成委外应付凭证
     │           │          │            │           ▼
     │           │          │            │    库存入库（产成品，含加工费）
     │           │          │            ▼
     │           │          │    供应商确认收货（portal）
     │           │          ▼
     │           │    库存出库（原材料，供应商领料）
     │           ▼
     │    BOM 展开（仅外协工序）
     ▼
审批流程
```

### 供应商协同流程（参考 Odoo Portal 模式）

```
本公司创建委外订单 → 供应商 Portal 接收 → 供应商确认生产 → 供应商提交完工报告
          │                 │                  │                  │
          ▼                 ▼                  ▼                  ▼
     生成发料需求      供应商查看BOM       记录生产进度      触发收货入库
```

## 与自制工单的区别

| 特性 | 自制工单 | 委外订单 |
|------|----------|----------|
| 生产方 | 本公司 | 供应商 |
| BOM | 完整 BOM | 仅外协工序 BOM |
| 领料 | 本公司仓库 | 发料到供应商 |
| 入库 | 产成品入库 | 委外成品入库 |
| 成本 | 材料+人工+制造费用 | 材料+加工费 |
| 发票 | 无 | 加工费发票 |
| 供应商协同 | 无 | Portal 接收/确认/报告 |
| 质量管控 | 内部质检 | 来料检验+过程监控 |

## 核心业务对象

| 对象 | 业务含义 | 头/行 |
|------|----------|-------|
| 委外订单（SubcontractOrder） | 与供应商签订的委外加工协议 | 头/行（物料需求） |
| 委外发料单（SubcontractIssue） | 发料给供应商的出库单据 | 头/行 |
| 委外收货单（SubcontractReceipt） | 供应商完工后的入库单据 | 头/行 |
| 委外发票（SubcontractInvoice） | 供应商开具的加工费发票 | 头/行 |

## 状态机设计

### 委外订单状态机

```
草稿 (DRAFT)
  ├─ 提交审核 → 已提交 (SUBMITTED)
  │                ├─ 审核通过 → 已审核 (APPROVED)
  │                │              ├─ 发料中 → 已发料 (ISSUED)
  │                │              │              ├─ 生产中 → 已生产 (PRODUCED)
  │                │              │              │              ├─ 收货中 → 已收货 (RECEIVED)
  │                │              │              │              │              ├─ 完成 → 已完成 (COMPLETED)
  │                │              │              │              └─ 退回 → 已退回 (RETURNED)
  │                │              │              └─ 取消 → 已取消 (CANCELLED)
  │                │              └─ 取消 → 已取消 (CANCELLED)
  │                └─ 驳回 → 已驳回 (REJECTED)
  └─ 取消 → 已取消 (CANCELLED)
```

### 状态定义

| 状态 | 业务含义 | 可修改 |
|------|----------|--------|
| 草稿 (DRAFT) | 订单创建，待提交 | 是 |
| 已提交 (SUBMITTED) | 待审核人决策 | 否 |
| 已审核 (APPROVED) | 审核通过，待发料 | 否 |
| 已发料 (ISSUED) | 材料已发给供应商 | 否 |
| 已生产 (PRODUCED) | 供应商已确认生产完成 | 否 |
| 已收货 (RECEIVED) | 成品已入库 | 否 |
| 已完成 (COMPLETED) | 委外订单完成 | 否 |
| 已退回 (RETURNED) | 成品退回供应商 | 否 |
| 已取消 (CANCELLED) | 订单取消 | 否 |
| 已驳回 (REJECTED) | 审核驳回 | 是 |

## 成本核算

### 成本构成

```
委外成品成本 = 发出材料成本 + 加工费 + 损耗成本
```

- **发出材料成本**：按发出材料的库存成本计算（FIFO/移动加权平均）
- **加工费**：按委外发票金额（不含税）
- **损耗成本**：超合同约定损耗部分由供应商承担

### 成本分配规则

| 成本类型 | 分配方式 | 科目映射 |
|----------|----------|----------|
| 材料成本 | 按BOM标准消耗分配 | 借：委外物资 / 贷：原材料 |
| 加工费 | 按发票金额分配 | 借：委外物资 / 贷：应付账款 |
| 损耗 | 超合同损耗挂账 | 借：其他应收款-供应商 / 贷：委外物资 |

### 业财集成（参考 Metasfresh EventBus 模式）

```
委外订单审核 → 发布事件 → 财务域监听 → 生成委外物资凭证
     │                         │
     ▼                         ▼
posted=false → 异步过账 → posted=true
```

## 发料方式

### 方式对比

| 方式 | 说明 | 适用场景 | 库存处理 |
|------|------|----------|----------|
| 供应商库存 | 发料到"供应商库存"虚拟仓库 | 长期合作供应商 | 库存减少，挂"委外物资"科目 |
| 在途库 | 发料到"在途库"虚拟仓库 | 一次性委外 | 库存减少，挂"在途物资"科目 |
| 直接消耗 | 发料直接消耗，不挂库存 | 小额/低值物料 | 库存减少，直接进成本 |

### 发料控制

| 控制点 | 规则 | 处理 |
|--------|------|------|
| 超发控制 | 发料数量 ≤ BOM标准数量×(1+损耗率) | 超发需审批 |
| 批次控制 | 启用批次管理的物料必须指定批次 | 强制批次 |
| 序列号控制 | 启用序列号管理的物料必须记录序列号 | 强制序列号 |

- 供应商只能查看分配给自己的委外订单
- 通过 `subcontractor_id` 字段实现数据隔离
- 参考 Odoo 的 Record Rules 机制

## 质量管控

### 来料检验

| 检验时机 | 检验内容 | 处理 |
|----------|----------|------|
| 收货入库时 | 外观、尺寸、功能 | 不合格退回供应商 |
| 批次抽检 | 按AQL标准抽检 | 不合格批次隔离 |

### 过程监控

| 监控点 | 监控内容 | 告警 |
|--------|----------|------|
| 发料超期 | 发料后超过N天未收货 | 预警通知 |
| 生产超期 | 供应商确认生产后超过N天未完工 | 预警通知 |
| 质量异常 | 同一供应商连续N次不合格 | 供应商评级下降 |

## 关键业务规则

1. **BOM 展开**：委外订单只展开外协工序的BOM子件。
2. **发料控制**：发料数量不能超过BOM标准数量×(1+损耗率)。
3. **收货校验**：收货数量不能超过发料数量（扣除损耗）。
4. **成本归集**：委外成品成本 = 材料成本 + 加工费。
5. **发票匹配**：委外发票必须与收货单匹配（数量、金额）。
6. **损耗处理**：超合同损耗由供应商承担，挂"其他应收款-供应商"。
7. **退货处理**：不合格品退回供应商，生成红字出库单。
8. **供应商评级**：根据交期、质量、成本自动计算供应商评级。
