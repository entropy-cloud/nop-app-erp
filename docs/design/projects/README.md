# 项目管理域（projects）

## 目的

说明项目管理域的业务语义、项目生命周期、任务与工时、项目作为辅助核算维度的业财打通。项目域用于跟踪项目进度、归集项目成本、管理任务与工时。

## 边界

- 本域负责：项目、任务、工时记录、项目模板、项目类型、活动类型与成本、项目成员。
- 本域不负责：科目表/科目主数据（master-data）；项目相关凭证生成（finance 域，本域提供工时成本数据）。
- 持久化字段、字典、状态码以 `model/app-erp-projects.orm.xml` 为准。

## 核心业务对象

| 对象 | 业务含义 |
|------|----------|
| 项目（Project） | 一个业务项目的总记录：编码、名称、类型、客户、起止日期、预算、状态 |
| 任务（Task） | 项目下的任务分解：标题、负责人、计划/实际起止、工时、依赖关系、状态 |
| 工时记录（Timesheet） | 成员投入项目的时间记录：日期、项目、任务、活动类型、时长、成本 |
| 项目类型（ProjectType） | 项目分类（研发/交付/内部等），绑定默认科目映射 |
| 活动类型（ActivityType） | 工时活动分类（开发/测试/实施等），用于成本归集 |
| 项目成员（ProjectUser） | 项目参与人员及其角色 |

### 项目与辅助核算

项目作为财务凭证的**辅助核算维度**：所有与项目相关的费用、采购、收款可在凭证分录行标注项目编码，支持按项目出成本报表与利润分析。项目本身不是独立会计科目，而是凭证行的业务维度。

### 工时计入成本

工时记录（Timesheet）按成员的标准成本率计算人工成本。工时提交后可触发财务域生成项目成本凭证（借项目成本/贷应付职工薪酬）。

## 状态机

项目有独立状态机（草稿→进行中→暂停/已完成/已取消），任务有独立状态机（待开始→进行中→已完成/阻塞）。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 项目成本归集 | finance | 项目作为凭证辅助核算维度；工时提交触发项目成本凭证 |
| 项目相关采购 | purchase | 采购单据可标注项目编码，费用归集到项目 |
| 项目相关销售 | sales | 销售单据可标注项目编码，收入归集到项目 |
| 项目成员 | master-data/nop-auth | 成员引用用户主数据 |
| 项目预算控制 | finance | 项目预算与实际发生额对比，超预算提示 |

## 关键业务规则

1. **项目编码唯一**：全局唯一。
2. **任务依赖**：任务可有前置依赖（前置未完成则后继不可开始）。
3. **工时成本计算**：工时 × 成员标准成本率 = 人工成本；不同活动类型可配不同成本率。
4. **项目辅助核算**：凭证分录行的 `projectId` 字段表达项目维度，支持按项目汇总。
5. **预算控制**：项目预算与实际归集成本对比，超预算可配置为警告或拦截。
6. **项目关闭影响**：已关闭/已取消的项目不可再被新单据引用。
