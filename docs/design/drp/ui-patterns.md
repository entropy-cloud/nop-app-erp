# 分销需求计划域页面设计要点

> 本文档定义分销需求计划（DRP）域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-drp.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`net-requirement-calculation.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **净需求计算可视化**：净需求计算过程的每一步（当前库存→已分配→在单量→预测需求→安全库存→净需求→建议补货量）在页面中逐列展示，支持用户追溯每个数字的来源。
2. **建议与批准分离**：DRP 引擎计算产出为"建议补货量"（suggestedQty），用户可调整后形成"批准补货量"（approvedQty），批准后才生成下游单据。建议与审批之间有明确的人工判断环节。
3. **补货类型自动决策**：系统根据物料×仓库的补货参数自动决策补货方式——仓间调拨（有首选调出仓库且有可用库存）or 向供应商采购（无内部可用库存），在明细行标记 replenishmentType。
4. **参数驱动算法**：补货策略（Min-Max/定期/按需）和参数（安全库存/提前期/订货倍数）配置在 ErpDrpParameter 中，DRP 计算引擎按参数运行，用户不修改算法本身。
5. **计划执行闭环**：补货建议批准后，系统自动生成采购订单或调拨单，执行后回写 DRP 行的 orderBillCode，形成计划→执行→完成的闭环。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 | 对应实体 |
|------|------|----------|--------|----------|
| DRP计划列表 | 列表页 | 计划员 | ★☆☆ | ErpDrpPlan |
| DRP计划详情 | 表单+子表 | 计划员 | ★★☆ | ErpDrpPlan + ErpDrpLine |
| **净需求计算报表** | **报表页** | **计划员** | **★★★** | ErpDrpLine（聚合） |
| 补货建议审批 | 操作列表页 | 采购经理 | ★★☆ | ErpDrpLine |
| 仓库补货参数 | 列表页+编辑 | 计划员 | ★☆☆ | ErpDrpParameter |

## 通用页面模式

### 列表页结构

```
┌────────────────────────────────────────────────────┐
│ [新建] [运行DRP计算] [导出]                         │
│ ┌─ 筛选区 ──────────────────────────────────────┐ │
│ │ 计划编号: [___]  期间: [从] ~ [至]  状态: [下拉]│ │
│ │ 物料: [选择]  仓库: [选择]                     │ │
│ │ [查询] [重置]                                  │ │
│ └───────────────────────────────────────────────┘ │
├────────────────────────────────────────────────────┤
│ 结果表格: (支持列排序/自定义列)                     │
│ ┌────────────────────────────────────────────────┐ │
│ │ # │ 计划号  │ 名称    │ 期间    │ 总补货 | 状态 │ │
│ │ 1 │ DRP-001 │ 6月计划 │ 06-01~30│ 5,000  │ 已计算│ │
│ │   │                     [查看] [操作▾]       │ │
│ └────────────────────────────────────────────────┘ │
│ 分页: [<] 1 2 3 ... [>]  每页: 20                  │
└────────────────────────────────────────────────────┘
```

**要点**：
- 状态标签颜色：DRAFT(草稿)=灰色、COMPUTED(已计算)=蓝色、APPROVED(已批准)=绿色、EXECUTED(已执行)=深绿
- "运行DRP计算"触发净需求计算引擎，计算完成后状态变为 COMPUTED
- 运行人和运行时间显示在头区和列表列

## 各页面设计要点

### 净需求计算报表（核心复杂页面）

**页面入口**：DRP管理 → 净需求计算报表

**布局结构**：
```
┌──────────────────────────────────────────────────────────────────────┐
│ [筛选栏] 物料: [选择]  仓库: [选择]  期间: [____]~[____]  [查询]   │
│ [运行DRP] [导出Excel]                                                 │
├──────────────────────────────────────────────────────────────────────┤
│ 物料分组 (可折叠)                                                     │
│ ┌──────────────────────────────────────────────────────────────────┐ │
│ │ 物料: A-001 轴承A      安全库存: 200    提前期: 3天               │ │
│ │ ┌───────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬─────┐│ │
│ │ │ 仓库  │ 当前 │ 已分 │ 在单 │ 预测 │ 净需 │ 建议 │ 补货 │ 状态││ │
│ │ │       │ 库存 │ 配量 │ 量   │ 需求 │ 求   │ 补货 │ 类型 │     ││ │
│ │ ├───────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼─────┤│ │
│ │ │ WH01  │ 500  │ 50   │ 100  │ 800  │ 450  │ 500  │ 采购 │ 已批准│ │
│ │ ├───────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼─────┤│ │
│ │ │ WH02  │ 50   │ 20   │ 0    │ 200  │ 170  │ 200  │ 调拨 │ 已批准│ │
│ │ │       │      │      │      │      │      │      │(从WH01)│     ││
│ │ └───────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴─────┘│ │
│ │ 汇总: 轴承A 净需求=620 建议补货=700                                 │ │
│ └──────────────────────────────────────────────────────────────────┘ │
│ ┌──────────────────────────────────────────────────────────────────┐ │
│ │ 物料: A-002 轴承B      安全库存: 100    提前期: 5天               │ │
│ │ ...                                                              │ │
│ └──────────────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────────────┤
│ 底部汇总: 总净需求: 2,500  总建议: 3,000  预计生成订单: 3笔        │
└──────────────────────────────────────────────────────────────────────┘
```

**关键逻辑列**（每列的来源与计算方式）：

| 列 | 来源 | 计算方式 |
|----|------|----------|
| 当前库存 | 库存余额表 | 截止查询时点的实物库存 |
| 已分配量 | 未发货的销售订单/预留单 | 已分配未发出的数量 |
| 在单量 | 采购在途/调拨在途 | 已下单未入库数量 |
| 预测需求 | 预测/销售订单/MPS | 取 max(预测, 已接单) |
| 安全库存 | ErpDrpParameter.safetyStock | 配置值 |
| **净需求** | **= max(0, 安全库存 + 预测需求 - 当前库存 - 在单量 + 已分配量)** | 自动计算 |
| 建议补货量 | 按补货方法计算 | Min-Max/定期/按需算法 |
| 补货类型 | 自动决策 | 有可用调出仓 → 调拨 / 否则 → 采购 |

**业务规则**：
- 净需求为0或负值时，建议补货量为0
- 建议补货量自动按订货倍数（orderMultiple）向上取整
- 补货类型决策逻辑：检查首选调出仓库（ErpDrpParameter.preferredSourceWarehouseId）的可用库存是否 ≥ 净需求
- 预测需求周期性更新（如每月滚动），运行DRP时取最新版

### 补货建议审批

**页面入口**：DRP管理 → 补货建议审批

**布局**：
```
┌───────────────────────────────────────────────────────────────┐
│ [筛选] 状态: [建议 ▼]  物料: [选择]  仓库: [选择]  [查询]    │
│ [批量批准] [批量驳回] [全部调整为...]                         │
├───────────────────────────────────────────────────────────────┤
│ 建议列表 (可编辑表格)                                          │
│ ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬─────────┐ │
│ │ 选择 │ 物料 │ 目标 │ 净需 │ 建议 │ 批准 │ 补货 │ 备注    │ │
│ │      │      │ 仓库 │ 求   │ 补货 │ 补货 │ 类型 │         │ │
│ ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼─────────┤ │
│ │ ☑    │ 轴承A│ WH01 │ 450  │ 500  │ 500  │ 采购 │ 急需    │ │
│ │ ☐    │ 轴承B│ WH01 │ 170  │ 200  │ 200  │ 调拨 │ 慢周转  │ │
│ │ ☑    │ 轴承A│ WH02 │ 170  │ 200  │ 150  │ 调拨 │ 减量    │ │
│ └──────┴──────┴──────┴──────┴──────┴──────┴──────┴─────────┘ │
└───────────────────────────────────────────────────────────────┘
```

**要点**：
- 建议补货量列可编辑，用户可直接修改数字
- 批准补货量默认 = 建议补货量，允许用户调整
- "批量批准"将勾选行的建议补货量→批准补货量
- 审批完成后，选中"生成单据"按钮触发下游单据创建
- 采购经理在此页面行使人工判断，可减少/增加/驳回补货建议

**后续操作**：
- 批准且生成单据后，DRPLine 状态变为 ORDERED
- 生成的采购订单/调拨单单号回写到 orderBillCode 字段
- DRPLine 与下游单据通过 orderBillType/orderBillCode 弱指针关联

### 仓库补货参数

**页面入口**：DRP管理 → 仓库补货参数

**布局**：标准列表页 + 编辑弹窗

**关键字段**：
- 仓库（必填）、物料（必填）
- 安全库存：低于此值触发补货建议
- 补货提前期（天）：影响补货建议的到货时间计算
- 订货倍数：建议补货量向上取整到此倍数
- 首选调出仓库：仓间调拨时的默认仓库
- 首选供应商：采购补货时的默认供应商
- 补货方法：最小-最大 / 定期 / 按需
- 最低/最高库存（Min-Max模式）：库存低于最低时补到最高
- 审视周期天数（定期模式）：每隔 N 天审视一次补货

**业务规则**：
- (仓库+物料) 维度唯一，同一物料在不同仓库可配置不同参数
- 未配置参数的 (仓库+物料) 组合不参与 DRP 计算
- 参数变更后，建议重新运行 DRP 计算

## 跨页面导航流

### 正向DRP流程导航

```
仓库补货参数 ──[配置]──→ 参数生效
      ↓
DRP计划列表 ──[新建]──→ DRP计划编辑
      ↓ [运行DRP计算]
      ↓ (净需求计算引擎运行)
净需求计算报表 ←──[查看结果]── DRP计划详情（明细行列表）
      ↓ [调整建议量]
补货建议审批 ←──[点击审批]── DRP计划详情
      ↓ [批准] + [生成单据]
      ↓ (生成采购订单/调拨单)
采购订单列表 ←──[采购]──→ DRP明细行(orderBillCode回写)
调拨单列表  ←──[调拨]──→ DRP明细行(orderBillCode回写)
      ↓ (单据执行完成)
      → DRPLine 状态变为 ORDERED → EXECUTED
      → DRP计划头状态变为 EXECUTED
```

### 逆向导航（追溯）

```
DRP计划详情
  ├── [明细行] → DRPLine → [订货倍数调整] → 补货参数
  ├── [明细行] → orderBillCode → 采购订单详情 / 调拨单详情
  ├── [库存计算] → 当前库存(联动库存余额) → 库存流水
  └── [预测需求] → 销售预测/销售订单(联动来源)
```

## 调研参考汇总

| 设计点 | 参考来源 | 应用方式 |
|--------|----------|----------|
| Material Request 建议→审批→执行 | ERPNext#Material Request | DRPLine 状态机 SUGGESTED→APPROVED→ORDERED |
| 净需求计算链逐列展示 | 通用 DRP 范式 | 7 列公式可视化：当前库存→已分配→在单→预测→安全→净需求→建议 |
| 补货参数配置 (warehouse+material) | ERPNext#Item Supplier/Buying 参数 | ErpDrpParameter 维度 (warehouse, material) |
| Min-Max / 定期 / 按需策略 | 通用库存补货理论 | 补货方法枚举 + 参数配置化 |
| 补货类型自动决策 | ERPNext#Material Request 类型 | 仓间调拨 vs 采购的自动判定规则 |
| 弱指针关联下游单据 | 设计决策 | orderBillType/orderBillCode 关联采购/调拨单 |
