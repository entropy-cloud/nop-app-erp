# DRP 安全库存优化设计

## 目的

设计统计安全库存计算功能：基于需求历史数据，按统计方法（标准差 × 服务水平）自动计算安全库存和再订货点。为 DRP 补货参数（ErpDrpParameter.safetyStock）提供推荐值，支持参数配置和人工覆盖。

## 设计依据

> 参考 **ERPNext Stock Replenishment**：安全库存计算基于历史消耗，支持按天/月配置分析周期。
>
> 参考 **DDMRP（Demand Driven MRP）** 方法论：基于需求变异性和提前期配置动态缓冲。
>
> 来源 `docs/analysis/erp-survey/2026-06-30-0000-axelor-open-suite.md` §供应链计划。

## 统计安全库存公式

### 核心公式

```
安全库存 = Z × σ_d × √L

其中：
  Z  = 服务水平对应的标准正态分布 Z 值
  σ_d = 需求标准差（基于历史期间需求）
  L  = 提前期（天）
```

| 服务水平 | Z 值 | 含义 |
|----------|------|------|
| 95% | 1.645 | 每 20 个补货周期中约 1 次缺货 |
| 97.5% | 1.960 | 每 40 个补货周期中约 1 次缺货 |
| 99% | 2.326 | 每 100 个补货周期中约 1 次缺货 |
| 99.5% | 2.576 | 每 200 个补货周期中约 1 次缺货 |

### 再订货点

```
再订货点 (ROP) = 安全库存 + 提前期需求
提前期需求 = 平均日需求 × 提前期（天）
```

### 可选计算方法

| 方法 | 公式 | 适用场景 |
|------|------|----------|
| STATISTICAL（标准差法） | `Z × σ_d × √L` | 需求较稳定、符合正态分布 |
| SIMPLE（简易法） | `平均日需求 × 提前期 × 安全系数` | 数据不足时，手动设定系数 |
| DDMRP（需求驱动法） | `平均日用量 × (提前期 + 需求波动天数 + 订单周期)` | 高变异、短提前期场景 |

## 需求历史分析

### 数据来源

安全库存计算基于以下历史数据：

| 数据源 | 周期粒度 | 说明 |
|--------|---------|------|
| 库存出库记录（StockMove OUT） | 天/周/月 | 实际发货量 |
| 销售订单行（已审核） | 月 | 客户需求 |
| DRP 历史预测 | 月 | 历史预测对比 |

### 分析期配置

```
用户配置：historyMonths = 6（最近 6 个月）
系统分析：
  ┌─────────────────────────────────────────────────┐
  │ 物料: MTL-001, 仓库: WH-01                      │
  │ 分析期间: 2026-01 ~ 2026-06 (6 个月)            │
  ├─────────────────────────────────────────────────┤
  │ 月份       │ 月需求量                             │
  │────────────┼────────────────────────────────────┤
  │ 2026-01    │ 1200                                │
  │ 2026-02    │ 980                                 │
  │ 2026-03    │ 1500                                │
  │ 2026-04    │ 1100                                │
  │ 2026-05    │ 1300                                │
  │ 2026-06    │ 1400                                │
  ├─────────────────────────────────────────────────┤
  │ 平均月需求: 1,246.67                             │
  │ 月需求标准差: 183.58                             │
  │ 提前期: 15 天 (0.5 月)                           │
  │ 服务水平: 95% (Z=1.645)                          │
  │                                                  │
  │ 安全库存 = 1.645 × 183.58 × √0.5                │
  │          = 1.645 × 183.58 × 0.707                │
  │          ≈ 213                                   │
  │                                                  │
  │ ROP = 213 + (1,246.67 × 0.5)                     │
  │     = 213 + 623                                  │
  │     = 836                                        │
  └─────────────────────────────────────────────────┘
```

### 数据清洗规则

| 场景 | 处理 |
|------|------|
| 历史月份中有 0 需求（停产/断供） | 保留 0 值在标准差计算中（反映真实变异），或排除（可配置） |
| 历史月份不足配置月数 | 以实际存在数据计算，降级使用 SIMPLE 方法提醒 |
| 异常 spike（促销/一次性大单） | 可选配置是否排除前 N% 的异常数据 |
| 新产品无历史数据 | 降级使用 SIMPLE 方法或手动输入安全库存 |

## 服务水平选择

### 服务水平权衡

| 服务水平 | 安全库存成本 | 缺货风险 | 适用品类 |
|----------|-------------|----------|----------|
| 95% | 低 | 5%（约18天/年） | 常规品、低成本、多替代物料 |
| 97.5% | 中 | 2.5%（约9天/年） | 关键物料、长提前期 |
| 99% | 高 | 1%（约3.7天/年） | A 类关键物料、独占/定制件 |
| 99.5% | 最高 | 0.5%（约1.8天/年） | 战略物料、安全/法规影响 |

### 按物料分类推荐

| 物料 ABC 分类 | 建议服务水平 | 说明 |
|-------------|-------------|------|
| A 类（高价值/高用量） | 99%~99.5% | 缺货影响大，需高服务水平 |
| B 类（中价值/中用量） | 97.5% | 平衡库存成本与服务水平 |
| C 类（低价值/低用量） | 95% | 安全库存成本低，可适当降低 |

## 与 DRP 集成

### 集成流程

```
安全库存优化模块                 DRP 模块
    │                             │
    │  计算建议安全库存            │
    │  → ErpInvDrpSafetyStockCalc │
    │      .calculatedSafetyStock │
    │           │                 │
    │           ▼                 │
    │  人工审查（可选覆盖）        │
    │  → overrideSafetyStock      │
    │           │                 │
    │           ▼                 │
    │  回写 DRP 参数               │
    │  → ErpDrpParameter          │
    │      .safetyStock = 结果    │
    │           │                 │
    │           ▼                 │
    │           │  下次 DRP 运行   │
    │           │  → 使用新安全库存│
    │           │  计算净需求      │
    │           │                 │
    └───────────│─────────────────┘
```

### 更新时机

| 触发方式 | 说明 |
|----------|------|
| 手动触发 | 计划员进入安全库存优化界面，选择物料+仓库后点击"计算" |
| 定时任务 | 按配置周期（如每月初）全量或增量计算 |
| DRP 运行前 | DRP 计算前自动触发一次安全库存刷新 |

## 业务规则

1. **计算结果仅是建议**：`calculatedSafetyStock` 是系统推荐值，人工可通过 `overrideSafetyStock` 覆盖。DRP 运行时优先取 `overrideSafetyStock`，其次取 `calculatedSafetyStock`，最后取 `ErpDrpParameter.safetyStock` 原值。
2. **覆盖审计**：人工覆盖需记录 `overwrittenBy`，用于后续追溯。
3. **计算再触发**：当 historyMonths、serviceLevel、leadTimeDays 任一参数变化时，`calculatedSafetyStock` 标记为 STALE，提示用户重新计算。
4. **批量计算**：支持按物料分类、仓库范围批量触发安全库存计算。
5. **计算结果对比**：新计算结果与当前 `ErpDrpParameter.safetyStock` 进行比较，差异超过 20%（可配置）时告警提示。

## 反模式警示

- ⛔ **安全库存与再订货点混为一谈**——安全库存是"缓冲量"，再订货点是"触发补货的库存水位"。两者都需计算，但语义不同。
- ⛔ **不做历史清洗直接使用原始数据**——促销 spike、停产期、数据缺失都会扭曲标准差，必须进行数据清洗。
- ⛔ **所有物料使用相同服务水平**——A/B/C 分类物料的缺货影响不同，应按分类差异化配置服务水平。
- ⛔ **计算结果直接覆写 DRP 参数**——必须经过人工审查确认后才能回写 ErpDrpParameter。
