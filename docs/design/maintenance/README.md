# 设备维护域（maintenance）

## 目的

说明设备维护域的业务语义、维护计划、维护访问、维护请求、停机记录。维护域负责资产的**实物维护**：预防性维护计划、维修执行、设备状态管理。与 assets 域（资产价值管理）分工协作。

## 边界

- 本域负责：设备（实物维护视角）、维护计划（周期性预防维护）、维护访问（实际执行）、维护请求（报修）、维护团队与任务、停机记录。
- 本域不负责：资产卡片与折旧（assets 域）；备件物料主数据（master-data）；维修消耗备件的库存出库（inventory 域，本域调用）。
- 持久化字段、字典、状态码以 `model/app-erp-maintenance.orm.xml` 为准。

## 与 assets 域的分工

| 关注点 | assets 域 | maintenance 域 |
|--------|-----------|----------------|
| 资产价值 | 卡片、原值、折旧、资本化、处置 | — |
| 资产实物 | — | 设备状态、维护计划、维修执行 |
| 触发凭证 | 折旧/处置凭证 | 维修成本（通常不直接凭证，通过备件消耗与工时） |

一项资产在 assets 域有卡片（价值管理），在 maintenance 域有设备记录（实物维护）。两者通过资产编码关联，但各自独立维护。

## 核心业务对象

| 对象 | 业务含义 |
|------|----------|
| 设备（Equipment） | 需维护的设备/资产实物记录：编码、名称、资产卡片引用、状态、所在位置 |
| 维护计划（MaintenanceSchedule） | 周期性预防维护计划：设备、维护项目、周期、下次执行日期 |
| 维护访问（MaintenanceVisit） | 一次实际维护执行的记录：计划/实际日期、执行人、维护内容、消耗备件、状态 |
| 维护请求（MaintenanceRequest） | 报修请求：设备、故障描述、优先级、状态 |
| 维护团队（MaintenanceTeam） | 维护人员与分工 |
| 维护任务（MaintenanceTask） | 标准维护作业项目模板 |
| 停机记录（DowntimeEntry） | 设备停机记录：起止时间、原因、影响（关联工作中心） |

### 维护计划与维护访问的关系

- **维护计划**是预防性维护的周期性模板（如"每月润滑""每季校准"）。
- 系统按计划周期自动生成**维护访问**（待执行），或由维护请求触发**维护访问**（响应性维护）。
- 维护访问执行时记录实际维护内容、消耗备件、工时。

### 设备状态与停机

设备状态（运行中/闲置/维护中/故障停机/已停用）由维护活动驱动：
- 开始维护 → 设备进入"维护中"。
- 故障报修 → 设备进入"故障停机"，记录停机时间。
- 维护完成 → 设备恢复"运行中"或"闲置"。

停机记录关联制造域的工作中心（若设备是生产设备），影响排产。

## 状态机

维护访问有独立状态机（草稿→已排程→执行中→已完成/已取消），维护请求有独立状态机（待受理→已受理→维修中→已完成/已拒绝/已取消）。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 维修消耗备件 | inventory | 维护访问消耗备件 → `IErpInvStockMoveBiz` 生成出库移动单 |
| 设备停机影响排产 | manufacturing | 停机记录关联工作中心，通知制造域调整排产 |
| 设备资产价值 | assets | 通过资产编码关联资产卡片（查询用，不改价值） |
| 备件/人员主数据 | master-data | 引用备件物料与维护人员 |

## 关键业务规则

1. **一物一设备**：每台需维护的设备一条记录，编码唯一。
2. **计划周期触发**：维护计划按周期（按日/按运行时长/按次数）自动生成维护访问。
3. **报修触发响应**：维护请求受理后生成维护访问执行维修。
4. **备件消耗入账**：维修消耗的备件通过库存移动单出库，计入维修成本。
5. **停机时间统计**：停机记录的起止时间用于计算设备 OEE（综合效率）。
6. **设备状态联动**：维护活动驱动设备状态变化。
