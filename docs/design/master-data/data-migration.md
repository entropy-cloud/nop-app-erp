# 数据迁移策略

## 目的

定义 nop-app-erp 从旧系统迁移数据的策略，确保数据完整性和可追溯性。

## 三字段设计

每条迁移记录携带三个字段：

| 字段 | 含义 |
|------|------|
| `importBatchId` | 迁移批次号 |
| `importSource` | 数据来源标识（如 "legacy-erp"、"manual-entry"） |
| `importStatus` | 导入状态（DRAFT/VALIDATED/IMPORTED/REJECTED） |

## 迁移状态

| 状态 | 含义 |
|------|------|
| DRAFT | 数据已上传，未校验 |
| VALIDATING | 校验中 |
| VALIDATED | 校验通过，等待确认 |
| IMPORTED | 已导入 |
| REJECTED | 校验失败 |

## 迁移范围

| 数据类型 | 迁移方式 | 说明 |
|----------|----------|------|
| 主数据 | 全量导入 | 物料/客户/供应商/仓库/科目 |
| 期初余额 | 一次性导入 | 总账/AR/AP/存货/资产 |
| 历史单据 | 选择性导入 | 未结清的 AR/AP 发票 |
| 配置数据 | 手工配置 | 科目映射、审批流、价格表 |

## 约束

- 主数据必须先于业务单据导入
- 期初余额必须在第一个 OPEN 期间开启前导入
- 历史单据导入后 `posted=true`（直接生效）
- 迁移数据不可修改，纠错只能通过常规业务调整
