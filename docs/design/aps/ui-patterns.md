# APS/排产域页面设计要点

> 本文档定义APS/排产域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-aps.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`scheduling-algorithm.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **工序级排产**：所有排产操作基于 OperationOrder（工序工单），非主工单（WorkOrder）级。同一主工单下多个工序可按不同工作中心/设备排程，实现有限产能约束调度。
2. **甘特图为核心交互**：排产结果以甘特图可视化呈现，支持拖拽调整工序起止时间、查看工作中心负荷、识别瓶颈资源。
3. **模拟与发布分离**：排产方案（Schedule）允许多版本模拟试算（DRAFT），确认后发布（PUBLISHED）为执行方案。模拟版本不产生实际业务影响。
4. **有限产能约束**：排产计算必须考虑设备日历（维护停机、班次）、刀具寿命、人员技能等约束，非无限产能前向填充。
5. **进度反馈闭环**：工序执行完成后反馈实际开工/完工时间，与计划时间对比，驱动重排或偏差分析。

**要点**：
- 状态列用标签展示：DRAFT(草稿)=灰色、PLANNED(已排程)=蓝色、IN_PROGRESS(执行中)=绿色、FINISHED(已完成)=深绿、CANCELLED(已取消)=灰色删除线

- 执行中的工序显示"实际开工时间"列，支持报工入口
- "运行排产"按钮在排产方案详情页出现，触发排产计算引擎

## 各页面设计要点

### 工序工单

**页面入口**：排产管理 → 工序工单

**头区特殊字段**：
- 所属主工单（必填）：链接到生产域 WorkOrder，显示主工单编号和状态
- 工作中心/设备（必填）：从设备主数据选择，选中的工作中心影响可用时间日历
- 优先级：数值越小优先级越高，影响排产引擎排序顺序
- 是否外协：标记为外协的工序跳过内部排产，进入外协管理流程

**子表（工序参数）**：
```
┌────────────────────────────────────────────────┐
│ ┌──────┬──────┬──────┬──────┬──────┬─────────┐ │
│ │参数名│参数值│单位  │上限  │下限  │说明     │ │
│ │ 温度 │ 800  │ ℃   │ 900  │ 700  │热处理   │ │
│ │ 压力 │ 10   │ MPa  │ 12   │ 8    │压制     │ │
│ └──────┴──────┴──────┴──────┴──────┴─────────┘ │
└────────────────────────────────────────────────┘
```

**状态联动**：
- 工序状态变更驱动主工单状态汇总：当同一主工单下所有工序完成后，主工单标记为"工序完成"
- 工序开始执行时，自动锁定工作中心该时段（不允许其他工序排入）

### 排产方案

**页面入口**：排产管理 → 排产方案

**头区特殊字段**：
- 排产模式：前向排产（从最早可用时间正向排）vs 后向排产（从交期倒推）
- 展望期：定义排产计算的时间窗口范围
- 排产日期：排产计算的基准日期

**排产结果子表**（排产方案发布后生成）：
```
┌────────────────────────────────────────────────────────────┐
│ ┌────┬────────┬────────┬────────┬────────┬──────┬────────┐│
│ │序号│工序编号 │工作中心│计划开工 │计划完工 │ 时长 │ 状态  ││
│ │ 1  │OP-001  │WC-01   │06-20 08│06-20 10│120min│ 已排程 ││
│ │ 2  │OP-002  │WC-01   │06-20 10│06-20 12│120min│ 已排程 ││
│ │ 3  │OP-003  │WC-02   │06-20 08│06-20 11│180min│ 已排程 ││
│ └────┴────────┴────────┴────────┴────────┴──────┴────────┘│
└────────────────────────────────────────────────────────────┘
```

**业务规则**：
- DRAFT 方案可反复运行排产引擎覆盖结果
- PUBLISHED 方案冻结排产结果，工序开始执行后不可回退到 DRAFT
- 同一时间一个工作中心只能发布一个排产方案 [Axelor#OperationOrder 工作中心绑定]

### 排产甘特图（核心复杂页面）

**页面入口**：排产管理 → 排产甘特图

**布局结构**：
```
┌──────────────────────────────────────────────────────────────┐
│ [筛选栏] 工作中心: [▼全选]  日期: [从____]~[____]  [查询]  │
│ [排产方案] [▼当前方案]  [运行排产] [发布方案] [保存布局]     │
├─────────┬────────────────────────────────────────────────────┤
│ 工作中心│ 06-20(周一)         06-21(周二)         06-22(周三)│
│ 列表    │ 08:00  10:00  12:00  08:00  10:00  12:00  08:00   │
├─────────┼────────────────────────────────────────────────────┤
│ 加工中心1│ ▓▓▓OP-001░░░ ▓▓▓▓OP-002░░░░     │ 维护停机      │
│ 加工中心2│ ░░░░ ▓▓▓▓▓▓OP-003░░░░░░░       │ ▓▓▓OP-005     │
│ 加工中心3│ ▓▓▓OP-004▓▓▓                    │ ░░░░          │
│ 热处理炉 │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │ ▓▓▓▓▓▓▓▓▓▓▓▓ │
│ 质检台   │ ░░░░ ▓▓▓▓OP-006 ▓▓▓▓▓          │               │
├─────────┴────────────────────────────────────────────────────┤
│ [图例] ▓=加工 ░=空闲 ▓▓▓▓=紧急(红色) ▓▓▓=正常(蓝色)  ═══=加班│
│ [统计栏] 总工序: 12  延期: 3  产能利用率: 78%  瓶颈: 加工中心1│
└──────────────────────────────────────────────────────────────┘
```

**关键交互模式**：
1. **拖拽调整**：鼠标拖拽甘特条水平移动改变工序开工/完工时间，垂直移动改变分配的工作中心。拖拽时显示对齐辅助线（按班次/小时粒度）
2. **右键菜单**：右键甘特条显示快捷操作——[查看工序详情] [跳转主工单] [标记开工] [标记完工] [锁定] [颜色标记]
3. **颜色编码系统**：
   - 蓝色 = 正常排程（交期内）
   - 黄色 = 接近交期（<2天）
   - 红色 = 已延期
   - 灰色斜纹 = 外协工序
   - 淡灰底色条 = 设备维护时段（不可排产）
4. **缩放控制**：时间轴支持切换天/周/月视图，甘特条粗细自适应
5. **约束叠加**：工作中心维护日历以半透明灰色遮罩叠加在工作中心行上
6. **选择性显示**：支持按优先级/订单号/工单号过滤显示工序条

**数据联动**：
- 甘特图的拖拽操作即时更新 ErpApsOperationOrder 的计划时间，触发排产方案"已修改未保存"标记
- "运行排产"按钮调用排产引擎，引擎结果覆盖甘特图显示
- 双击甘特条跳转到工序工单详情页

**业务规则**：
- 平衡排产 vs 优化目标：默认按"最小化最大延迟"排产，可选"最短加工时间优先""最早交期优先"策略
- 约束冲突提示：当拖拽导致某工作中心超负荷时，页面显示黄色警告条并给出建议调整方案 [Axelor#OperationOrder 约束]

### 排产约束配置

**页面入口**：排产管理 → 排产约束

**布局**：列表页+编辑弹窗，字段包括：
- 工作中心/设备（必填）
- 约束类型：维护停机 / 刀具寿命 / 人员约束 / 特殊事件
- 开始/结束时间
- 约束描述
- 是否周期性（每周一、每月1号等）

**业务规则**：
- 约束生效期内，排产引擎不会将工序排入该时段
- 约束的 blocking_level：INFO 仅提示，WARN 允许排产但告警，ERROR 禁止排产
- 维护停机约束可关联工单（工单结束自动删除约束）

## 跨页面导航流

### 正向排产流程导航

```
工序工单列表 ──[新建]──→ 工序工单编辑
      ↓ (批量选中提交排产)
排产方案列表 ──[新建/复制]──→ 排产方案编辑
      ↓ [运行排产]
      ↓ (排产引擎计算)
排产甘特图 ←──[查看方案]──→ 排产方案详情
      ↓ [手动拖拽调整]
      ↓ [发布方案]
      → 排产方案 PUBLISHED
      ↓ (工序状态变为 PLANNED 已排程)
工序工单列表 → [开始执行] → 工序执行
      ↓ (报工反馈)
      → 工序 FINISHED
```

### 逆向导航（追溯）

```
工序工单详情
  ├── [所属主工单] → 生产工单详情 → BOM → 物料
  ├── [所属排产方案] → 排产方案详情 → 甘特图定位
  ├── [工作中心] → 设备主数据 → 维护日历
  └── [操作工] → 员工信息
```

## 主交易实体 form 布局分组

> 适用范围：APS 域 4 个主交易实体（不含已 1500-1 覆盖的 `ErpApsSchedule`）独立 `view.xml` 的 `<form id="view">` / `<form id="edit">` 分组。
> 决策来源：`docs/plans/2026-07-20-2059-2-f3-p2p3-ext-masterdata-form-layout.md` Phase 0.E。
> 工序工单（OperationOrder）等核心实体按 工作中心/机器 + 计划起止时间 + 优先级/状态 等业务关键字段分组。

### query 表单基线

所有 APS 域主实体的 `<form id="query">` 至少含 5 个查询字段。`code`/`ruleName`/`operationName` 配 `filterOp=like`；`workOrderId`/`machineId`/`operationId`/`orgId`/`status`/`enableAuto`/`isDefault`/`isEnabled` 配 `filterOp=eq`；含日期字段（`businessDate`/`plannedStartDateT`）配 `filterOp=date-between`。
