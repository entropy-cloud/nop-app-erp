# B2B集成域页面设计要点

> 本文档定义B2B集成/EDI域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-b2b.orm.xml` 为准，业务语义与状态机见 `edi-state-machine.md`、`asn-flow.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **EDI信封状态机可视化**：每条EDI事务（ErpB2bEdiDoc）有完整状态机流转——TO_SEND→SENT→ACKNOWLEDGED/ERROR/CANCELLED。页面需以时间线形式清晰展示状态变迁历史，每条状态记录精确到秒。
2. **ASN入站五阶段流程**：ASN从EDI网关接收→解析匹配采购订单→校验物料/数量→通知仓库准备收货→完成入库，五个阶段在ASN详情页以流程条（step bar）可视化呈现。
3. **报文原文可查看**：所有EDI交互的请求报文和响应报文原文（XML/JSON）可查看和下载，便于集成排错和审计追溯。
4. **阻断级别指示**：每条EDI事务的 blockingLevel（INFO/WARN/ERROR）指示当前问题的严重程度。INFO=仅记录，WARN=不影响业务流转但需关注，ERROR=阻塞下游操作。
5. **代码映射集中管理**：内外代码映射（物料/伙伴/计量单位）在统一页面维护，EDI收发自动查映射表转换，用户无需在多个模块重复配置。

## 通用页面模式

### 列表页结构

```
┌────────────────────────────────────────────────────┐
│ [新建] [重试] [导出]                                │
│ ┌─ 筛选区 ──────────────────────────────────────┐ │
│ │ 编码: [___]  格式: [下拉]  状态: [下拉]       │ │
│ │ 方向: [下拉]  时间: [从] ~ [至]              │ │
│ │ [查询] [重置]                                  │ │
│ └───────────────────────────────────────────────┘ │
├────────────────────────────────────────────────────┤
│ 结果表格: (支持列排序/自定义列)                     │
│ ┌────────────────────────────────────────────────┐ │
│ │ # │ 编码    │ 格式    │ 方向│ 关联单号│ 状态    │ │
│ │ 1 │ EDI-001│ UBL发票 │ 出站│ SI-001 │ ● 已发送│ │
│ │   │                     [查看] [操作▾]       │ │
│ └────────────────────────────────────────────────┘ │
│ 分页: [<] 1 2 3 ... [>]  每页: 20                  │
└────────────────────────────────────────────────────┘
```

**要点**：
- 状态列用标签+图标展示：TO_SEND(待发送)=灰色、SENT(已发送)=蓝色、ACKNOWLEDGED(已确认)=绿色、ERROR(错误)=红色、TO_CANCEL(待取消)=橙色、CANCELLED(已取消)=灰色删除线、RECEIVED(已接收)=蓝色、ARCHIVED(已归档)=灰色
- 阻断级别列用醒目标记：INFO=ℹ️、WARN=⚠️、ERROR=🚫
- "重试"按钮对 ERROR/TO_SEND 状态的 EDI 事务重新发送

**关键交互**：
- 状态时间线按时间顺序由近到远或由远到近排列，最新状态高亮
- 每条状态显示精确时间戳、状态码和简要消息
- 报文查看区支持语法高亮（XML/JSON）、格式化、复制、下载
- ERROR 状态行显示错误详情，点击可查看完整错误堆栈 [Odoo#account_edi 错误处理]
- 重试操作：将状态重置为 TO_SEND，清空错误信息，重新提交

## 各页面设计要点

### EDI格式配置

**页面入口**：B2B集成 → EDI格式配置

**布局**：标准列表页+编辑弹窗

**关键字段**：
- 编码（唯一）、格式名称
- EDI标准：UBL / X12 / EDIFACT / 自定义
- 方向：出站 / 入站 / 双向
- 需要 WebService：是否通过网关 HTTP 交互
- 是否启用

**业务规则**：
- 启用状态的格式才能参与EDI收发
- 一种格式可关联多个业务单据类型（如UBL Invoice格式同时用于销售发票和 credit note）

### EDI事务列表/详情

**页面入口**：B2B集成 → EDI事务

**列表页特殊筛选**：
- 状态筛选器支持多选组合（如同时筛选 ERROR+SENT）
- 阻断级别筛选（仅显示 ERROR 级别）
- 时间范围快速选择：今天/最近7天/本月

**详情页状态时间线数据来源**：
- 时间线数据来自 ErpB2bEdiLog 表按 ediDocId 分组
- 每条日志记录方向、请求/响应报文、结果码、结果消息
- 时间线最多保留最近50条日志

### ASN列表/详情

**页面入口**：B2B集成 → ASN管理 / 采购管理 → ASN入站

**ASN详情页布局**：
```
┌──────────────────────────────────────────────────────────────┐
│ ASN编码: ASN-202606-0001  状态: [已入库]  供应商: xxx       │
├──────────────────────────────────────────────────────────────┤
│ 五阶段流程条                                                  │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ ① 已接收 → ② 已匹配 → ③ 已校验 → ④ 待入库 → ⑤ 已入库  │ │
│ │ ●●○○○○○                                                    │ │
│ │ 当前阶段: 已匹配 (2026-06-30 10:35)                       │ │
│ └──────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────┤
│ 基本信息                                                     │
│ ┌───── ASN信息 ────────────┐ ┌─── 物流信息 ─────────────┐ │
│ │ 供应商: xxx               │ │ 发货日期: 06-29          │ │
│ │ 来源EDI: EDI-0001 [查看]  │ │ 预计到货: 06-30          │ │
│ │ 关联PO: PO-202606-0100   │ │ 物流单号: SF-123456789   │ │
│ └────────────────────────────┘ └──────────────────────────┘ │
├──────────────────────────────────────────────────────────────┤
│ ASN明细行                                                     │
│ ┌────┬──────────┬──────────┬────────┬──────┬──────────────┐│
│ │行号│ 物料     │ 供应商   │ 订单   │ 发货 │ 匹配状态     ││
│ │    │          │ 料号     │ 数量   │ 数量 │              ││
│ │ 1  │ 轴承A    │ BRG-001  │ 100    │ 100  │ ✓ 已匹配     ││
│ │ 2  │ 轴承B    │ BRG-002  │ 50     │ 48   │ ⚠ 数量差异   ││
│ └────┴──────────┴──────────┴────────┴──────┴──────────────┘│
├──────────────────────────────────────────────────────────────┤
│ 底栏: 来源EDI追溯 | 关联采购订单 | 入库单链接              │
└──────────────────────────────────────────────────────────────┘
```

**五阶段流程详细说明**：
1. **已接收（RECEIVED）**：EDI 网关接收到 ASN 报文，解析后创建 ErpB2bAsn 记录
2. **已匹配（MATCHED）**：系统自动匹配关联采购订单（通过 relatedBillCode），匹配成功推进到下一步
3. **已校验（VALIDATED）**：校验物料存在性、数量合理性（超量/短量容忍度）、供应商一致性
4. **待入库（PENDING_RECEIVE）**：通知仓库，创建预入库待办，等待仓库实际操作
5. **已入库（RECEIVED_TO_STOCK）**：仓库完成入库操作，生成采购入库单，状态闭环

**明细行匹配状态**：
- ✓ 已匹配：供应商料号成功映射到内部物料，且数量在容差内
- ⚠ 数量差异：发货数量与订单数量差异超出容差
- ✗ 未匹配：供应商料号无对应代码映射
- — 新物料：供应商料号可映射但内部物料无对应主数据

**业务规则**：
- ASN 匹配失败（物料映射失败）不会阻塞整个 ASN，异常行标记后正常行继续处理
- 数量差异在 ±5% 容差内自动通过，超出标记"数量差异"等待人工确认
- ASN 入库后生成采购入库单，ASN 行与入库单行一一对应

### 代码映射

**页面入口**：B2B集成 → 代码映射

**布局**：列表页 + 编辑弹窗

```
┌──────────────────────────────────────────────────────────────┐
│ [新建] [批量导入] [导出]                                     │
│ ┌─ 筛选区 ───────────────────────────────────────────────┐  │
│ │ 映射类型: [物料 ▼]  合作方: [选择]  [查询]            │  │
│ └────────────────────────────────────────────────────────┘  │
├──────────────────────────────────────────────────────────────┤
│ ┌────┬──────────┬──────────┬──────────┬──────────┬────────┐ │
│ │ #  │ 映射类型 │ 内部代码  │ 外部代码  │ 合作方   │ 操作  │ │
│ │ 1  │ 物料     │ MAT-001  │ BRG-001  │ 供应商A  │ [编辑] │ │
│ │ 2  │ 物料     │ MAT-002  │ BRG-002  │ 供应商A  │ [编辑] │ │
│ │ 3  │ 伙伴     │ SUP-001  │ VENDOR01 │ 供应商A  │ [编辑] │ │
│ │ 4  │ 单位     │ PCS      │ EA       │ 供应商A  │ [编辑] │ │
│ └────┴──────────┴──────────┴──────────┴──────────┴────────┘ │
└──────────────────────────────────────────────────────────────┘
```

**要点**：
- 映射按 (合作方, 映射类型) 维度组织，不同合作方可配置不同的内外代码映射
- 支持批量导入（CSV/Excel）方便初始化
- EDI 出站时自动查映射：内部代码→外部代码；入站时反向查：外部代码→内部代码
- 未配置映射时，EDI 事务标记 blockingLevel=WARN 或 ERROR 并记录日志

### EDI日志

**页面入口**：B2B集成 → EDI日志

**布局**：标准日志列表 + 点击行查看详情弹窗

**特殊字段**：
- 方向标签（出站/入站）用不同颜色区分
- 日志时间精确到毫秒
- 报文内容可展开/收起，长报文默认折叠

## 跨页面导航流

### 正向EDI处理流程导航

```
EDI格式配置 ──[配置格式]──→ 格式生效
      ↓ (业务单据提交触发EDI)
EDI事务列表 ←──[自动创建]── 业务单据(销售发票/采购订单等)
      ↓ [发送]
      ↓ (调用EDI网关)
EDI事务详情 ←──[点击查看]── EDI事务列表
      ↓ [网关响应/ACK]
      ↓ (状态机流转)
EDI日志 ←──[查看日志]──  EDI事务详情 (时间线 + 报文)
      ↓ [若为ASN入站]
ASN列表 ←──[自动创建]── EDI事务(ASN报文)
      ↓ [匹配采购订单]
ASN详情(五阶段流程条) ←──[查看]── ASN列表
      ↓ [入库操作]
采购入库单列表 ←──[自动生成]── ASN入站完成
      → 闭环
```

### 逆向导航（追溯）

```
EDI事务详情
  ├── [状态时间线] → EDI日志列表 → [报文原文] → 下载/查看
  ├── [关联单据] → 业务单据详情(销售发票/采购订单/ASN)
  ├── [EDI格式] → EDI格式配置 → 格式参数
  └── [错误信息] → 错误详情 → [重试]

ASN详情
  ├── [来源EDI] → EDI事务详情 → [报文原文]
  ├── [关联采购订单] → 采购订单详情
  ├── [物料映射] → 代码映射（按供应商过滤）
  └── [入库单] → 采购入库单详情
```
