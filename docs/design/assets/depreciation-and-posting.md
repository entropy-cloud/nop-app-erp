# 固定资产域 - 折旧与财务打通

## 目的

详细说明资产域与财务域的业财打通机制，包括折旧凭证生成、资本化入账、资产处置清理的完整流程。

---

## 一、折旧凭证生成机制

### 1.1 折旧业务类型注册

资产域实现 `IErpFinAcctDocProvider` 接口，注册以下业务类型：

| 业务类型 | 触发场景 | 凭证方向 |
|----------|----------|----------|
| DEPRECIATION | 折旧执行 | 借：折旧费用 / 贷：累计折旧 |
| CAPITALIZATION | 资产资本化 | 借：固定资产 / 贷：在建工程或存货 |
| DISPOSAL_SCRAP | 资产报废 | 借：累计折旧 / 借：清理损失 / 贷：固定资产 |
| DISPOSAL_SALE | 资产出售 | 借：累计折旧 / 借：银行存款 / 贷：固定资产 / 贷：清理收益 |
| VALUE_ADJUSTMENT | 资产减值/重估 | 借：资产减值损失（或贷：重估增值） / 贷：资产减值准备（或借：固定资产） |

### 1.2 折旧执行流程

```
折旧执行触发
        │
        ├─► 校验：当前期间是否已结账
        │           │
        │           └─► 已结账 → 拒绝执行，提示反结账
        │
        ├─► 查询待折旧资产（状态 IN_SERVICE）
        │
        ├─► 遍历每资产计算本期折旧额
        │           │
        │           └─► 直线法：(原值 - 残值) / 使用年限 / 12
        │           └─► 双倍余额递减：2 × 账面净值 / 使用年限 / 12
        │           └─► 工作量法：单位工作量折旧 × 本期工作量
        │
        ├─► 更新折旧计划条目状态（PENDING → EXECUTED）
        │
        ├─► 发布过账事件（post-commit）
        │
        └─► 返回成功
                │
                ▼ (异步)
        财务域消费过账事件
                │
                ├─► 读取资产类别科目映射
                │           │
                │           └─► 折旧费用科目（借）
                │           └─► 累计折旧科目（贷）
                │
                ├─► 生成凭证分录行
                │
                ├─► 创建凭证与业财回链
                │
                └─► 设置资产折旧计划条目 posted=true
```

### 1.3 折旧金额计算规则

| 折旧方法 | 计算公式 | 特殊处理 |
|----------|----------|----------|
| 直线法 | (原值 - 残值) / 使用年限 / 12 | 每期相等，最后一期调整到残值 |
| 双倍余额递减法 | 2 × 账面净值 / 使用年限 / 12 | 最后两年改为直线法，确保不低于残值 |
| 工作量法 | (原值 - 残值) / 预计总工作量 × 本期工作量 | 需维护累计工作量 |

### 1.4 残值约束

- 折旧后的账面净值 **不得低于** 残值
- 直线法在计算时已考虑残值，不会出现净值低于残值
- 双倍余额递减法在最后两年改为直线法确保残值约束

## 二、资本化入账流程

### 2.1 资本化场景

| 场景 | 贷方科目 | 说明 |
|------|----------|------|
| 在建工程转固 | 在建工程 | 自建资产完成后转入固定资产 |
| 库存物料转固 | 存货 | 购入物料转为自用固定资产 |
| 直接购置 | 银行存款/应付账款 | 直接购入固定资产 |

### 2.2 资本化流程

```
资本化入账触发
        │
        ├─► 校验：资产卡片状态为 DRAFT
        │
        ├─► 校验：取得日期、原值、使用年限已配置
        │
        ├─► 生成资产编码（自动或手动）
        │
        ├─► 生成折旧计划（每期一条，按折旧方法计算）
        │
        ├─► 更新资产状态为 IN_SERVICE
        │
        ├─► 库存转固场景：调用 IErpInvStockMoveBiz 生成出库移动单
        │
        ├─► 发布资本化过账事件
        │
        └─► 返回成功
                │
                ▼ (异步)
        财务域生成资本化凭证
                │
                ├─► 借：固定资产（资产类别配置的固定资产科目）
                │
                └─► 贷：对应贷方科目（在建工程/存货/银行存款）
```

### 2.3 在建工程转固

```
在建工程转固特点：
        │
        ├─► 需先在财务域创建在建工程科目余额
        │
        ├─► 资本化时结转在建工程余额到固定资产
        │
        └─► 转入金额 = 所有在建工程支出合计
```

## 三、资产处置清理

### 3.1 处置类型

| 类型 | 处置方式 | 账务处理 |
|------|----------|----------|
| 报废 | 资产无残值退出 | 借：累计折旧 / 借：固定资产清理 / 贷：固定资产 → 借：营业外支出 / 贷：固定资产清理 |
| 出售 | 资产转让获利/亏损 | 借：累计折旧 / 借：银行存款 / 借：固定资产清理（亏损） / 贷：固定资产 / 贷：固定资产清理（收益） |

### 3.2 处置流程

```
资产处置触发
        │
        ├─► 校验：资产状态为 IN_SERVICE 或 IDLE
        │
        ├─► 校验：累计折旧与原值一致（无异常）
        │
        ├─► 计算清理损益 = 出售收入（如有） - (原值 - 累计折旧)
        │
        ├─► 更新资产状态为 SCRAPPED 或 SOLD
        │
        ├─► 发布处置过账事件
        │
        └─► 返回成功
                │
                ▼ (异步)
        财务域生成清理凭证
                │
                ├─► 结转原值与累计折旧
                │
                ├─► 记录清理损益
                │
                └─► 更新资产卡片清理标志
```

### 3.3 清理损益计算

```
清理损益 = 处置收入 - 账面净值
        │
        ├─► 处置收入 > 账面净值 → 清理收益（营业外收入）
        │
        └─► 处置收入 < 账面净值 → 清理损失（营业外支出）

账面净值 = 原值 - 累计折旧
```

## 四、资产价值调整

### 4.1 调整类型

| 类型 | 账务处理 | 影响 |
|------|----------|------|
| 减值 | 借：资产减值损失 / 贷：固定资产减值准备 | 减少账面净值，不影响折旧（减值后折旧基数调整） |
| 重估增值 | 借：固定资产 / 贷：资本公积-其他资本公积 | 增加账面净值，可调整折旧基数 |
| 重估减值 | 借：资产减值损失 / 贷：固定资产 | 减少账面净值 |

### 4.2 减值测试

```
定期减值测试（每年末或有迹象时）
        │
        ├─► 评估资产可收回金额
        │
        ├─► 比较可收回金额与账面净值
        │
        ├─► 可收回金额 < 账面净值 → 计提减值准备
        │
        └─► 生成减值凭证
```

## 五、期末批量折旧

### 5.1 触发方式

| 方式 | 触发时机 | 适用场景 |
|------|----------|----------|
| 手动触发 | 用户主动执行 | 临时补提、调整折旧 |
| 期末自动触发 | 财务期末结账时 | 常规月度折旧 |
| 定时任务触发 | 每月固定时间 | 自动执行，无需人工干预 |

### 折旧调度规则

- **折旧起始月**：资本化入账的次月开始计提折旧（当月增加下月提）
- **折旧终止月**：处置/报废的当月停止计提折旧（当月减少当月停）
- **并行度**：按资产类别分组并行执行，每组独立事务
- **幂等性**：同一期间重复执行折旧时，先冲销已执行的折旧凭证再重新生成
- **异常中断**：部分资产折旧失败不影响其他资产，失败资产记录错误等待人工处理

### 5.2 批量折旧流程

```
期末批量折旧
        │
        ├─► 锁定当前会计期间（防止期间变更）
        │
        ├─► 查询所有 IN_SERVICE 状态资产
        │
        ├─► 并行处理（按资产类别分组）
        │           │
        │           └─► 计算本期折旧
        │           └─► 更新折旧计划条目
        │
        ├─► 汇总生成批量折旧凭证（单张凭证多行分录）
        │
        ├─► 标记所有已折旧资产
        │
        └─► 解锁会计期间
```

### 5.3 批量折旧优化

- **并行处理**：按资产类别分组并行计算，提高性能
- **汇总凭证**：同类资产折旧合并到同一张凭证，减少凭证数量
- **错误隔离**：单资产折旧失败不影响其他资产

## 六、科目映射配置

### 6.1 资产类别科目映射

每个资产类别配置以下科目：

| 科目类型 | 用途 | 示例 |
|----------|------|------|
| 固定资产科目 | 资本化时借方 | 1601 固定资产 |
| 累计折旧科目 | 折旧时贷方 | 1602 累计折旧 |
| 折旧费用科目 | 折旧时借方 | 5101 制造费用 / 6602 管理费用 |
| 清理损益科目 | 处置时损益 | 6711 营业外支出 / 6301 营业外收入 |
| 在建工程科目 | 转固时贷方 | 1603 在建工程 |

### 6.2 科目映射优先级

```
资产级别（单个资产配置）
        │
        └─► 资产类别级别（默认）
                │
                └─► 全局默认（兜底）
```

## 七、与财务域的交互协议

### 7.1 事件内容

过账事件包含：业务类型（DEPRECIATION/CAPITALIZATION/DISPOSAL_*/VALUE_ADJUSTMENT）、资产编码、会计期间、金额、明细信息。

### 7.2 错误处理

| 错误类型 | 处理策略 |
|----------|----------|
| 科目映射缺失 | 记录错误，等待人工配置，兜底扫描重试 |
| 期间已结账 | 拒绝过账，提示反结账 |
| 凭证生成失败 | 记录错误，异步重试（最多3次） |
| 并发折旧 | 乐观锁冲突，重试 |

## 八、审计与追溯

### 8.1 折旧追溯链

```
折旧凭证 → 折旧计划条目 → 资产卡片
        │
        └─► 可追溯到每期折旧明细、资产原值、折旧方法、使用年限
```

### 8.2 处置追溯链

```
清理凭证 → 资产处置记录 → 资产卡片
        │
        └─► 可追溯到处置原因、处置方式、清理损益计算过程
```

### 8.3 审计日志

所有操作记录审计日志：
- 折旧执行：执行人、时间、资产编码、折旧金额
- 资本化：执行人、时间、资产编码、入账金额
- 处置：执行人、时间、资产编码、处置方式、损益金额
- 价值调整：执行人、时间、调整类型、调整金额

## 九、关键业务规则总结

1. **期间控制**：已结账期间不允许补提折旧
2. **残值约束**：折旧后净值不低于残值
3. **终态不可恢复**：已处置资产不可重新激活，需通过冲销处理
4. **科目映射**：按资产类别配置，支持资产级别覆盖
5. **批量优化**：期末批量折旧支持并行处理
6. **审计追溯**：所有操作保留完整审计轨迹
