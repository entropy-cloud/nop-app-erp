# 固定资产域（assets）

## 目的

说明固定资产域的业务语义、资产生命周期、折旧与财务域的业财打通。资产域负责资产的**价值管理**：卡片登记、折旧计提、资本化、价值调整、报废处置。资产的**实物维护**（维修、巡检、停机）由 maintenance 域负责。

## 边界

- 本域负责：固定资产卡片、资产类别、折旧计划与执行、资产移动（位置/部门转移）、资产价值调整、资产资本化（在建工程转固）、资产报废/出售。
- 本域不负责：资产的实物维护（maintenance 域）；科目表/科目主数据（master-data）；折旧凭证的科目映射（finance 域，本域触发）。
- 持久化字段、字典、状态码以 `model/app-erp-assets.orm.xml` 为准。
- 跨域协作规则见 `../domain-design-guidelines.md`，全局流程见 `../flow-overview.md`。

## 核心业务对象

| 对象 | 业务含义 |
|------|----------|
| 固定资产卡片（Asset） | 一项固定资产的主记录：编码、名称、类别、取得日期、原值、残值、使用部门、存放位置、状态 |
| 资产类别（AssetCategory） | 资产分类，绑定默认折旧方法、折旧年限、对应会计科目（固定资产科目、累计折旧科目、折旧费用科目） |
| 折旧计划（DepreciationSchedule） | 按资产生成的折旧明细：每期折旧额、累计折旧、账面净值、计划日期 |
| 资产移动（AssetMovement） | 资产位置/部门/使用人的转移记录（不改变资产价值，只改变归属） |
| 资产价值调整（AssetValueAdjustment） | 资产减值/重估增减值的记录，影响账面净值 |
| 资产资本化（AssetCapitalization） | 在建工程/库存物料转为固定资产的过程，产生资产入账凭证 |
| 资产报废/出售（AssetDisposal） | 资产退出使用的处置：报废（清理损失）或出售（清理收益/损失） |

### 资产与折旧的关系

资产卡片登记后，按资产类别配置的折旧方法（直线法/双倍余额递减/工作量法）与折旧年限，系统生成折旧计划（每期一条明细）。折旧执行（手动或期末自动触发）时：
- 更新折旧计划的"已执行"状态。
- 触发财务域生成折旧凭证（借：折旧费用，贷：累计折旧）。

## 状态机

资产卡片有独立状态机（草稿→使用中→闲置→报废/出售）。折旧计划条目有简单状态（待执行/已执行）。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 折旧触发折旧凭证 | finance | 折旧执行 → 财务域生成折旧凭证（借折旧费用/贷累计折旧），按资产类别映射科目 |
| 资本化入账 | finance/inventory | 在建工程/库存转固 → 生成资产入账凭证（借固定资产/贷在建工程或存货） |
| 报废/出售清理 | finance | 处置 → 生成清理凭证（结转原值、累计折旧、清理损益） |
| 资产价值调整 | finance | 减值/重估 → 生成减值/重估凭证 |
| 期末折旧批量 | finance | 财务期末结账触发本期所有资产折旧批量执行 |
| 实物维护查询 | maintenance | maintenance 域引用资产编码查询资产信息（`IErpAstAssetBiz`） |
| 资产类别科目映射 | master-data | 资产类别引用科目编码（固定资产/累计折旧/折旧费用科目） |

跨域调用走 `I*Biz` 接口，不做 ORM 层跨工程 `refEntityName`。

## 关键业务规则

1. **一物一卡**：每项固定资产一张卡片，编码全局唯一。
2. **折旧方法**：支持直线法、双倍余额递减法、工作量法等，按资产类别配置默认值，单个资产可覆盖。
3. **折旧触发凭证**：每次折旧执行生成一张折旧凭证，科目来自资产类别的科目映射。
4. **残值约束**：折旧后的账面净值不低于残值（直线法按残值计算每期折旧）。
5. **资本化时机**：在建工程完工或库存物料转为自用时执行资本化，产生资产卡片 + 入账凭证。
6. **处置清理**：报废/出售时结转原值与累计折旧，差额计入清理损益。
7. **期间控制**：已结账的会计期间不允许补提/冲销折旧。
8. **资产移动不改价值**：位置/部门转移只改归属，不影响原值与折旧。

## 实现落位提示

| 设计含义 | 默认实现落位 |
