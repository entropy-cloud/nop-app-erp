# 质量管理域 - 质检与业务集成

## 目的

详细说明质检单与业务单据的联动机制、质检流程与业务流程的同步、不合格处理与 CAPA 闭环。

---

## 一、质检触发机制

### 1.1 质检触发场景

| 检验类型 | 触发业务域 | 触发单据 | 触发时机 |
|----------|------------|----------|----------|
| 来料检验（INCOMING） | purchase | 采购入库单 | 审核通过后 |
| 制程检验（IN_PROCESS） | manufacturing | 工单报工单 | 工序报工后 |
| 完工检验（FINAL） | manufacturing | 工单 | 完工提交后 |
| 出货检验（OUTGOING） | sales | 销售出库单 | 审核通过后 |

### 1.2 触发流程

```
业务单据审核通过
        │
        ├─► 检查物料是否配置强制质检（inspection_required=true）
        │           │
        │           └─► 否 → 直接继续业务流程
        │
        └─► 是 → 发布质检触发事件
                    │
                    ▼ (异步)
            质量域消费事件
                    │
                    ├─► 根据物料获取质检模板
                    │           │
                    │           └─► 无模板 → 使用全局默认模板
                    │
                    ├─► 创建质检单（PENDING 状态）
                    │
                    ├─► 关联业务单据（reference_type + reference_id）
                    │
                    └─► 产生 TODO 分配质检员
```

### 1.3 强制质检与可选质检

| 类型 | 配置位置 | 行为 |
|------|----------|------|
| 强制质检 | 物料/SKU 主数据 | 质检完成前业务单据阻塞流转 |
| 可选质检 | 物料/SKU 主数据 | 质检不影响业务单据流转 |
| 抽检 | 质量域配置 | 按抽检比例触发质检 |

## 二、质检结果反馈机制

### 2.1 质检结果类型

| 结果 | 状态 | 业务影响 |
|------|------|----------|
| 合格 | ACCEPTED | 业务单据继续流转 |
| 让步接收 | CONDITIONAL | 业务单据继续（附让步记录） |
| 不合格 | REJECTED | 触发退货/返工/报废 |

### 2.2 反馈流程

```
质检结果录入
        │
        ├─► 更新质检单状态
        │
        ├─► 发布质检结果事件
        │
        └─► 业务域消费事件
                    │
                    ├─► ACCEPTED → 业务单据继续流转
                    │
                    ├─► CONDITIONAL → 业务单据继续（记录让步理由）
                    │
                    └─► REJECTED → 触发不合格处理流程
```

### 2.3 不合格处理路径

| 不合格场景 | 处理路径 | 触发单据 |
|------------|----------|----------|
| 来料检验不合格 | 采购退货 | 采购退货单 |
| 出货检验不合格 | 阻止发货 / 销售退货 | 销售退货单 |
| 制程检验不合格 | 返工 | 返工工单 |
| 完工检验不合格 | 返工 / 报废 | 返工工单 / 报废单 |

## 三、让步接收机制

### 3.1 让步接收条件

| 条件 | 说明 |
|------|------|
| 非关键项不合格 | 关键特性合格，次要特性不合格 |
| 降级使用 | 降等级使用（如一等品降为二等品） |
| 挑选使用 | 挑选合格部分使用 |
| 特采 | 客户特批接受 |

### 3.2 让步接收流程

```
质检员发现部分不合格（非关键项）
        │
        ├─► 提出让步接收建议
        │
        ├─► 填写让步理由、使用方式、审批人
        │
        ├─► 提交审批
        │           │
        │           └─► 审批拒绝 → 按不合格处理（退货/返工）
        │
        └─► 审批通过 → 质检单状态转为 CONDITIONAL
                    │
                    └─► 记录让步记录（质检单关联）
```

### 3.3 让步记录内容

让步记录包含：质检单编码、让步理由、使用方式（降级/挑选/特采）、审批人、审批时间。

## 四、NCR 与 CAPA 闭环

### 4.1 NCR 创建

```
质检不合格（REJECTED）
        │
        ├─► 自动创建 NCR（不符合项报告）
        │
        ├─► 填写不合格现象、影响范围、严重程度
        │
        └─► NCR 状态为 OPEN（待处理）
```

### 4.2 CAPA 制定

```
NCR 评审
        │
        ├─► 分析根本原因
        │
        ├─► 制定纠正措施（即时修复）
        │
        ├─► 制定预防措施（防止再发）
        │
        └─► 分配责任人与计划完成时间
```

### 4.3 CAPA 执行与验证

```
CAPA 执行
        │
        ├─► 实施纠正措施
        │
        ├─► 实施预防措施
        │
        ├─► 效果验证（检查问题是否解决、是否预防再发）
        │           │
        │           └─► 验证失败 → 返回重新制定措施
        │
        └─► 验证通过 → NCR 状态转为 RESOLVED（已解决）
```

### 4.4 NCR 状态流转

```
待处理 (OPEN)
  ├─► 评审 → 评审中 (IN_REVIEW)
  │           ├─► CAPA 制定完成 → 执行中 (IN_EXECUTION)
  │           │           ├─► 效果验证通过 → 已解决 (RESOLVED)
  │           │           └─► 验证失败 → 返回评审中
  │           └─► 取消（误开）→ 已取消 (CANCELLED)
  └─► 取消 → 已取消 (CANCELLED)
```

## 五、质检模板管理

### 5.1 质检模板内容

质检模板包含：物料编码（可为空表示全局模板）、检验项目名称、单位、规格下限、规格上限、检验方法、是否关键项。

### 5.2 模板优先级

```
物料级别模板（特定物料）
        │
        └─► 物料类别级别模板（某类物料）
                │
                └─► 全局默认模板（兜底）
```

### 5.3 检验读数判定

```
检验读数录入
        │
        ├─► 获取对应检验项目模板
        │
        ├─► 判定：lowerLimit ≤ 读数 ≤ upperLimit？
        │           │
        │           ├─► 是 → 合格
        │           │
        │           └─► 否 → 不合格
        │                       │
        │                       └─► 关键项不合格 → 整体不合格
        │                       │
        │                       └─► 非关键项不合格 → 可让步接收
```

## 六、质量指标与报表

### 6.1 关键质量指标

| 指标 | 计算方式 | 用途 |
|------|----------|------|
| 合格率 | 合格数 / 检验总数 × 100% | 衡量整体质量水平 |
| 不良率 | 不合格数 / 检验总数 × 100% | 衡量缺陷比例 |
| 让步接收率 | 让步接收数 / 检验总数 × 100% | 衡量质量风险 |
| NCR 闭环率 | 已解决 NCR / 总 NCR × 100% | 衡量问题处理效率 |
| CAPA 按时完成率 | 按时完成 CAPA / 总 CAPA × 100% | 衡量改进措施执行效率 |

### 6.2 质量报表

| 报表 | 内容 | 周期 |
|------|------|------|
| 质量月报 | 各检验类型合格率、不良率趋势 | 月度 |
| NCR 汇总表 | NCR 数量、严重程度分布、闭环情况 | 月度 |
| CAPA 跟踪表 | CAPA 执行进度、完成情况 | 月度 |
| 供应商质量排名 | 各供应商来料合格率排名 | 月度 |

## 七、跨域协作协议

### 7.1 事件内容

| 事件 | 关键字段 |
|------|----------|
| 质检触发事件 | 业务单据类型、业务单据编码、物料编码、仓库编码 |
| 质检结果事件 | 质检单编码、业务单据类型、业务单据编码、质检状态（ACCEPTED/CONDITIONAL/REJECTED）、让步理由（CONDITIONAL 时） |

### 7.2 业务域响应

| 质检结果 | 采购域响应 | 销售域响应 | 制造域响应 |
|----------|------------|------------|------------|
| ACCEPTED | 入库继续 | 出库继续 | 工序继续/入库继续 |
| CONDITIONAL | 入库继续（记录让步） | 出库继续（记录让步） | 工序继续/入库继续（记录让步） |
| REJECTED | 触发采购退货 | 阻止发货/触发销售退货 | 触发返工/报废 |

## 八、审计与追溯

### 8.1 质检追溯链

```
质检单 → 业务单据 → 物料批次 → 供应商/生产工单
        │
        └─► 可追溯检验时间、检验员、检验设备、检验方法
```

### 8.2 NCR 追溯链

```
NCR → 质检单 → 业务单据 → 纠正预防措施 → 效果验证
        │
        └─► 可追溯不合格原因、处理过程、改进措施、验证结果
```

### 8.3 审计日志

所有操作记录审计日志：
- 质检单创建/修改：操作人、时间、变更内容
- 质检结果录入：检验员、时间、实测值
- 让步接收审批：审批人、时间、理由
- NCR 创建/关闭：操作人、时间、状态变更

## 九、抽样检验规则

### 9.1 抽样类型

| 类型 | 适用场景 | 说明 |
|------|----------|------|
| 全检（100%） | 关键安全件、小批量 | 所有单位逐一检验 |
| 计数抽样（AQL） | 大批量来料/出货 | 按 ANSI/ASQ Z1.4（原 MIL-STD-105E），以不合格品数判定批合格 |
| 计量抽样 | 关键尺寸/性能 | 按 ANSI/ASQ Z1.9，以样本均值与规格界限判定 |
| 免检（SKIP） | 信誉供应商、低风险 | 跳过质检，直接入库 |

### 9.2 AQL 抽样方案

- **正常检验**（Normal）：初始方案，按样本量字码表 + AQL 值确定样本量与判定数（Ac/Re）。
- **加严检验**（Tightened）：连续批不合格时升级，样本量不变但判定标准更严（Ac 减小/Re 增大）。
- **放宽检验**（Reduced）：连续批合格时降级，样本量减少。
- **转移规则**：

```
正常 ←→ 加严 ←→ 暂停（暂停后需改进才恢复）
  ↑__________________________|
正常 → 放宽（连续 10 批合格）→ 正常（任一不合格即回退）
```

### 9.3 抽样配置

| 配置项 | 默认 | 说明 |
|--------|------|------|
| 抽样标准 | Z1.4 | Z1.4（计数）/ Z1.9（计量） |
| AQL 值 | 物料类别配置 | 严重/主要/次要缺陷分别设 |
| 检验水平 | II | I/II/III，影响样本量 |
| 转移规则启用 | true | false 时固定为正常检验 |

## 十、关键业务规则总结

1. **强制质检阻塞**：配置强制质检的物料，业务单据必须经质检合格才能继续流转
2. **关键项否决**：关键检验项不合格则整体判定为不合格
3. **让步审批**：让步接收需质量主管审批，记录让步理由
4. **NCR 强制**：不合格必须开 NCR，追踪到 CAPA 闭环
5. **CAPA 验证**：纠正预防措施需效果验证才能关闭 NCR
6. **模板优先级**：物料级别 > 物料类别级别 > 全局默认
