# 质量管理域页面设计要点

> 本文档定义质量管理域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-quality.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`inspection-integration.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **检验结果判定自动化**：质检读数录入后系统自动根据规格上下限判定合格/不合格，前端即时显示判定结果（绿色 PASS / 红色 FAIL）。
2. **不合格即触发 NCR**：质检单判定为不合格时，页面直接引导创建 NCR（不符合项报告），减少流程断裂。
3. **来料/制程/完工/出货四类检验统一入口**：四种检验类型使用同一套质检模板框架，通过 inspection_type 区分，页面结构复用。
4. **强制质检视觉约束**：若物料配置了 `inspection_required`，关联业务单据无法从"已审核"进入下一环节，直至关联质检单合格。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 |
|------|------|----------|--------|
| 质检单录入 | 表单+子表 | 质检员 | ★★★ |
| 质检结果录入 | 表单+读数子表 | 质检员 | ★★★ |
| 质检模板配置 | 表单+项目子表 | 质量工程师 | ★★★ |
| NCR 管理 | 列表+详情 | 质量工程师 | ★★★ |
| CAPA 管理 | 表单 | 质量工程师 | ★★★ |
| 质检仪表板 | 卡片式概览 | 质量主管 | ★★☆ |

## 各页面设计要点

### 质检单录入与结果判定

**页面入口**：质量管理 → 待检列表 → 执行检验

```
┌────────────────────────────────────────────────────────┐
│ 质检单: QC-2026-001  检验类型: 来料检验                  │
├────────────────────────────────────────────────────────┤
│ ┌─ 关联信息 ───────────────────────────────────────┐   │
│ │ 来源业务单: PURCHASE_RECEIVE-001 [查看]            │   │
│ │ 物料: MAT-001 螺丝 M6 × 1000 PCS                  │   │
│ │ 供应商: 某某五金  批次: LOT-2026-001                │   │
│ │ 质检模板: 螺丝 M6 来料检验 (v2)                   │   │
│ └───────────────────────────────────────────────────┘   │
├────────────────────────────────────────────────────────┤
│ 检验读数子表                                              │
│ ┌────┬────────┬──────┬──────┬──────┬──────┬────────┐  │
│ │ #  │ 检验项目│ 规格下限│ 上限 │ 实测值│ 单位  │ 判定   │  │
│ │ 1  │ 外径   │ 5.8   │ 6.2  │ 6.01 │ mm   │ ✅ PASS│  │
│ │ 2  │ 长度   │ 19.5  │ 20.5 │ 20.3 │ mm   │ ✅ PASS│  │
│ │ 3  │ 硬度   │ HRC30 │ HRC40│ HRC28│      │ ❌ FAIL│  │
│ │ 4  │ 表面粗糙度│ ≤1.6 │      │ 1.2  │ μm   │ ✅ PASS│  │
│ │ 5  │ 外观   │ 无锈蚀 │      │ 无锈蚀│      │ ✅ PASS│  │
│ └────┴────────┴──────┴──────┴──────┴──────┴────────┘  │
│ ────────────────────────────────────────────────────── │
│ 判定: ❌ 不合格 (4/5 PASS)  不合格项: 硬度                │
│ ────────────────────────────────────────────────────── │
│ 操作: [提交合格] [触发 NCR] [让步接收]                   │
│ ────────────────────────────────────────────────────── │
│ 备注: [___]  附件: [上传]                               │
└────────────────────────────────────────────────────────┘
```

**要点**：
- 规格值从质检模板自动带出，质检员只需录入"实测值"
- 判定列自动计算（实测值在规格内=PASS，否则=FAIL）
- 顶部显示"合格 X/Y 项"计数，全部 PASS 时整体判定"合格"
- 存在 FAIL 项时显示"触发 NCR"按钮（点击自动创建 NCR，预填不合格信息）
- "让步接收"按钮在部分不合格时可用，需填写让步理由 + 审批人
- 读数字段支持测量设备自动采集（预留接口）
- [注：ORM 字典 erp-qa/inspection-result 使用 ACCEPTED/REJECTED 标签，UI 显示 PASS/FAIL，需业务层映射。ErpQaInspection 尚无 templateId/lotNo 列，ErpQaInspectionTemplateLine 尚无 inspectionMethod 列，为设计意图待补充 ORM。]

## 跨页面导航流

```
采购入库/工单完工/销售出库 → [检测触发] → 质检待办
    ↓
质检模板 → [选择模板] → 质检单录入 → [读数] → [判定]
    ↓                    ↓          ↓
  ✅ 全部合格          ❌ 不合格    ⚠️ 让步接收
    ↓                    ↓
 更新业务单为"已通过"   创建 NCR
                         ↓
                       NCR 评审 → [创建 CAPA] → [措施执行] → [效果验证] → 关闭
```

| 设计点 | 参考来源 | 应用方式 |
