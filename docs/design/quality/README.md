# 质量管理域（quality）

## 目的

说明质量管理域的业务语义、质检单与业务单据的关联、不合格品处理（NCR）、纠正预防措施（CAPA）。质量域管理从进料到出货到制程的全链路质量管控。

## 边界

- 本域负责：质检单、质检模板与参数、质量目标、不符合项报告（NCR）、纠正预防措施（CAPA）、质量评审。
- 本域不负责：被检物料/SKU 主数据（master-data）；质检触发的退货/返工执行（purchase/sales/manufacturing 域，本域只反馈结果）。
- 持久化字段、字典、状态码以 `model/app-erp-quality.orm.xml` 为准。

## 核心业务对象

| 对象 | 业务含义 |
|------|----------|
| 质检单（Inspection） | 一次质量检验记录：检验类型、关联业务单据、物料、检验结果、状态 |
| 质检模板（InspectionTemplate） | 按物料配置的检验项目标准：参数、规格上下限、检验方法 |
| 质检读数（InspectionReading） | 质检单下的具体检验项实测值与判定 |
| 质量目标（QualityGoal） | 质量指标目标值（如合格率、不良率） |
| 不符合项报告（NonConformance） | 不合格事件的记录与追踪：现象、原因、影响、状态 |
| 纠正预防措施（Action） | 针对 NCR 的纠正/预防措施：类型、责任人、计划完成、效果验证 |
| 质量评审（Review） | 定期质量体系评审记录 |

### 质检单与业务单据的关联

质检单通过 `reference_type` + `reference_name` 关联业务单据：

| 检验类型（inspection_type） | 关联业务单据 | 触发时机 |
|----------------------------|--------------|----------|
| 来料检验（INCOMING） | 采购入库单 | 采购入库审核后 |
| 制程检验（IN_PROCESS） | 工单/作业卡 | 工序报工后 |
| 完工检验（FINAL） | 工单 | 工单完工后 |
| 出货检验（OUTGOING） | 销售出库单 | 销售出库审核后 |

## 状态机

质检单有独立状态机（待检→合格/让步接收/不合格），不符合项有独立状态机（待处理→评审中→已解决/已取消）。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 来料检验 | purchase | 采购入库审核触发质检；不合格反馈触发采购退货 |
| 出货检验 | sales | 销售出库审核触发质检；不合格反馈阻止发货或触发销售退货 |
| 制程/完工检验 | manufacturing | 工单报工/完工触发质检；不合格触发返工 |
| 物料/检验标准 | master-data | 质检模板按物料配置 |

### 质检对制造域的约束声明

质检判定直接影响制造工单状态迁移，双方必须显式声明：

| 质检判定 | 对工单状态的影响 | 约束声明位置 |
|----------|------------------|------------|
| ACCEPTED（合格） | 工单可从 INSPECTING → COMPLETED | 本文 + `manufacturing/state-machine.md` |
| CONDITIONAL（让步接收） | 工单可从 INSPECTING → COMPLETED（附让步记录） | 本文 + `manufacturing/state-machine.md` |
| REJECTED（不合格） | 工单停留在 INSPECTING，触发返工工单 | 本文 + `manufacturing/state-machine.md` |

> 双方文档都显式声明此约束，确保两个域的读者都能看到质检对工单状态的影响。

## 关键业务规则

1. **质检模板按物料配置**：每种物料可配置不同的检验项目与规格标准。
2. **强制质检**：物料配置 `inspection_required` 时，关联业务单据必须经质检合格才能继续流转。
3. **让步接收**：不合格品经审批后可让步接收（降级使用、挑选使用），记录让步理由与审批人。
4. **不合格触发**：不合格结果反馈给业务域，触发退货/返工/报废。
5. **NCR 追踪**：不合格事件必须开 NCR，追踪到纠正预防措施闭环。
6. **CAPA 闭环**：纠正预防措施需效果验证才能关闭。
