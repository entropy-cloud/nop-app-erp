# 销售合同（Sales Contract）

## 目的

销售合同是与客户签订的具有法律约束力的销售框架协议，定义销售条款、价格、付款条件、交付计划。合同与销售订单的关系是"框架 → 执行"：一个合同可生成多个销售订单，每个订单是合同的部分执行。

## 边界

- 本模块负责：合同生命周期、合同条款（价格/付款/交付）、合同变更、合同执行跟踪。
- 本模块不负责：客户主数据（master-data 域）；实际发货/开票/收款（sales 域销售订单执行）。
- ORM 实体见 `module-sales/model/app-erp-sales.orm.xml`（ErpSalSalesContract、ErpSalSalesContractLine、ErpSalSalesContractMilestone 等）。

## 核心流程

```
合同创建（DRAFT）
  │
  ├─ 合同类型：框架合同（FRAMEWORK）/ 一次性合同（ONE_TIME）/ 服务合同（SERVICE）
  ├─ 合同条款：价格表、付款条件（付款比例及账期）、交付期限
  └─（可选）里程碑计划：每期金额、预计日期、完成条件
       │
       ▼（审核）
合同生效（ACTIVE）
       │
       ├─ 按需创建销售订单（引用合同）
       │    └─ 订单行引用合同行，单价锁定为合同价
       │
       ├─ 里程碑跟踪
       │    └─ 里程碑完成 → 触发开票申请 / 收入确认
       │
       └─ 合同变更
            └─ 补充协议（AMENDMENT）：修改条款、金额，保留历史版本
       │
       ▼
合同完结（COMPLETED）
  ├─ 全部订单执行完毕 / 全部里程碑完成
  └─ 关闭前校验：无未结销售订单、无未开票金额
       │
合同终止（TERMINATED）
  └─ 提前终止（违约/协商解约），记录终止原因及责任归属
```

## 关键业务规则

1. **合同价格锁定**：销售订单引用合同行时，单价锁定为合同签约价。后续合同变更不追溯已生成的订单。
2. **合同总量控制**：所有引用该合同的订单行累计数量 + 金额不得超过合同约定总量（`ErpSalSalesContractLine.quantity × unitPrice`）。
3. **里程碑开票**：里程碑完成条件满足时（如交货完成、验收通过），可生成开票申请。里程碑金额自动汇总为合同应收。
4. **变更版本化**：每次合同变更（补充协议）生成新版本号，历史版本只读存档。变更保存后原价格/条款不受影响。
5. **信控关联**：引用合同的销售订单占用客户信用额度（`ErpMdPartner.creditLimit`），与普通订单一致。

## 与销售订单的关系

```
ErpSalSalesContract（合同）
       │
       └─ has many → ErpSalSalesOrder（销售订单）
                        ├─ 订单行 → ErpSalSalesContractLine（合同行，锁定单价）
                        └─ 订单金额汇总 → 合同已执行金额
```

- 非强制引用：销售订单可不引用合同直接创建（现货销售场景）。
- 引用合同后，订单审批时自动校验"合同总量未超限"。

## 配置选项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 合同自动编号规则 | orgId 内唯一 | 前缀 + 年月 + 流水号 |
| 里程碑开票模式 | MILESTONE_COMPLETE | MILESTONE_COMPLETE（里程碑完成时）/ SCHEDULED（按计划日期） |
| 超额控制 | HARD_BLOCK | HARD_BLOCK（超额禁止生成订单）/ WARNING（警告允许） |
