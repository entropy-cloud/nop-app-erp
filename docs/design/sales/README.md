# 销售域（sales）

## 目的

说明销售域的业务语义、单据生命周期、状态机与跨域协作。销售域管理向客户销售物料的完整业务闭环，是"应收（AR）侧"业务单据的来源。结构上与采购域镜像对称。

## 边界

- 本域负责：销售订单、销售出库、销售发票、收款、销售退货。
- 本域不负责：物料/客户主数据（master-data 域）；出库写库存（inventory 域，本域调用）；应收凭证生成（finance 域，本域单据触发）。
- 持久化字段、字典、状态码以 `module-sales/model/app-erp-sales.orm.xml` 为准。
- 跨域协作规则见 `../domain-design-guidelines.md`，全局流程见 `../flow-overview.md`。

## 核心业务对象

| 对象 | 业务含义 | 头/行 |
|------|----------|-------|
| 销售订单（SalesOrder） | 与客户确认的销售意向 | 头/行 |
| 销售出库单（SalesDelivery） | 实际发货单据，触发库存出库 | 头/行 |
| 销售发票（SalesInvoice） | 向客户开出的发票，应收凭证来源 | 头/行 |
| 收款单（Receipt） | 从客户收款 | 头/核销明细行 |
| 销售退货单（SalesReturn） | 客户退货，触发库存入库 | 头/行 |

### 单据关系

```
销售订单 → [部分/全部] → 销售出库 → 销售发票 → 收款 → 闭环
```

- 销售订单可多次出库（部分发货）。
- 出库后产生应收，开票时确认收入。
- 发票与收款分离：可先发货后开票，也可先开票后收款。

## 状态机

销售单据采用三轴状态分离（业务/审核/收款），与采购域镜像对称，按 `docs/skills/state-machine-business-review-prompt.md` 的 10 个审查维度设计。详细规则见 [`state-machine.md`](state-machine.md)。

## 跨域协作

| 协作场景 | 对端域 | 协作方式 |
|----------|--------|----------|
| 销售出库扣减库存 | inventory | 出库单审核 → `IErpInvStockMoveBiz` 生成出库移动单，含可用量校验 |
| 销售退货入库 | inventory | 退货单审核 → 生成入库移动单 |
| 销售发票触发应收凭证 | finance | 发票审核 → 财务域生成应收凭证（AR_INVOICE） |
| 收款触发收款凭证 | finance | 收款单审核 → 财务域生成收款凭证（RECEIPT） |
| 物料/客户/仓库引用 | master-data | 单据引用主数据编码 + `IErpMd*Biz` |

## 关键业务规则

1. **出库与开票可分离**：可先发货后开票（按出库金额暂估应收），也可开票时指定出库。
2. **部分发货**：一张销售订单可多次出库，订单未交量 = 订单数量 − 累计出库数量。
3. **收款核销**：收款与发票多对多，头级收款进度状态聚合行级核销结果。
4. **价税分离**：行级金额、税额、税率独立列。
5. **多币种**：单据引用币种，金额按业务日期汇率转换本位币。
6. **可用量校验**：出库前校验库存可用量（由库存域提供），不足时拒绝。
7. **赠品与折扣**：赠品行金额为 0 但计入数量；折扣可按行或单头，影响应收金额。
8. **退货退款**：退货生成的反向凭证（红字）由财务域处理，退款冲销原收款。

### 信用额度控制

销售订单审核时检查客户信用额度，按三级可配策略处理：

| 级别 | 策略 | 行为 |
|------|------|------|
| SOFT_WARNING | 软警告 | 超额度时弹窗提示，允许继续提交 |
| SPECIAL_APPROVAL | 特别审批 | 超额度时需额外审批人审批 |
| HARD_BLOCK | 硬拦截 | 超额度时直接拒绝提交 |

**额度计算**：客户信用额度 − 未结算应收余额（AR_INVOICE 未核销金额）− 未出库订单金额。配置项：`erp-sal.credit-check-level`（默认 SOFT_WARNING），按客户可覆盖。

## 与采购域的对称性

销售域的单据结构与状态机与采购域镜像对称，便于理解与维护：

| 采购域 | 销售域 | 对称关系 |
|--------|--------|----------|
| 供应商 | 客户 | 往来单位方向相反 |
| 采购订单 | 销售订单 | 意向单据 |
| 采购入库（写库存增加） | 销售出库（扣库存减少） | 库存方向相反 |
| 采购发票（应付） | 销售发票（应收） | 应收应付方向相反 |
| 付款单 | 收款单 | 资金方向相反 |
| 采购退货（出库） | 销售退货（入库） | 库存方向相反 |
| 付款状态（paidStatus） | 收款状态（receivedStatus） | 收付款进度 |

差异点：
- 销售域增加**可用量校验**（出库前校验库存是否充足），采购入库无此约束。
- 销售域支持**赠品与折扣**行，采购域通常无赠品（但可有采购折扣）。
- 销售域涉及**收入确认**（开票即确认收入），采购域涉及**费用确认**。
