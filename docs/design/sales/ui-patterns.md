# 销售域页面设计要点

> 本文档定义销售域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `model/app-erp-sales.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`returns.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **对称于采购**：销售域页面结构与采购域镜像对称（订单/出库/发票/收款/退货），差异点在于出库需要可用量校验、支持赠品/折扣行、退货涉及退款。
2. **可用量校验视觉反馈**：出库单审核时若库存不足，前端应有明确的可用量不足提示（当前可用量 / 需求数量 对比），而非仅后端返回错误。
3. **赠品与折扣显式标记**：赠品行（单价 0、数量非 0）必须在行上有显式标记（如标签"赠品"），折扣行或头折扣在金额计算中单独展示。
4. **价格管理优先级**：订单取价时按"手工价 > 价格清单 > SKU 默认档"优先级，前端显示价格来源标签（如"清单价""手工改价"）。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 |
|------|------|----------|--------|
| 销售订单编辑 | 表单+子表 | 销售员 | ★★★ |
| 销售出库编辑 | 表单+子表 | 仓管员 | ★★★ |
| 销售发票编辑 | 表单+子表 | 财务员 | ★★★ |
| 收款单编辑 | 表单+子表 | 财务员 | ★★☆ |
| 销售退货编辑 | 表单+子表 | 销售员 | ★★★ |

## 各页面设计要点

### 销售订单

**头区特殊字段**：
- 客户（必填）：M2M 弹窗选择，显示信用额度、应收余额（若超信用额度给予警告）
- 价格清单：下拉选择（关联客户主数据配置的价格清单），选定的价格清单影响明细行取价
- 付款条件、交货地址、业务员

**子表特殊行为**：
- 物料选择弹窗额外显示：库存可用量、最近销售价
- 价格自动填充逻辑：手工改价后使用特殊颜色标记，鼠标悬停显示"清单价: ¥XX"
- **赠品行**：行级别设有"赠品"开关，标记后单价锁定为 0、数量可编辑，库存扣减正常
- **折扣行**：行级别支持折扣率或折扣金额，行级折扣后金额参与汇总计算
- 头级折扣：整单折扣%输入框，自动按比例分摊到各明细行
- 可用量预检：保存时可选预检（非强制），显示物料可用量不足预警

### 销售出库

**与采购入库的差异**：
- 出库单审核前必须通过可用量校验，前端在子表行上显示"当前可用量"列
- 校验失败时高亮标记不可出库的物料行，并显示具体原因（可用量不足/批次过期/序列号已售）

**头区特殊字段**：
- 来源订单：[选择] 弹窗选择已审核销售订单，选中后复制订单行到子表
- 仓库（必填）、出库类型：下拉选择（正常出库 / 样品出库 / 赠品出库）
- 发货地址、物流信息（承运商、运单号）

**子表特殊行为**：
- 每行显示"订单数量 / 已出库数量 / 本次出库数量 / 可用量"四列
- 启用批次管理时：批次选择器需过滤该物料有效批次，显示批次库存量
- 启用序列号管理时：序列号录入弹窗支持扫描枪批量录入，校验序列号状态（未售）
- 赠品行正常出库扣库存（尽管单价为 0）

### 销售发票

**头区特殊字段**：
- 客户（必填）、客户发票号、发票类型（增值税专用/普通/收据） [注：当前 ORM 中 ErpSalInvoice 尚无 invoiceType 字段，设计意图，待补充 ORM]
- 发票日期、收款条件

**子表特殊行为**：
- 发票行关联出库单行（可选），关联后带出物料/数量/金额
- 价税分离自动计算，显示：不含税单价 × 数量 = 不含税金额，税额 = 不含税金额 × 税率，价税合计 = 不含税金额 + 税额

**状态信息区**：
```
┌─────────────────────────────────────┐
│ 收款状态: ● 未收                    │
│ 已核销金额: ¥ 0.00                  │
│ 关联收款单: [REC-001▶] [REC-002▶]   │
│ 凭证状态: ● 已过账                  │
│ 凭证号: ACC-202406-002              │
└─────────────────────────────────────┘
```

### 收款单

结构与采购付款单对称。核销明细子表显示该客户所有未收清发票，"自动核销"按先进先出分配。支持多账户收款（拆分到不同银行账户）。

### 销售退货

**头区特殊字段**：
- 来源出库单：[选择] 弹窗选择已审核销售出库单
- 退货类型：单选（部分退货 / 全额退货 / 换货）
- 退款方式：下拉选择（原路退回 / 其他账户 / 预收款抵扣 / 现金退款）

**子表特殊行为**：
- 从来源出库单导入出库行数据，显示"原出库数量 / 可退数量 / 本次退货数量"三列
- 若原出库单已开票，头区显示"已开票"标记，提示将生成红字发票
- 若原出库单已收款，头区显示"已收款"标记及所选退款方式

## 跨页面导航流

```
销售订单 → [审核] → 待出库
    ↓
销售出库 → [审核] → 扣库存 + 结转成本
    ↓
销售发票 → [审核] → 应收凭证
    ↓
收款单   → [审核] → 收款凭证 + 核销
    ↓
闭环
```

逆向追溯：销售出库详情 → [源订单] → [关联发票] → [关联收款单]
