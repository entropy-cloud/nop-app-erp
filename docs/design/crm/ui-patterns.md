# CRM 域页面设计要点

> 本文档定义 CRM 域关键业务页面的结构布局、交互模式与导航流程。
> 字段定义以 `module-crm/model/app-erp-crm.orm.xml` 为准，业务语义与状态机见 `state-machine.md`、`marketing.md`。
> 调研引用格式 `[源项目#要点]`，详见 `docs/analysis/erp-survey/`。

## 设计原则

1. **线索/商机统一视图**：单实体 + type 判别（LEAD/OPPORTUNITY），列表页通过筛选切换显示类型，不拆分两个页面。
2. **阶段驱动操作**：操作按钮和可编辑字段由 docStatus + stageId 联合控制，不依赖角色硬编码。
3. **活动时间线作为核心交互**：线索详情页以纵向时间线展示所有关联活动（通话/邮件/会议/备注），支持直接创建新活动。
4. **漏斗可视化**：商机阶段推进通过 Kanban 视图直观展示，支持拖拽跨阶段移动。
5. **UTM 归因自动填充**：外部渠道创建的线索自动携带活动归因信息，页面上显示为只读标签。

## 页面清单

| 页面 | 类型 | 主要用户 | 复杂度 | 对应实体 |
|------|------|----------|--------|----------|
| 线索列表 | 列表页 + 筛选面板 | 销售员 | ★★☆ | ErpCrmLead |
| 线索详情 | 仪表板式详情 | 销售员 | ★★★ | ErpCrmLead + ErpCrmEvent |
| 商机 Kanban | 看板视图 | 销售员 | ★★★ | ErpCrmLead（leadType=OPPORTUNITY） |
| 活动日历 | 日历视图 | 销售员 | ★★☆ | ErpCrmEvent |
| 阶段配置 | 表单 + 排序 | 管理员 | ★☆☆ | ErpCrmStage |
| 转化向导 | 分步弹窗 | 销售员 | ★★★ | Lead→Customer→Opportunity→Quotation |
| 营销活动列表 | 列表页 | 营销主管 | ★★☆ | ErpCrmCampaign |
| 营销活动详情 | 仪表板式详情 | 营销主管 | ★★★ | ErpCrmCampaign + ErpCrmLead |

## 通用页面模式

### 列表页结构（线索/营销活动）

```
┌────────────────────────────────────────────────────┐
│ [新建]  [批量操作▾]  [导出]                         │
│ ┌─ 筛选区 ──────────────────────────────────────┐ │
│ │ leadType: [●线索 ●商机]                        │ │
│ │ 线索来源: [下拉] 阶段: [下拉] 负责人: [选择]   │ │
│ │ 创建日期: [从] ~ [至]  关键字: [____]          │ │
│ │ [查询] [重置] [保存筛选方案▾]                   │ │
│ └───────────────────────────────────────────────┘ │
├────────────────────────────────────────────────────┤
│ 结果表格: (支持列排序/自定义列)                     │
│ ┌────────────────────────────────────────────────┐ │
│ │ # │ 名称/公司│ 联系人 │ 来源  │ 预期收入 │ 阶段  │ │
│ │ 1 │ 某某有限 │ 张三  │ 展会  │ ¥50,000 │ 谈判中│ │
│ │   │ [查看] [操作▾]                            │ │
│ └────────────────────────────────────────────────┘ │
│ 分页: [<] 1 2 3 ... [>]                            │
└────────────────────────────────────────────────────┘
```

**要点**：
- leadType 切换使用 Tab 或 Chip 组（"全部 / 线索 / 商机"），切换时刷新列表数据源
- 阶段列显示为彩色标签，颜色从 ErpCrmStage 配置读取
- 预期收入列右对齐，千分位格式，空值显示为"—"
- 操作按钮根据 docStatus 动态渲染：NEW 显示[编辑][验证][取消]，QUALIFIED 显示[跟进][转化][标记丢失]
- 支持"我的线索"筛选快捷方式（当前用户负责的线索）

- [注：归因信息中 campaignId 和 utmTerm/utmContent 在当前 ORM 中已设计，但页面是否需要全部展示需产品确认。ErpCrmLead 无 utmTerm/utmContent 字段，为设计意图待补充 ORM。]

## 各页面设计要点

### 商机 Kanban 视图

**页面入口**：CRM → 商机看板

```
┌──────────────────────────────────────────────────────────┐
│ [新建商机]  筛选: [我的] [团队] [全部]                     │
├──────┬────────┬────────┬────────┬────────┬────────┬──────┤
│ 新线索│ 已联系 │ 需求分析│ 方案演示│ 谈判中 │ 赢单   │ 丢失 │
│      │        │        │        │        │        │      │
│ [卡片]│ [卡片] │ [卡片] │ [卡片] │ [卡片] │ [卡片] │[卡片]│
│ 某公司│ 某公司 │ 某公司 │ 某公司 │ 某公司 │ 某公司 │某公司│
│ 张三  │ 李四   │ 王五   │ 赵六   │ 钱七   │ 孙八   │ 周九 │
│ ¥50K  │ ¥80K   │ ¥30K   │ ¥120K  │ ¥200K  │ ¥100K  │ 已关闭│
│ 拖拽→ │ 拖拽→  │ 拖拽→  │ 拖拽→  │ 拖拽→  │        │      │
└──────┴────────┴────────┴────────┴────────┴────────┴──────┘
```

**要点**：
- 每个阶段列从 ErpCrmStage 表动态读取，按 sequence 排序
- isWonStage 列（赢单）显示绿色背景，禁止拖出
- 丢失列（LOST 状态）显示灰色背景，只读
- 每个卡片显示：公司名、负责人、预期收入、阶段停留天数
- 拖拽触发 stageId 变更：校验 sequence 单向递增约束（不可回拖到前序阶段）
- 拖拽成功后自动记录 ErpCrmLeadConvLog
- 支持阶段列折叠（展开/收起）
- 参考 🟢 Odoo CRM kanban view + 🟢 Axelor CRM views

**页面入口**：系统设置 → CRM → 漏斗阶段

```
┌────────────────────────────────────────────────────────┐
│ 销售团队: [下拉选择]  (阶段按团队隔离)                   │
├────────────────────────────────────────────────────────┤
│ 阶段列表 (拖拽排序)                                      │
│ ┌────┬──────────────┬──────┬─────────┬────────┬────┐  │
│ │ 排序 │ 阶段名称    │ 概率%│ 赢单阶段 │ 颜色  │ 操作│  │

│ │ 2   │ 已联系       │ 20   │ ☐       │ 🟡     │ ✎ ✕│  │
│ │ 3   │ 需求分析     │ 30   │ ☐       │ 🟡     │ ✎ ✕│  │
│ │ 4   │ 方案演示     │ 50   │ ☐       │ 🟠     │ ✎ ✕│  │
```

- 删除阶段前检查是否有线索处于该阶段，有则阻止删除

### 转化向导

**触发场景**：线索/商机详情页 → 点击"转化"

```
步骤 1: 转化目标选择
├─ ● 转化为客户 + 商机  (LEAD → Customer + OPPORTUNITY)
├─ ○ 直接转化为商机     (LEAD → OPPORTUNITY，不创建客户)
└─ ○ 转化为报价单       (OPPORTUNITY with isWonStage → Quotation)
   [下一步]

步骤 2: 信息确认
├─ 客户信息 (从 Lead 派生，可编辑):
│  公司名: 某某有限公司  联系人: 张三
│  电话: 138xxxx        邮箱: zhang@xx.com
├─ 商机信息:
│  预期收入: ¥50,000   预期签单日: 2026-08-15
├─ 报价单信息 (商机→报价时显示):
│  报价日期: [自动]    有效期: 30天
│  明细行: (从商机信息填充或手动输入)
└─ [上一步] [确认转化]

步骤 3: 转化成功 ✓
├─ 已创建客户: P-2026-001 [查看▶]
├─ 已创建商机: OP-2026-002 [查看▶]
├─ 已创建报价单: QT-2026-001 [查看▶]
└─ [关闭]
```

**要点**：
- 转化不可逆，确认前二次确认弹窗
- 转化时校验：LEAD 类型不可直接转报价单（🟢 state-machine.md §4 异常路径）
- 转化后原 Lead 显示为已转化（CONVERTED），只读
- 参考 🟢 Axelor `ConvertLeadWizardService.java`

## 跨页面导航流

```
线索列表 → [新建] → 线索详情
    ↓                      ↓
[验证通过后]           [活动时间线] → [创建活动] → 活动日历
    ↓                      ↓
商机 Kanban → [阶段推进] → 活动详情浮层
    ↓                      ↓
[赢单阶段] → [转化向导] → 创建客户 → 创建报价单
    ↓
报价单详情 (sales 域)
```
