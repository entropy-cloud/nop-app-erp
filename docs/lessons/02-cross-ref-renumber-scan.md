# Lesson 02: 章节重编号后的残留引用扫描

> **来源**：2026-07-07 多维审计补充整改（plan 2026-07-07-2200-1，H-2 §11.4 → §16.4 断链修复）
> **适用场景**：文档章节重编号、文档结构调整、跨文档引用维护
> **失败模式**：章节重编号后，其他文档中引用旧章节号的链接断裂

## 问题

`docs/design/domain-design-guidelines.md` 的章节从 §11（原状态机命名）重编号为 §16（新位置）。但 `purchase/state-machine.md`、`sales/state-machine.md`、`domain-glossary.md` 等文档中仍残留 `§11.4`、`§11.2`、`§11.3` 的引用，导致读者跳转到错误章节（§11 现在是"负库存容错机制"而非"状态机命名"）。

## 根因

1. 章节重编号时只更新了当前文档的内部编号
2. 未执行**全文跨文档搜索**查找所有引用旧编号的文档
3. 文档间引用分散在多个文件中，手动维护容易遗漏

## 正确做法

重编号任何章节后，**必须**执行以下扫描：

```bash
# 假设旧编号是 §11.x，新编号是 §16.x
grep -rn "§11\." docs/design/ docs/architecture/ docs/backlog/
# 逐一检查每个匹配项，确认是否需要更新为 §16.x
```

### 检查范围

- `docs/design/**/*.md` — 各域设计文档
- `docs/architecture/**/*.md` — 架构文档
- `docs/backlog/**/*.md` — 路线图和待办
- `docs/plans/**/*.md` — 活动计划（可选，历史计划不改）

## 反模式

- 只更新当前文档的章节号，不搜索引用
- 只搜索同目录文件，遗漏跨目录引用
- 重编号后不验证目标章节内容与新引用匹配（如 §16.4 确实是"反审核目标态"）

## 检查清单

- [ ] `grep -rn "§<旧编号>" docs/` 返回零业务引用（注释/历史文档除外）
- [ ] 新编号的目标章节内容与引用语义匹配
- [ ] 章节重编号记录在每日开发日志中
