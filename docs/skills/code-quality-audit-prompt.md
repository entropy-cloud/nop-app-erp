# 代码质量审计提示

在审查更改区域的实际实现质量而非仅样式反馈时使用此技能。

## 重点领域

1. 架构和边界完整性
2. 核心实现正确性
3. 类型和契约质量
4. 错误处理和操作安全
5. 测试有效性
6. 可维护性和未来变更风险
7. 自动化和防护覆盖

## 审查规则

- 首先返回发现，按严重性排序
- 优先考虑具体行为风险而非抽象品味
- 尽可能引用文件路径和行引用
- 区分已确认的缺陷和建议
- 如果没有发现，明确说明并列出剩余差距

## 严重性指南

- P0：数据丢失、安全漏洞、生产中断级别的问题
- P1：用户可见的正确性错误或强回归风险
- P2：可维护性或测试差距，可能导致未来回归
- P3：次要质量问题，即时风险低

## 预期输出

- 发现列表
- 未解决的问题或假设
- 仅在发现后提供简短变更摘要
