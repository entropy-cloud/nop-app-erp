- **关键工程发现（既有移动加权平均已实现 + COGS 通道已立）**：① `StockMoveBookkeeper` 既有 bookIncoming（重算 `balance.avgCost`）/bookOutgoing（取 avgCost 为 COGS）/writeLedger 已实现移动加权平均，仅硬编码 costMethod——FIFO 缺失的根因是「无方法分派 + cost layer 无维护者」。② `InvPostingDispatcher` 已从 `ledger.getTotalCost()` 汇总为 `TOTAL_COST`——故 FIFO 只须让记账器对 FIFO 物料写出经 FIFO 计算的 ledger.unitCost/totalCost，**派发器零改动拾取**，COGS 经既有 SALES_OUTPUT/PURCHASE_INPUT 通道流动。
- **Phase 2（FIFO 策略 + 红冲对称 + 首次无成本拒绝）**：`FifoCostingStrategy.onIncoming` 追加 `ErpInvCostLayer`（incomingQuantity=remainingQuantity、unitCost、incomingDate、incomingMoveId、costMethod=FIFO），余额 avgCost 置空（FIFO 语义）/totalCost 累加；`onOutgoing` 按 incomingDate 升序多层消耗各层 remainingQuantity，汇总加权出库 unitCost（costing-methods.md §FIFO 出库逻辑步骤2-3），扣减余额 totalQuantity/totalCost；首次出库无 remainingQuantity>0 层抛 `ErpInvErrors.ERR_COST_NOT_AVAILABLE`。
  - **Decision（FIFO 红冲语义，执行者裁决）**：选 (a) 反向入库按原出库刷新的加权 unitCost 追加新层——而非 (b) 直接恢复被消耗层 remainingQuantity。理由：reverse 重走正常 DONE 流程致反向入库触发 onIncoming，(b) 恢复消耗层会与既有消耗记账双计；onOutgoing 末尾刷回 `line.unitCost`，reverse 透传至反向入库行，保证红冲后 Σ layer remaining×unitCost 恢复至原出库前（成本不变量，容差 0.01 舍入残差）。
