  - **诊断方法**：临时新建 `ErpAllWebPagesDiagnosticTest.java`（catch `Throwable` 而非 `Exception`——`StackOverflowError` 是 `Error` 不被原 test 的 `catch(Exception)` 捕获，会穿透 abort 整个 test），逐页面打印 `LOADING:` + `FAILED:` 定位到 4 个失败页面；通过 `git stash` + `mvn install -pl module-master-data/erp-md-web -am` 二分定位到 ErpMdMaterialCategory 单文件；通过逐字段剔除 query form cells 定位到 `parentId` cell。
  - **修复**：从 `ErpMdMaterialCategory.view.xml` 的 query form 移除 `<cell id="parentId" filterOp="eq"/>`（layout 中也对应移除 `parentId[上级分类]`）。理由：树形实体的 parentId 导航已由 `tree-list` grid + `add-child` action + tree-select 父节点选择控件完整提供，query 不需要再按 parentId 筛选；这与 F10 范式 ErpMdSubject 的 query form（`code/name/status/subjectClass`，无 parentId）一致。
