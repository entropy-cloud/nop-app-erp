### 2026-06-22 (修正 docs-for-ai 主键设计文档，支持 BIGINT 方案 + id 保留规则)

- 用户指出 docs-for-ai 中 `orm-model-design.md` 的主键设计节只展示了 VARCHAR(36) 一种方案，过于强制。
- 实际 Nop 平台支持两种主键方案：
  - VARCHAR(36) + `tagSet="seq"`（UUID，推荐分布式场景）
  - BIGINT + `tagSet="seq-default"`（自增，适用于单机/简单场景）
- 修正 `orm-model-design.md`：
  - 主键设计节从"固定配置"改为"方案 A/B"两种选择
  - 新增选择建议表（按场景推荐方案）
  - 外键字段节同步更新，支持两种类型的外键配置
- 修正 `code-style.md`：
  - 主键格式从 `VARCHAR(36), tagSet="seq"` 改为 `VARCHAR(36)+tagSet="seq" 或 BIGINT+tagSet="seq-default"`
- 新增 `id` 属性名保留规则文档：
  - `id` 是系统保留属性名，专用于主键
  - 单主键实体：主键列名为 `id` 时必须设置 `ext:allowIdAsColName="true"`，否则会被重命名为 `id_`
  - 复合主键实体：各主键列保持原始名称（如 `partitionId`、`sid`），系统创建 `OrmCompositePKModel` 作为 `id` 属性
  - 推荐做法：单主键使用 `name="id"` + `ext:allowIdAsColName="true"`
  - 用户纠正：只有主键是 `id` 的情况下，设置 `ext:allowIdAsColName` 才有意义
- 新增字段类型选择指南：
  - BigDecimal 使用场景：金额、数量、汇率等精确数值
  - financial 字段必须使用 domain 标注业务语义（amount/price/quantity/exchangeRate/interestRate/taxRate）
  - precision/scale 选择建议表
  - VARCHAR 长度是字节数不是字符数，中文需除以 3
  - DATE/DATETIME/TIMESTAMP 类型选择指南
- 修正 domain 作用说明（用户纠正）：
  - domain 是类型模板，用于定义可复用的列类型配置（stdDomain/stdSqlType/stdDataType/precision/scale）
  - 不是用于标记业务语义，而是类型复用机制
  - 源码依据：`OrmModelInitializer.syncDomains()` 将 domain 的属性复制到 column
- 修正字典 code 与 value 区别说明（用户纠正）：
  - `code` 是 Java 常量后缀，与字典名前缀组合生成常量（源码：`_{moduleClassPrefix}DaoConstants.java.xgen`）
  - 常量命名：`{DICT_PREFIX}_{CODE}`，如 `ORDER_STATUS_PENDING`
  - 常量类型：int 类型生成 `int`，string 类型生成 `String`
  - 生成的接口：`_{moduleClassPrefix}DaoConstants`
- 结论：当前项目的 88 个实体使用 BIGINT 主键是可接受的，只需补充 `tagSet="seq-default"` 标注。
