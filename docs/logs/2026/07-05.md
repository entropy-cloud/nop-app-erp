### 关键决策

- **核销落库机制**：复用 0300-3 `create`+`post` 路径（不重写核销原语），引擎只负责生成候选行。rejected：引擎直接操作 ArApItem 跳过核销单（丢失审计载体 + 绕过约束校验）。
- **分摊策略**：三策略经 config + 方法参数覆盖。BY_AMOUNT 非唯一金额退化为 UNMATCHED（报告列出等待人工），不强行 1:N 匹配。
- **定时执行载体**：nop-job-local 直调 BizModel（单次量未达数万阈值，过度工程 rejected）。nop-batch 分 chunk 迁移留 Deferred。
- **双面一致性口径**：经 ArApItem.sourceBillCode = 域级发票 code 关联，比较 finance settled vs domain paidAmount/receivedAmount。仅报告 + 告警，不自动修复。
