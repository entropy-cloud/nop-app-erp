### 关键决策与执行现实

- **跨域依赖环约束**：master-data 是基础域（purchase/inventory 等均 compile 依赖 master-data-dao），master-data 不得反向依赖下游域。Phase 2/3 价格表解析与删除引用校验均经 SPI 端口模式（`IErpMdSupplierPriceResolver`/`IErpMdSkuReferenceChecker`，erp-md-dao/spi 包），默认无实现时空转，下游接线归 Deferred。避免了 plan 原文「I*Biz 注入或 IDaoProvider 直接查 ErpPurSupplierPriceList」会构成的编译依赖环。
- **G1~G5 模型缺口处理**：G1 barcode DB 唯一索引 / G2 SKU.status 列 / G3 minPrice 列 / G4 销售 PriceList 实体 均为保护区域不改 ORM，以应用层校验 + Deferred successor 承接；G5 priceValidationLevel 列默认值 `"20"` 孤儿以字典 string 值 OFF/WARN/HARD 为权威编码，非字典值统一 WARN 宽松。
- **测试桩 beans 注册**：`@NopTestConfig(testBeansFile = "/erp/md/beans/test-*.beans.xml")` 显式声明（镜像 mfg `test-aps-load-source.beans.xml` 范式），VFS 测试目录不自动合并需 testBeansFile 指定。
- **(IServiceContext context) 末参 + @Optional 可空参**：所有 IBiz 方法遵循 skill P2 规则（context 末参）；可空参经 `@Optional` 注解（GraphQL 默认拒绝 null/empty args）。
