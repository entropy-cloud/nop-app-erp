### 关键技术发现

- **`@Nullable` optional inject 解决 reactor-cycle**：quality-service 测试不引入 purchase-service/sales-service（避免 reactor 环），`NcrReturnOrchestrator` 的 `@Inject IErpPurReturnBiz`/`IErpSalReturnBiz` 在无桩测试中导致 bean 初始化失败。加 `@jakarta.annotation.Nullable` 使 Nop IoC `DefaultBeanClassIntrospection` 检测到 `isAnnotationPresent(Nullable.class)` → `optional=true` → `resolveBeanByType` 返回 null → `buildInjectResolver` 返回 null → 属性跳过注入。运行时（全 app）或测试（test-mock beans）有对应 bean 时正常注入。
- **物理出库简化 Decision**：NCR_SCRAP 凭证贷记存货（1401）已表达报废消耗会计影响；物理库存量（ErpInvStockBalance.totalQuantity）同步扣减需调 `IErpInvStockMoveBiz.generateMove`，但出库移动会触发 InvPostingDispatcher SALES_OUTPUT 双计存货贷方。本期简化为凭证过账独占，物理量扣减归 inventory 域 successor。
