外部注入三层模型

按"介入的具体程度"把用户每次介入分三层

A 类：明确指明 Nop 平台机制

给机制名 + 配置项 + 源码位置。AI 自由度极低 → 9 次，全在 06-22~07-03

B 类：指明工程/设计原则方向

给原则或质疑，不给 API。AI 中等自由度 → 9 次，散布全程

C 类：只让 AI 自查/对比

只提要求，不暗示答案。AI 高自由度 → 绝大多数，尤其 07-04 后

关键洞察：用户有清晰边界意识——平台机制明确介入，业务逻辑让 AI 对标开源自查

A 类案例：用户当"平台知识权威"

三个 Takeaway

AI 辅助开发的本质 = 知识转移 + 系统化质量门控

人机协作有边界

用户 = 平台权威（A 类）/ 方向指南（B 类）/ 非业务设计者（C 类）

早期密集介入 → 知识转移完成 → 后期放手

这不是"AI 自动写代码"，而是"AI 辅助 + 人类定向 + 系统化质量门控"

AGE 工作流有效

八阶段递进，进入条件 + 退出门控明确

强制双审计：草案拦方向错误 + 结束验交付完整

内部信息循环 + 自我纠偏：评分偏差回顾、子代理误判纠正、方法论持续演进

模型驱动的产品化路径可行
